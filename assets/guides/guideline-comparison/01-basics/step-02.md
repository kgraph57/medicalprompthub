## 2. 比較対象となるガイドラインの選定とAIへの入力準備

### 2.1. 比較対象ガイドラインの選定基準

AIによる比較分析の質は、入力するガイドラインの選定にかかっています。以下の基準に基づき、比較対象を明確にしてください。

1.  対象疾患・病態の同一性: 比較するガイドラインが、同じ疾患や病態を対象としていることを確認します。
2.  発行機関の多様性: 異なる国や地域の学会、専門機関（例：日本循環器学会、AHA/ACC、ESCなど）が発行しているものを選択することで、推奨の多様性を把握できます。
3.  発行年の近接性: 最新のエビデンスを反映しているかを確認するため、発行年が近いものを選ぶのが理想です。ただし、意図的に新旧を比較する場合はこの限りではありません。
4.  比較したい臨床的疑問: どの治療法、診断法、管理法について比較したいのか、具体的な臨床的疑問（PICO形式など）を明確にします。

### 2.2. AIに入力するためのテキストデータ準備

LLMは、PDFや画像形式のガイドラインを直接処理するのが苦手です。AIに正確に分析させるためには、ガイドラインの該当箇所をテキストデータとして準備する必要があります。

| データ準備方法 | メリット | デメリット |
| :--- | :--- | :--- |
| 手動コピー＆ペースト | 必要な推奨事項部分のみを正確に抽出できる。 | 時間と労力がかかる。見落としや誤入力のリスクがある。 |
| PDFからのテキスト抽出 | 大量のテキストを一度に取得できる。 | 図表やレイアウト情報が混ざり、テキストが不整形になりやすい。 |
| ウェブサイトからの取得 | 比較的整形されたテキストが得られることが多い。 | サイトの構造によっては抽出が難しい場合がある。 |

### 2.3. テキストデータの前処理と整形

抽出したテキストは、AIが処理しやすいように整形します。

1.  不要な情報の削除: 著作権表示、図表のキャプション、参考文献リストなど、比較分析に不要なテキストは削除します。
2.  推奨事項の特定: 比較対象となる推奨事項（Recommendation）のセクションとその推奨度（Class/Level of Evidence）を明確に区別できるようにマークアップします。
3.  ガイドライン名の明記: 複数のガイドラインのテキストを一つのプロンプトに入力する場合、各推奨事項の前に必ずどのガイドラインからの引用かを明記します。

### 実用的なプロンプト例：推奨事項の構造化と整形

抽出したテキストをAIに提供し、比較しやすい形式に構造化させます。

```prompt
以下の2つのガイドラインから抽出したテキストデータを提供します。
あなたは医療情報アナリストとして、各ガイドラインの「推奨事項」と「推奨度/エビデンスレベル」のみを抽出し、以下のMarkdown形式の表に整形してください。

# ガイドラインA: [ガイドライン名と発行年]
[ガイドラインAの推奨事項に関するテキスト]

# ガイドラインB: [ガイドライン名と発行年]
[ガイドラインBの推奨事項に関するテキスト]

---
## 出力形式
| ガイドライン名 | 推奨事項の番号/見出し | 推奨事項の要約 | 推奨度/エビデンスレベル |
| :--- | :--- | :--- | :--- |
| ガイドラインA | | | |
| ガイドラインB | | | |
```

### 注意点とTips

*   Tips: 著作権に配慮し、AIに入力するテキストは必要最小限に留め、分析結果を公開する際は必ず原典を引用してください。
*   注意点: LLMのコンテキストウィンドウ（一度に処理できるテキスト量）には限界があります。長大なガイドライン全体ではなく、比較したい特定のテーマに関する推奨事項のセクションに絞って入力してください。

## 3. 構造化データを用いた推奨事項の横断的比較分析

### 3.1. 比較分析の実行と焦点の設定

ステップ2で整形した構造化データ（Markdownテーブルなど）は、AIが比較分析を行うための最適なインプットとなります。このステップでは、具体的な臨床的疑問に基づき、AIに共通点と相違点を明確に抽出させます。

手順:

1.  臨床的疑問の再確認: 「[疾患名]における[治療法/診断法]の推奨は、ガイドラインAとBでどう異なるか？」といった具体的な疑問を再確認します。
2.  構造化データの入力: ステップ2で作成した表形式のデータをプロンプトに含めます。
3.  比較軸の指定: AIに対し、単なるテキストの比較ではなく、「推奨の方向性（推奨する/推奨しない）」「推奨度」「対象患者集団」といった具体的な比較軸を指定します。

### 3.2. 共通点と相違点の明確化

AIに、比較結果を視覚的に分かりやすい形式（表や箇条書き）で出力させます。

### 実用的なプロンプト例：推奨事項の共通点と相違点の抽出

```prompt
あなたは医療ガイドラインの比較分析専門家です。
以下のMarkdownテーブルには、[疾患名]に関する3つのガイドライン（A, B, C）の主要な推奨事項が構造化されています。

[ここにステップ2で作成したMarkdownテーブルを貼り付け]

このデータに基づき、以下のタスクを実行してください。

1.  共通点の抽出: 3つのガイドライン全てで、推奨の方向性（例：強く推奨される、治療の第一選択である）が一致している推奨事項を3つ抽出し、簡潔に要約してください。
2.  相違点の特定: 推奨の方向性や推奨度（例：AではClass I、BではClass IIa）に最も大きな違いが見られる推奨事項を2つ特定し、その違いを具体的に指摘してください。
3.  出力形式: 結果を以下の形式で出力してください。

---
## 比較分析結果

### 共通点（3ガイドライン全てで一致）
- [推奨事項1の要約]
- [推奨事項2の要約]
- [推奨事項3の要約]

### 主要な相違点
| 臨床的テーマ | ガイドラインAの推奨（推奨度） | ガイドラインBの推奨（推奨度） | 相違点の要点 |
| :--- | :--- | :--- | :--- |
| [テーマ1] | | | |
| [テーマ2] | | | |
```

### 注意点とTips

*   Tips: 比較の粒度を調整するために、「推奨の文言が完全に一致するもの」と「推奨の意図が一致するもの」を分けて分析させることで、より深い洞察が得られます。
*   注意点: AIが「相違点」と判断したものが、実はガイドラインの対象集団の違い（例：小児と成人）によるものである可能性があります。抽出された相違点については、必ず次のステップで背景を深掘りしてください。

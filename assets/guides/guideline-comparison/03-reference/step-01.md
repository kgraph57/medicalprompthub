## 6. 比較分析の自動化とテンプレート化

### 6.1. 比較分析ワークフローのテンプレート化

ガイドライン比較分析は、疾患が変わっても基本的な手順は共通しています。この手順をテンプレート化することで、分析にかかる時間を大幅に短縮し、再現性を高めることができます。

テンプレート化の要素:

1.  役割設定（ペルソナ）: 「あなたは医療情報アナリストです」といった、AIの役割を固定する。
2.  入力データ形式の指定: ステップ2で定義したMarkdownテーブル形式を標準とする。
3.  出力形式の指定: ステップ3で定義した「共通点/相違点」の表形式を標準とする。
4.  分析軸の固定: 「推奨の方向性」「推奨度」「対象集団」など、必ず比較すべき軸を固定する。

### 6.2. 比較分析を自動化する「マスタープロンプト」

これまでのステップで用いた複数のプロンプトを統合し、一度の入力で分析を完了させる「マスタープロンプト」を作成します。

### 実用的なプロンプト例：マスタープロンプト（ワンショット分析）

```prompt
# マスタープロンプト：多ガイドライン比較分析

## 1. 役割設定
あなたは、複数の診療ガイドラインを横断的に比較分析し、臨床的洞察を提供する専門家です。

## 2. 入力データ
以下のMarkdownテーブルには、[疾患名]に関する3つのガイドライン（A, B, C）の主要な推奨事項が構造化されています。

[ここにステップ2で作成したMarkdownテーブルを貼り付け]

## 3. 分析指示
上記のデータに基づき、以下の3つのタスクを順に実行し、結果をまとめて出力してください。

### タスクA: 共通点の抽出
3つのガイドライン全てで、推奨の方向性が一致している推奨事項を3つ抽出し、簡潔に要約してください。

### タスクB: 主要な相違点の特定と分析
推奨の方向性や推奨度に最も大きな違いが見られる推奨事項を2つ特定し、その違いを具体的に指摘してください。さらに、その相違点の背景にある推定要因（エビデンスレベル、対象集団の違いなど）を考察してください。

### タスクC: 臨床的示唆の導出
分析結果に基づき、日本の臨床現場でこれらのガイドラインを適用する際に、特に注意すべき点や、統合的な推奨として考慮すべき点を2点記述してください。

## 4. 出力形式
結果を以下のMarkdown形式で出力してください。

---
### 共通点（3ガイドライン全てで一致）
- [要約1]
- [要約2]
- [要約3]

### 主要な相違点と背景分析
| 臨床的テーマ | ガイドラインAの推奨 | ガイドラインBの推奨 | 相違点の推定要因 |
| :--- | :--- | :--- | :--- |
| [テーマ1] | | | |
| [テーマ2] | | | |

### 臨床的示唆
1. [示唆1]
2. [示唆2]
```

### 注意点とTips

*   Tips: マスタープロンプトは、分析のたびに「疾患名」や「ガイドライン名」などの変数を書き換えるだけで済むように設計してください。
*   注意点: ワンショット分析は便利ですが、AIが途中で論理を破綻させるリスクも高まります。複雑な分析を行う場合は、実践編のようにステップを分けて実行する方が、結果の精度は高くなります。

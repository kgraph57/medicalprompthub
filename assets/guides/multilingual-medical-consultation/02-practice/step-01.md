# 診察開始時のAI通訳導入と問診プロンプト

## 1. 診察開始時の定型的な導入手順

AI通訳を導入する際、患者に安心感を与え、スムーズに診察を始めるための定型的な手順を確立することが重要です。これにより、患者の不安を軽減し、AIの利用に対する信頼性を高めます。

### 手順：AI通訳の導入

1.  自己紹介と挨拶: 医師（または医療従事者）が患者に挨拶し、自身の名前を伝えます。
2.  AI通訳の使用説明: 「本日は、正確なコミュニケーションのためにAI通訳システムを使用します」と説明します。
3.  プライバシーへの配慮: 「会話の内容は、診察の目的以外には使用されません。ご安心ください」と付け加えます。
4.  同意の確認: 患者にAIの使用について同意を求めます（例：「よろしいでしょうか？」）。
5.  AIへの開始指示: 事前に設定したプロンプト（01-basics/step-02.md参照）をAIに送信し、通訳を開始させます。

### 2. 問診を円滑に進めるためのプロンプト活用

問診は、患者の主訴や病歴を正確に把握するための最も重要なステップです。AI通訳の精度を最大限に引き出し、効率的な問診を行うために、質問の構造化とプロンプトの調整を行います。

#### プロンプト例：主訴の深掘り（OPQRST/PQRST）

主訴（Chief Complaint）を深掘りする際、AIに質問のフレームワークを理解させ、それに基づいて通訳させることで、より体系的な情報を引き出せます。

```
あなたは、日本の医療機関で働く医療従事者と、日本語を解さない外国人患者との間の会話を、正確かつリアルタイムで通訳するAIです。

追加の指示:
医療従事者が患者の主訴について質問する際、以下のPQRSTフレームワークに基づいた質問が続くことを認識し、患者の回答に含まれる時間経過、症状の性質、悪化・軽減因子などの情報を特に正確に翻訳してください。

PQRSTフレームワーク:
-   P (Provocation/Palliation): 悪化・軽減因子（何で悪くなり、何で良くなるか）
-   Q (Quality): 症状の性質（どんな痛みか、鋭い、鈍い、締め付けられるなど）
-   R (Region/Radiation): 部位と放散（どこが痛いか、どこに広がるか）
-   S (Severity): 重症度（10段階でどのくらいか）
-   T (Time): 時間経過（いつから、どのくらいの頻度か）

開始指示: 準備ができたら、「通訳を開始します。話し始めてください。」と応答してください。
```

### 3. Tips：会話のテンポと誤訳への対処

*   短く区切って話す: AIが正確に翻訳するためには、医療従事者も患者も、一度に長い文章を話すのではなく、意味のまとまりごとに短く区切って話すよう心がけます。
*   確認を怠らない: 重要な情報（例：アレルギー、服用中の薬、手術歴）については、翻訳された内容を鵜呑みにせず、「私が理解した内容で合っていますか？」と患者に確認を求めるステップを挟みます。
*   誤訳の修正: 明らかな誤訳が発生した場合は、すぐにAIを停止し、より簡単な言葉に言い換えるか、筆談や図を用いて確認します。AIに「この単語の翻訳が不正確でした。別の表現を試してください」と指示することも有効です。

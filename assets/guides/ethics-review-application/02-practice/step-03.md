# 02-practice/step-03.md: 個人情報保護とデータ管理計画の厳密化

![データ保護とプライバシー](/assets/guides/ethics-review-application/data_protection.png)

## 個理審査の最重要項目：個人情報保護とデータ管理

倫理審査で最も厳しくチェックされ、差し戻し（修正指示）の原因となりやすいのが、個人情報保護とデータ管理に関するセクションです。AIを活用することで、このセクションの記述を、法規制やガイドラインに準拠した、抜け漏れのない厳密な計画に仕上げることができます。

### 具体的な手順

1.  データフローの明確化: 研究開始から終了までのデータが辿る経路（収集、匿名化、保管、利用、廃棄）を箇条書きで整理します。
2.  匿名化レベルの定義: どの段階で、どの程度の匿名化（例：連結可能匿名化、非連結匿名化）を行うかを明確にします。
3.  AIによるリスク評価と対策提案: データフローと匿名化レベルをAIに入力し、「データ漏洩のリスク」と「不正利用のリスク」を評価させ、それに対する具体的な技術的・組織的対策を提案させます。

### 実用的なプロンプト例

データ管理計画の厳密化と、想定されるリスクへの対策を記述させるためのプロンプトです。

```prompt
あなたは、医療情報セキュリティの専門家です。
以下の[研究データ管理計画の骨子]に基づき、倫理審査申請書の「個人情報保護とデータ管理」セクションの記述を作成してください。

制約条件:
1.  記述は、「個人情報の保護に関する法律」および「医学系研究に関する倫理指針」に準拠していること。
2.  以下の項目について、具体的な対策を記述すること：
    *   データ収集時: 誰が、どこで、どのようにデータを収集し、識別子を付与するか。
    *   匿名化: 匿名化の方法（例：氏名削除、生年月日を年齢に変換）と、匿名化後のデータの保管場所。
    *   保管とアクセス: データの保管場所（例：鍵のかかるキャビネット、パスワード保護されたサーバー）、アクセス権限を持つ者、保管期間。
    *   廃棄: 研究終了後のデータの具体的な廃棄方法。
3.  データ漏洩が発生した場合の対応手順を、緊急連絡先を含めて簡潔に記述すること。

[研究データ管理計画の骨子]:
-   収集データ: [例：診療録、検査結果、アンケート]
-   識別子: [例：氏名、生年月日、ID番号]
-   匿名化のタイミング: [例：データ入力後、速やかに]
-   保管場所: [例：研究室内の専用サーバー]
```

### 注意点とTips

*   技術的対策の具体性: 「パスワードで保護する」だけでなく、「二要素認証を導入した研究専用のクラウドストレージに保管する」など、具体的な技術的対策を記述することで、審査員に安心感を与えます。
*   データ廃棄の明記: 研究終了後のデータ廃棄方法（例：物理的な破砕、専用ソフトウェアによる完全消去）を明確に記述することは、倫理的配慮の重要な証拠となります。

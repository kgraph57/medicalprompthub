# 03-reference/step-02.md: 申請書作成プロセスのテンプレート化と継続的な改善

## AI活用プロセスのテンプレート化による効率化

倫理審査申請書の作成は、一度きりの作業ではありません。研究テーマが変わるたびに、同様のプロセスを繰り返す必要があります。AIを活用した一連の作業（ステップ1～6）をテンプレート化することで、次回以降の申請書作成時間を大幅に短縮し、研究者が本来集中すべき研究活動に時間を割くことができます。

### 具体的な手順

1.  プロンプト集の整理: ステップ1から6で使用した「役割設定」「制約条件」「骨子案」を含むプロンプトを、研究分野や申請書のタイプ（例：介入研究、観察研究）ごとに分類し、テンプレートとして保存します。
2.  申請書構造のMarkdown化: 所属機関の申請書の必須項目をMarkdown形式のテンプレートとして作成し、AIが生成したテキストを流し込むだけで完成するように構造化します。
3.  AIによるフィードバックの学習: 審査委員会からの「修正指示（差し戻し）」の内容をAIに入力し、「次回以降、この指摘を避けるためには、申請書のどの部分をどのように記述すべきか」を分析させます。この分析結果をプロンプトテンプレートに反映させることで、AIの活用精度を継続的に向上させます。

### 実用的なプロンプト例

審査委員会からのフィードバックを分析し、次回以降の申請書作成に活かすためのプロンプトです。

```prompt
あなたは、倫理審査のフィードバック分析コンサルタントです。
以下の[審査委員会からの修正指示]を分析し、次回以降の申請書作成に役立つ3つの具体的な教訓を抽出してください。

教訓の形式:
1.  教訓の要点: [例：IC文書におけるリスクの記述が抽象的であった]
2.  具体的な改善策: [例：リスクを「身体的」「精神的」「プライバシー」に分類し、それぞれ具体的な事例を記述する]
3.  次回使用するプロンプトへの反映案: [例：IC文書作成プロンプトに「リスクは3分類し、具体的な事例を含めること」という制約条件を追加する]

[審査委員会からの修正指示]:
[ここに、審査委員会から受け取った修正指示の全文を貼り付け]
```

### 注意点とTips

*   プロンプトのバージョン管理: 改善されたプロンプトは、日付やバージョン番号を付けて管理し、常に最新かつ最も効果的なプロンプトを使用するようにしましょう。
*   AIの役割の多様化: 申請書作成だけでなく、「研究計画書（Protocol）の作成」「中間報告書のドラフト作成」「利益相反（COI）申告書のチェック」など、倫理審査に関連する他の文書作成にもAIの役割を広げることができます。

## 臨床シナリオに基づいたメールのドラフト作成

基本編で学んだ「構造化プロンプトの型」を、実際の臨床シナリオに適用し、コンサルテーションメールのドラフトをAIに作成させます。このステップでは、電子カルテや診療記録から抽出した情報を、AIが最も効率よく処理できる形式で提供することが鍵となります。

### 1. シナリオ設定と情報の抽出

今回は、「原因不明の持続する発熱」を主訴とする患者について、感染症専門医にコンサルテーションを行うシナリオを設定します。

| 項目 | 抽出情報（匿名化済み） |
| :--- | :--- |
| 患者情報 | 65歳、女性、ID: 12345、既往歴: 慢性関節リウマチ（MTX内服中） |
| 主訴 | 3週間続く38℃台の発熱 |
| 現病歴 | 3週間前より発熱、近医で抗生剤投与（セフトリアキソン）されるも改善なし。入院時より血液培養陰性。関節症状の悪化なし。 |
| 検査データ | WBC 12,000 (好中球 85%)、CRP 15.0 mg/dL、フェリチン 800 ng/mL。画像検査（胸腹部CT）では明らかな感染巣なし。 |
| 現在の診断 | 不明熱（FUO）の鑑別中。MTX関連の薬剤熱、成人Still病、潜在性感染症などを考慮。 |
| 相談内容 | 鑑別診断の進め方、特にMTXの一時中止の是非、および追加で考慮すべき特殊検査（例：骨髄検査）について。 |

### 2. ドラフト作成のためのプロンプト例

抽出した情報を「入力データ」として組み込み、AIにメールのドラフト作成を依頼します。

```
役割:
あなたは、総合内科に勤務する医師です。

タスク:
以下の[入力データ]に基づき、感染症専門医の[専門医の氏名]先生宛に、不明熱患者のコンサルテーションメールのドラフトを作成してください。

制約・形式:
1.  件名、本文、署名の3部構成とすること。
2.  件名には【不明熱コンサルト】と明記し、患者の年齢と主訴を含めること。
3.  本文は、挨拶、患者情報、経過、検査データ、相談内容、結びの順で構成すること。
4.  敬語を適切に用い、プロフェッショナルなトーンを維持すること。

入力データ:
-   患者: 65歳、女性、既往歴: 慢性関節リウマチ（MTX内服中）
-   主訴: 3週間続く38℃台の発熱
-   経過: 近医で抗生剤投与されるも改善なし。血液培養陰性。関節症状の悪化なし。
-   検査データ: WBC 12,000、CRP 15.0、フェリチン 800。CTで明らかな感染巣なし。
-   現在の診断: 不明熱（FUO）の鑑別中。薬剤熱、成人Still病、潜在性感染症などを考慮。
-   相談内容: 鑑別診断の進め方、MTXの一時中止の是非、追加で考慮すべき特殊検査（例：骨髄検査）について。
```

### 注意点とTips

*   情報の階層化: 検査データは、正常値ではなく異常値や特筆すべき値のみを記載することで、専門医が短時間で重要な情報にアクセスできるようにします。
*   「役割」の重要性: AIに「総合内科の医師」という役割を与えることで、専門医に対して「ある程度の鑑別は進めている」という前提で話を進める、適切なトーンのメールが生成されやすくなります。

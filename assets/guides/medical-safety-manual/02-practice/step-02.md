## 4. マニュアル本文の自動生成と既存文書との統合

### 章・節ごとのマニュアル本文の生成

前ステップで確定した構成案とリスクシナリオに基づき、AIに具体的なマニュアル本文を生成させます。この際、マニュアルの読者（医療従事者）の行動を促すような、簡潔で実践的なトーンを指定することが重要です。

1.  プロンプトの設計: ターゲット読者、目的（行動規範）、参照すべき情報源（RAGで参照可能な内部文書やガイドライン）を明確に指示します。
2.  段階的な生成: マニュアル全体を一括で生成するのではなく、章や節ごとに生成させることで、AIの出力の精度を高め、検証を容易にします。

### 実用的なプロンプト例：急変時対応マニュアルの生成

```
あなたは救急医療の専門家であり、新人看護師向けのマニュアルを作成しています。
以下の条件と参照情報を基に、「病棟での急変時対応」の章の本文を、箇条書きとフローチャート形式を組み合わせて作成してください。

条件：
1. トーン：冷静かつ行動志向（「〜しなさい」ではなく「〜する」）
2. 構成：初期対応（発見からコールまで）、チーム対応（役割分担）、記録・報告の3つのセクションに分ける。
3. 参照情報：[RAGシステムが参照する院内の急変時対応プロトコルの要約]

特に、「AIが生成した」という痕跡を残さず、人間が書いたような自然で実践的な文章にしてください。
```

### 既存文書（手順書など）との統合戦略

マニュアルには、既に検証され、現場で使われている既存の手順書やプロトコルを組み込む必要があります。AIは、これらの既存文書を新しいマニュアルのトーン＆マナーに合わせてリライトし、統一感を持たせるのに役立ちます。

| 統合の課題 | AIによる解決策 | プロンプトのヒント |
| :--- | :--- | :--- |
| 文体の不統一 | 既存文書を新しいマニュアルのトーン（例：簡潔、行動志向）にリライトさせる。 | 「以下の手順書を、新人でも理解できるように、箇条書きで簡潔かつ行動志向の文体にリライトしてください。」 |
| 冗長な表現 | 既存文書から重要なポイント（KISS原則：Keep It Simple, Stupid）を抽出し、要約させる。 | 「以下の長文から、医療従事者が緊急時に記憶すべき核となる手順を3点に絞って抽出してください。」 |
| 図表の作成 | テキストベースの手順をフローチャートやチェックリストのMarkdown形式に変換させる。 | 「以下の手順を、Markdown形式のフローチャート（またはチェックリスト）に変換してください。」 |

### 注意点とTips

*   ハルシネーション対策の徹底: AIが生成した本文は、必ず参照情報（RAGで参照した内部文書や公的ガイドライン）と照らし合わせて、事実誤認（ハルシネーション）がないかを確認してください。
*   現場での検証: 生成された草案は、必ず現場の医療従事者（特にマニュアルの主要な利用者）にレビューしてもらい、「本当に現場で使えるか」という視点でフィードバックを得てください。

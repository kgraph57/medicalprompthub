## 2. AIツールの選定とセキュアな環境構築

### 医療安全マニュアル作成に適したAIツールの選定

AIツールを選定する際は、機密性の確保と出力の信頼性を最優先します。

| 選定基準 | 詳細 | 考慮すべきAIモデル/サービス |
| :--- | :--- | :--- |
| セキュリティ | データが学習に利用されないこと、暗号化が徹底されていること、HIPAA/GDPRなどの規制に対応していること。 | 企業向けAPI（例：OpenAI Enterprise、Azure OpenAI Service）、オンプレミス型LLM。 |
| カスタマイズ性 | 医療機関固有の過去のインシデントデータや既存マニュアルを学習させ、ファインチューニングできる機能。 | ファインチューニング機能を持つモデル、RAG（Retrieval-Augmented Generation）構成が容易なサービス。 |
| 出力の信頼性 | ハルシネーション（誤情報生成）のリスクが低いこと、根拠となる情報源を提示できること。 | 医療分野に特化した学習済みモデル、またはRAG構成による内部文書参照。 |
| コストとスケーラビリティ | 継続的な利用コスト、将来的な利用拡大への対応力。 | 従量課金制のAPI、またはオープンソースモデルの自前運用。 |

### 医療機関におけるセキュアなAI利用環境の構築

AIを医療安全マニュアル作成に利用する際は、情報漏洩リスクを最小限に抑えるための環境構築が不可欠です。

1.  閉域ネットワークの利用: 可能な限り、インターネットから隔離された専用の閉域ネットワーク内でAIサービスを利用します（例：Azure OpenAI ServiceのVNet統合）。
2.  RAG（Retrieval-Augmented Generation）の導入: 医療機関の内部文書（インシデントレポート、既存マニュアル）を外部に送信せず、内部のデータベースを参照させるRAG構成を採用します。これにより、機密データをAIの学習に利用されるリスクを回避しつつ、最新かつ正確な内部情報に基づいたマニュアル草案を生成できます。
3.  アクセス制御の徹底: AIツールへのアクセス権限を、医療安全管理者やマニュアル作成担当者など、特定のユーザーに限定します。

### 実用的なプロンプト例：セキュリティポリシーの草案作成

AIにセキュリティガイドラインの草案を作成させ、環境構築の指針とします。

```
あなたは医療情報セキュリティの専門家です。
当院（500床規模の総合病院）が、医療安全マニュアル作成のために大規模言語モデル（LLM）を導入するにあたり、以下の項目を含む「AI利用セキュリティポリシー」の草案を作成してください。

1. 利用可能なAIサービスの定義（例：データが学習に利用されないことを保証するAPIのみ）
2. 入力データに関する規定（例：個人情報、機密情報の入力禁止、匿名化手順）
3. 出力結果の利用に関する規定（例：最終確認の義務付け、責任の所在）
4. ログの管理と監査に関する規定
```

### 注意点とTips

*   RAGの優先: 医療機関の機密性の高い情報（インシデントレポートなど）を扱う場合は、外部APIにデータを送信するのではなく、RAG構成を最優先で検討してください。
*   プロンプトのバージョン管理: セキュリティポリシーの策定やマニュアルの改訂に利用したプロンプトは、Gitなどのツールでバージョン管理し、監査に備えてください。

# 緩和ケア計画立案ガイド：実践編 ステップ2

## 症状マネジメント：個別化された症状緩和計画の立案

緩和ケア計画の重要な柱の一つは、患者の身体的・精神的な苦痛を和らげる症状マネジメントです。AIは、患者の個別データと最新のエビデンスを統合し、最も効果的で副作用の少ない症状緩和計画を立案するための支援を行います。

### 1. AIによる症状マネジメントの支援プロセス

AIは、以下のステップで症状マネジメントを支援します。

| ステップ | AIの具体的な役割 | 医療従事者の役割 |
| :--- | :--- | :--- |
| データ統合 | 症状評価スケール（NRS, VAS, POSなど）、投薬履歴、検査結果、既往歴を統合し、症状と関連要因の相関関係を分析。 | データの正確性を確認し、AIの分析結果を臨床的な視点から解釈。 |
| 治療オプション提示 | 症状、併存疾患、現在の投薬状況に基づき、最新のガイドラインやエビデンスに基づいた複数の治療オプション（薬物療法、非薬物療法）を提示。 | 患者の価値観、希望、経済状況を考慮し、提示されたオプションから最適なものを選択。 |
| 投薬相互作用チェック | 複数の薬剤の組み合わせにおける相互作用や副作用のリスクを迅速にチェックし、警告を発する。 | AIの警告に基づき、処方内容を調整し、患者にリスクを説明。 |

### 2. 実用的なプロンプト例：疼痛マネジメント計画の最適化

複雑な病態を持つ患者の疼痛マネジメント計画を、エビデンスに基づき最適化するためのプロンプトです。

#### プロンプト：多角的データに基づく疼痛マネジメント計画の提案

\`\`\`
あなたは、難治性疼痛のマネジメントに特化した緩和ケア薬剤師です。以下の患者データに基づき、現在の疼痛マネジメント計画を評価し、副作用を最小限に抑えつつ、疼痛を効果的にコントロールするための最適化案を提案してください。

患者データ:
*   主要症状: 疼痛（NRS 7/10、体動時増強）、倦怠感（POS 3/5）。
*   現在の投薬: [現在のオピオイドの種類、用量、投与経路、レスキュー薬の使用頻度]、[その他の併用薬（抗うつ薬、抗てんかん薬など）]
*   既往歴: [腎機能、肝機能、消化器系の問題など、投薬に影響する情報]
*   目標: 疼痛をNRS 4/10以下にコントロールし、意識の明晰さを保つこと。

提案の要件:
1.  現在の計画の課題点（例：副作用リスク、鎮痛効果の不足）を指摘してください。
2.  課題点を解決するための具体的な投薬調整案（薬剤の変更、増量、投与経路の変更、補助薬の追加など）を、その根拠となるエビデンス（例：WHO三段階除痛ラダー、特定のガイドライン）とともに提示してください。
3.  提案された計画における主要な副作用リスクと、その予防・対処法を明記してください。
\`\`\`

### 3. 注意点とTips

*   エビデンスの検証: AIが提示するエビデンスやガイドラインは、必ず最新かつ信頼できる情報源（例：PMID、専門学会のガイドライン）に基づいているかを確認し、必要に応じて原文を参照してください。
*   非薬物療法の統合: AIは薬物療法に偏りがちです。マッサージ、アロマテラピー、音楽療法などの非薬物療法も、患者の希望と状況に合わせて計画に統合することを忘れないでください。

# 02-practice/step-02.md: 複数試験の比較と最適な選択肢の特定

## 複数の臨床試験を比較する

特定の疾患や介入に関する臨床試験を検索すると、しばしば複数の候補が見つかります。患者にとって最適な選択肢を見つけるためには、これらの試験を並列で比較検討する必要があります。AIは、この比較検討プロセスを、構造化されたテーブル形式で実行することで支援します。

### 実用的なプロンプト例：比較検討表の作成

目的: 複数の臨床試験の概要テキストを入力し、主要な比較項目を抽出したテーブルを生成させる。

```
あなたは臨床試験の専門家です。以下の3つの臨床試験の概要を比較し、医療従事者が意思決定に役立つように、指定された項目を含むMarkdown形式の比較表を作成してください。

【試験A概要】
[試験Aの概要テキスト]

【試験B概要】
[試験Bの概要テキスト]

【試験C概要】
[試験Cの概要テキスト]

【比較項目】
1.  試験ID
2.  フェーズ
3.  対象疾患/患者集団の特異性
4.  主要介入（薬剤名など）
5.  主要評価項目（Primary Outcome）
6.  募集状況（Recruiting/Completedなど）
7.  実施施設（主要な施設または国）

【出力形式】
Markdown形式のテーブルで出力してください。
```

AIの期待される応答（例）:

| 比較項目 | 試験A (NCT01234567) | 試験B (NCT09876543) | 試験C (NCT00001111) |
| :--- | :--- | :--- | :--- |
| フェーズ | Phase 3 | Phase 2b | Phase 3 |
| 対象疾患/患者集団 | 進行性肺癌（PD-L1高発現） | 早期肺癌（術後補助療法） | 進行性肺癌（全患者対象） |
| 主要介入 | 免疫チェックポイント阻害薬X | 薬剤X + 化学療法 | 薬剤Y + 放射線療法 |
| 主要評価項目 | 全生存期間（OS） | 無再発生存期間（RFS） | 客観的奏効率（ORR） |
| 募集状況 | Recruiting | Completed (結果待ち) | Terminated (安全性懸念) |
| 実施施設 | 日本、米国、欧州 | 日本国内のみ | 米国のみ |

## 最適な試験の特定と意思決定支援

比較表を基に、AIに患者の状況や臨床的ニーズに最も合致する試験を特定させることができます。

### 実用的なプロンプト例：最適試験の選定理由の明確化

目的: 比較表と特定の患者情報に基づき、最も適切な試験とその理由を論理的に説明させる。

```
あなたは臨床意思決定支援AIです。上記の比較表と、以下の患者情報に基づき、最も推奨される臨床試験（A, B, Cのいずれか）とその理由を詳細に説明してください。

【患者情報】
- 疾患: 進行性肺癌
- PD-L1発現: 高発現
- 治療歴: なし（ファーストライン治療を検討中）
- 居住地: 東京都内（国内施設へのアクセスを優先）

【出力形式】
1.  推奨試験: [試験IDと名称]
2.  推奨理由（詳細）:
    - 臨床的適合性:
    - 実施状況の適合性:
    - アウトカムの重要性:
```

AIの期待される応答（例）:
1.  推奨試験: 試験A (NCT01234567)
2.  推奨理由（詳細）:
    - 臨床的適合性: 患者はPD-L1高発現の進行性肺癌であり、試験Aの対象集団（PD-L1高発現）に最も特異的に適合している。
    - 実施状況の適合性: 試験Aは現在「Recruiting」であり、患者が参加できる可能性がある。試験Bは完了済み、試験Cは安全性懸念で中止されており、選択肢から除外される。
    - アウトカムの重要性: 主要評価項目が「全生存期間（OS）」であり、進行性肺癌のファーストライン治療において最も重要な臨床的アウトカムである。

---
注意点とTips:
*   情報の正確性: AIは入力されたテキストに基づいて比較を行います。元の試験概要のテキストが正確であることを確認してください。
*   患者の意向: 最終的な意思決定には、患者の価値観や治療に対する意向を考慮に入れる必要があります。AIの推奨はあくまで情報提供の一つとして活用しましょう。

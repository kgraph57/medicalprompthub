# 02-practice/step-01.md: 検索結果の迅速な要約と評価

## 臨床試験概要のAIによる迅速な要約

ClinicalTrials.govやUMIN-CTRなどで検索を実行し、関心のある臨床試験を見つけたら、その試験概要（Summary）や詳細説明（Description）をAIに読み込ませることで、内容を迅速に把握できます。特に、長文のプロトコルや専門用語が多い記述を理解するのに有効です。

### 実用的なプロンプト例：試験概要の構造化要約

目的: 臨床試験の概要テキストを入力し、医療従事者が知りたい主要な情報を構造化して抽出・要約させる。

```
あなたは臨床試験の専門家です。以下の臨床試験の概要を読み、医療従事者向けに以下の項目を抽出・要約してください。

【臨床試験概要】
[ここにClinicalTrials.govの「Brief Summary」や「Detailed Description」のテキストを貼り付ける]

【抽出・要約項目】
1.  試験の目的（Primary Objective）: この試験が最も知りたいことは何か？
2.  対象患者（Inclusion/Exclusion Criteriaの要点）: どのような患者が対象か？
3.  介入内容（Intervention）: どのような治療や診断が行われるか？
4.  主要評価項目（Primary Outcome Measure）: 試験の成功を測る主要な指標は何か？
5.  フェーズと実施状況: 試験のフェーズ（例：Phase 3）と現在の募集状況（例：Recruiting）

![臨床試験フェーズ](/Helix/assets/guides/clinical-trial-search/trial_phases.png)
6.  一言評価: この試験が臨床的意義を持つ可能性について、簡潔に評価してください。
```

AIの期待される応答（例）:
1.  試験の目的: 〇〇疾患に対する新規薬剤Xの、標準治療Yに対する優越性を、主要評価項目である無増悪生存期間（PFS）で評価する。
2.  対象患者: 18歳以上の組織学的に確認された進行性〇〇癌患者。過去にXの治療歴がある者、重度の肝機能障害を持つ者は除外。
3.  介入内容: 介入群は新規薬剤Xと標準治療Yの併用、対照群はプラセボと標準治療Yの併用。
4.  主要評価項目: 無増悪生存期間（PFS）。副次評価項目は全生存期間（OS）と奏効率（ORR）。
5.  フェーズと実施状況: 第III相試験。現在、参加者を募集（Recruiting）中。
6.  一言評価: 標準治療に対する新規薬剤の追加効果を検証する重要な試験であり、結果次第では標準治療の変更につながる可能性がある。

## 評価と次のアクションの決定

AIによる要約は、その試験が自身の臨床疑問や患者に適しているかを判断するためのスクリーニングとして機能します。

| 要約結果のポイント | 評価の視点 | 次のステップ |
| :--- | :--- | :--- |
| 対象患者が一致しない | 包含/除外基準を詳細に確認する。 | 患者の特性と照らし合わせ、参加可能性を検討する。 |
| 介入内容が不明瞭 | プロトコルの詳細（可能であれば）をAIに読み込ませる。 | 実施施設や担当医師に問い合わせる。 |
| 主要評価項目が臨床的に重要でない | 試験のバイアスリスクを評価する（次ステップで詳述）。 | 別の試験を検索するか、メタアナリシスを探す。 |

---
注意点とTips:
*   著作権と機密性: ClinicalTrials.govなどの公開情報を使用してください。未公開のプロトコルや機密情報をAIに入力することは避けてください。
*   出力形式の指定: AIの回答の質は、プロンプトで指定する出力形式に大きく依存します。上記のように構造化された形式を常に要求しましょう。

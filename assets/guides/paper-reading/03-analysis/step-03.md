# Step 3: AIに「要約して」と頼むだけ！〜論文分析の自動化〜

### 📋 このステップの目的

ここが、このガイドの心臓部です。
AIという超優秀なアシスタントに、面倒な要約と分析を丸投げします。
従来、私たちがウンウン唸りながらやっていた作業をAIに任せることで、私たちはもっと創造的で、人間らしい「考える」仕事に集中できます。

### ⏱️ 所要時間

10〜15分

### 🎯 達成基準

- 論文の各セクションの要点を、AIを使ってサクッと抽出できている。
- 論文の「一番言いたいこと」を、自分の言葉で1〜2文で語れる。
- 「で、これって臨床でどう使えるの？」という問いに、明確に答えられる。

### 📝 詳細手順

![AIツールの比較](/assets/guides/paper-reading/images/ai_tools_comparison.png)

#### 手順1: 論文PDFをAIに「渡す」

まずは、相棒となるAIに論文PDFを渡すところから。
お好みのツールを選んでください。

- ChatGPT: PDFを直接アップロードOK。一番手軽かも。
- Claude: 長い論文や複雑な内容もお任せあれ。
- Gemini: Google謹製。PDFの要約から質問応答までスムーズ。
- NotebookLM: 複数の論文を同時に比較・分析するならコレ。

#### 手順2: 「構造化要約」をオーダーする

AIに「この論文、いい感じにまとめといて」と頼みます。
以下の「型」で要約させると、後で見返すときに最高に便利です。

```
# 論文の構造化要約

## 基本情報
- タイトル, 著者, ジャーナル, 発行年

## この研究の「問い」 (PICO)

## 研究デザイン

## 主要な結果

## 臨床での「使い道」

## この研究の「弱点」
```

#### 手順3: 「質問」で深掘りする

要約を読んだら、AIと対話しましょう。
優秀な研修医に質問する感覚でOKです。

- 「で、結局一番大事なことって何？」
- 「これ、明日の外来からどう使える？」
- 「この研究のツッコミどころは？」
- 「似たような研究って他にある？」

### 💡 AIへの「話し方」

#### プロンプト1: 「この論文、イケてる感じで要約しといて」

```
# 指示
この論文を読んで、以下の「型」で構造化要約を作って。僕が後で見てすぐ分かるように、イケてる感じで頼む。

# 論文
[論文のPDFをアップロード]

# 要約の「型」
- 基本情報: タイトル, 著者, ジャーナル, 発行年, DOI
- この研究の問い (PICO): 
- 研究デザイン: 
- 主要な結果: 
- 臨床での使い道: 
- この研究の弱点: 
- 結論 (一言で): 
```

#### プロンプト2: 「で、これって臨床でどう使うの？」

```
# 指示
この論文の結果、実際の臨床でどう応用できるか、具体的なアクションプランを3つ提案して。

# 論文の要約
[AIが作った要約を貼り付け]
```

### ⚠️ よくある「落とし穴」

- 失敗: AIの要約を鵜呑みにして、原典を読まない。
  - 対処法: AIはあくまで「アシスタント」。
重要な数字や結論は、必ず自分の目で原文を確認しましょう。
AIも間違うことはあります。

- 失敗: 曖昧な指示で、AIを困らせる。
  - 対処法: 「要約して」より「PICOと結果と臨床的意義を3行でまとめて」の方が、AIは的確に動いてくれます。

### 💪 上級者への道

- 「俺専用プロンプト」を作る: 「循環器内科医の視点で、この論文の意義を教えて」のように、自分の専門性を加えると、アウトプットの質が劇的に上がります。
- AIの「使い分け」: サクッと要約ならChatGPT、複数論文の比較ならNotebookLM、のように、用途でツールを使い分けるのがプロ。

# 実践編 - ステップ1: 診断フローの図解化

## はじめに

このステップでは、診断フローを図解化する方法を学びます。
診断フローは、臨床現場でよく使用される図解で、診断プロセスを段階的に表示します。

---

## 診断フロー図の基本構造

診断フロー図には以下の要素が含まれます：

- **開始点**: 症状や所見
- **判断点**: 診断の分岐点（Yes/No）
- **診断**: 最終的な診断
- **治療**: 治療方針

---

## 効果的なプロンプト例

### 例1: 胸痛の診断フロー

```
以下の胸痛の診断フローを図解化してください：

【開始】
患者の胸痛

【判断1】
急性冠症候群の可能性？
- Yes → 心電図、心筋マーカー検査
- No → 次の判断へ

【判断2】
肺塞栓の可能性？
- Yes → D-ダイマー、CTPA
- No → 次の判断へ

【判断3】
大動脈解離の可能性？
- Yes → 胸部CT
- No → 次の判断へ

【判断4】
その他の原因？
- 消化器疾患、筋骨格系疾患など

【視覚的な要素】
- 判断点をダイヤモンド型で表示
- 診断を四角形で表示
- 矢印でフローの流れを示す
- Yes/Noを明確にラベル付け
```

### 例2: 発熱の診断フロー

```
以下の発熱の診断フローを図解化してください：

【開始】
患者の発熱

【判断1】
感染症の可能性？
- Yes → 血液培養、画像検査
- No → 次の判断へ

【判断2】
自己免疫疾患の可能性？
- Yes → 自己抗体検査
- No → 次の判断へ

【判断3】
悪性腫瘍の可能性？
- Yes → 画像検査、生検
- No → その他の原因を検討

【視覚的な要素】
- フローチャート形式で表示
- 各判断点を明確に区別
- 診断への経路を矢印で示す
```

---

## プロンプトのポイント

### 1. 判断点を明確にする

各判断点で何を評価するかを明確にします。

### 2. Yes/Noの分岐を明確にする

各判断点でのYes/Noの分岐を明確にします。

### 3. 診断への経路を明確にする

各診断に至る経路を明確にします。

### 4. 視覚的な構造を指定

フローチャートの形式を指定します。

---

## 実践例

実際にNanobananaで診断フロー図を作成してみましょう：

1. 上記のプロンプト例をコピー
2. NotebookLMのNanobananaに貼り付け
3. 「生成」ボタンをクリック
4. 生成された図解を確認
5. 必要に応じてプロンプトを修正して再生成

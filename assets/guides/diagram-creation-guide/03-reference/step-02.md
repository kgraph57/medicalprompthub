# 応用編 - ステップ2: 応用テクニックとトラブルシューティング

## はじめに

このステップでは、Nanobananaを使った医学図解作成の応用テクニックと、よくある問題の解決方法を学びます。

---

## 応用テクニック

### 1. 複雑な内容の段階的図解化

複雑な内容は、複数の図解に分けて作成します。

**例：**
```
以下の内容を3つの図解に分けて作成してください：

図解1: 正常な生理的プロセス
図解2: 疾患による変化
図解3: 治療による改善
```

### 2. 既存の図解の改善

生成された図解が期待通りでない場合、具体的な修正指示を追加します。

**例：**
```
以下の図解を改善してください：
- 矢印の色を赤から青に変更
- ラベルのフォントサイズを大きく
- 背景色を白に変更
```

### 3. スタイルの統一

複数の図解を作成する場合、スタイルを統一します。

**例：**
```
以下の図解を作成してください：
- 色スキーム: 青、赤、緑を使用
- フォント: ゴシック体
- 背景: 白
- 矢印: 太い実線
```

---

## トラブルシューティング

### 問題1: 図解が期待通りに生成されない

**解決方法：**
- プロンプトをより具体的にする
- 視覚的な要素を明確に指定する
- 例を追加する

### 問題2: 医学的内容が正確でない

**解決方法：**
- 医学的な用語を正確に使用する
- 専門的な内容を確認する
- 必要に応じて専門家に確認する

### 問題3: 図解が複雑すぎる

**解決方法：**
- 内容を簡略化する
- 複数の図解に分ける
- 重要な要素のみに絞る

---

## ベストプラクティス

### 1. プロンプトの設計

- 明確で具体的な指示を出す
- 視覚的な要素を指定する
- 医学的な正確性を確保する

### 2. 図解の品質管理

- 生成された図解を確認する
- 必要に応じて修正する
- 専門家に確認を求める

### 3. 継続的な改善

- フィードバックを反映する
- プロンプトを改善する
- 新しいテクニックを試す

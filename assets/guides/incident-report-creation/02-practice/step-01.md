# 02-practice/step-01.md: 根本原因分析（RCA）の補助：AIに「なぜ」を深く掘り下げさせる

## 表面的な原因で終わらせない：AIを活用した真の原因究明

インシデントレポートの最も重要な目的は、真の根本原因（Root Cause）を特定し、再発を防止することです。しかし、多忙な現場では「ヒューマンエラー」という表面的な原因で分析が止まってしまいがちです。AIは、客観的な事実経過に基づき、「なぜなぜ分析」などの手法を適用することで、真の原因を深く掘り下げる強力な補助ツールとなります。

### 1. 根本原因分析（RCA）におけるAIの役割

AIは、人間が陥りやすい認知バイアス（例：個人への責任追及、安易な結論への飛びつき）を排除し、論理的かつ構造的に原因を分析するのに役立ちます。

| RCAのステップ | AIの活用方法 |
| :--- | :--- |
| 事実の整理 | ステップ02で作成した事実経過の構造化。 |
| 原因の深掘り | 「なぜなぜ分析」や「要因連関図」のドラフト作成。 |
| 分類と特定 | 原因を「人的要因」「組織的要因」「環境要因」などに分類し、根本原因候補を提示。 |

### 2. プロンプト例：AIによる「なぜなぜ分析」の実行

基本編で作成した事実経過のドラフトをAIに渡し、インシデントに至った背景にある複数の要因を段階的に掘り下げさせます。

プロンプト:
> あなたは医療安全管理の専門家であり、インシデントの根本原因分析（RCA）を支援します。以下の【事実経過】に基づき、「なぜなぜ分析」を5段階以上実行し、インシデントの発生につながった組織的・構造的な要因を特定してください。
>
> 【事実経過】
> 患者による内服薬の過剰服用（自己服用）が発生した。14:00に看護師Xが患者Y（80代男性、認知症あり）に降圧剤を配薬したが、他の患者対応のためA室を離れた。14:30頃、患者Yが自力で起き上がり、ベッドサイドの手の届く範囲に置かれていた薬を全て服用した。14:45に発見され、医師に報告。バイタルサインに異常はないが経過観察中である。
>
> 分析の視点:
> 1.  患者が薬を自己服用できたのはなぜか？
> 2.  看護師が薬を手の届く範囲に置いたのはなぜか？
> 3.  看護師が離室せざるを得なかった背景には何があるか？

### 3. AIの分析結果の解釈とレポートへの反映

AIが生成した「なぜなぜ分析」の結果は、そのままレポートに記載するのではなく、真の原因として最も妥当なものを選択し、簡潔にまとめます。

*   例（AIの分析結果）: 「看護師が離室せざるを得なかった」→「他の患者の緊急対応があった」→「病棟全体の看護師配置が手薄だった」→「夜勤明けのスタッフが日勤帯の業務を一部兼任していた」
*   レポートへの反映: 根本原因として「病棟の標準的な看護師配置基準と、突発的な緊急対応時の人員リソースのミスマッチ」を挙げ、組織的要因として記載します。

#### Tips：AIに「システム思考」を促す

「ヒューマンエラー」を原因とするのではなく、「エラーを誘発したシステム」に焦点を当てるようAIに指示することで、より本質的な原因究明が可能になります。プロンプトに「個人の責任ではなく、システムやプロセス上の問題点を指摘せよ」と明記しましょう。

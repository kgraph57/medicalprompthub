# 02-practice/step-03.md: レポートの最終レビューと客観性の担保

## AIを「医療安全監査役」として活用する

インシデントレポートは、事実の記録であると同時に、組織の安全文化を反映する公的な文書です。最終提出前に、AIにレポート全体を第三者の視点でレビューさせ、客観性、論理性、そして医療安全の観点から問題がないかを確認します。

### 1. AIによる最終レビューの観点

AIは、人間が作成した文章に含まれる無意識のバイアスや論理の飛躍を指摘するのに優れています。以下の観点から、レポートの完成度を高めます。

| レビュー観点 | AIへの指示内容 | 期待される指摘事項 |
| :--- | :--- | :--- |
| 客観性 | 感情的・推測的な表現の有無をチェック。 | 「〜としか考えられない」などの主観的な表現。 |
| 論理性 | 事実経過、原因分析、対策の間の論理的な繋がりを評価。 | 「原因」と「対策」が対応していない部分。 |
| 網羅性 | 報告書として必要な要素（5W1H、RCA、対策）が全て含まれているか。 | 発生後の患者への心理的ケアに関する記載の欠落など。 |
| 責任追及の排除 | 特定の個人を不当に非難する表現がないか。 | 「看護師Xの不注意」といった表現。 |

### 2. プロンプト例：レポート全体の客観性・論理性のレビュー

完成したレポートのドラフト全体をAIに渡し、上記の観点からレビューを依頼します。

プロンプト:
> あなたは、医療安全管理委員会の委員長です。以下の【インシデントレポート・ドラフト】を、客観性、論理性、そして責任追及の排除という3つの観点から厳しくレビューしてください。
>
> 指摘事項の出力形式:
> 1.  客観性に関する指摘: 修正すべき主観的な表現とその理由。
> 2.  論理性に関する指摘: 事実、原因、対策の間の論理的な飛躍や矛盾点。
> 3.  責任追及の排除に関する指摘: 特定の個人を不当に非難している可能性のある表現。
> 4.  総合評価: レポートの最終的な完成度と、提出前の最終チェックポイント。
>
> 【インシデントレポート・ドラフト】
> （ステップ01、02で作成した事実経過、原因分析、対策案の全文をここに貼り付ける）

### 3. AIの指摘に基づく最終修正

AIの指摘は、レポートの質を一段階引き上げるための貴重なフィードバックです。特に、責任追及の排除に関する指摘は、組織の安全文化を維持するために最も重要です。

*   修正例: 「看護師Xが離室したミス」→「看護師Xが他の患者の緊急対応のため離室せざるを得なかった状況」
*   修正例: 「患者Yの認知症による問題行動」→「患者Yの認知機能の状態と、それに対応する環境整備の必要性」

#### Tips：AIに「ポジティブな表現」への変換を依頼する

インシデントレポートは、ネガティブな事象を扱う文書ですが、対策の部分は未来志向であるべきです。AIに「指摘されたネガティブな表現を、医療安全の向上につながるポジティブな表現に変換してください」と依頼することで、レポート全体のトーンを改善できます。

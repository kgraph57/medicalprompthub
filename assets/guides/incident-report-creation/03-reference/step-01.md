# 03-reference/step-01.md: 応用編：AIによるインシデント情報の構造化と傾向分析

## 個別報告から組織的改善へ：AIを活用した集計・分析の自動化

インシデントレポートは、個別の事象を記録するだけでなく、集計・分析することで組織全体の医療安全上の傾向を把握し、予防策を講じるための重要なデータソースとなります。応用編では、AIを活用して、報告書から必要な情報を抽出し、傾向分析を自動化する方法を学びます。

### 1. 非構造化データからの情報抽出

インシデントレポートの多くは、自由記述形式の非構造化データです。AIの自然言語処理（NLP）能力を活用することで、この非構造化データから、分析に必要な構造化された情報を効率的に抽出できます。

| 抽出したい情報 | 抽出の目的 | AIへの指示例 |
| :--- | :--- | :--- |
| 事象の分類 | 転倒・転落、薬剤、処置など、事象の種類を標準化。 | 「事象の種類を、厚生労働省の分類基準に従って分類せよ。」 |
| 発生時間帯 | 発生時刻を「早朝」「日中」「夜間」などに分類し、シフトとの関連を分析。 | 「発生時刻を4区分（0-6時、6-12時、12-18時、18-24時）に分類せよ。」 |
| 関与職種 | 医師、看護師、薬剤師など、関与した職種を特定。 | 「レポートから関与した医療従事者の職種を全て抽出せよ。」 |

### 2. プロンプト例：複数レポートからの構造化データ抽出

過去のインシデントレポートのテキストデータをAIに渡し、傾向分析のためのデータセットを生成させます。

プロンプト:
> あなたは医療安全管理部門のデータアナリストです。以下の【過去のインシデントレポート抜粋】から、傾向分析に必要な情報を抽出し、指定されたJSON形式で出力してください。
>
> 【過去のインシデントレポート抜粋】
> （過去数件のレポートの「事実経過」部分のテキストをここに貼り付ける）
>
> 出力形式（JSON）:
> ```json
> [
>   {
>     "report_id": "R20251201",
>     "event_type": "薬剤",
>     "severity_level": "レベル3a",
>     "time_slot": "日中（12-18時）",
>     "involved_staff": ["看護師", "薬剤師"],
>     "root_cause_category": "組織的要因"
>   },
>   // ... 他のレポートデータ
> ]
> ```

### 3. 抽出データの傾向分析と可視化（次世代の活用）

AIが抽出した構造化データ（JSONなど）は、表計算ソフトやデータ分析ツール（PythonのPandasなど）に取り込むことで、発生頻度の高い事象、特定の時間帯に集中するインシデント、特定の職種が関与する傾向などを可視化できます。

#### Tips：AIによる分析結果の解釈

データ分析の結果（例：夜間の転倒・転落が多い）をAIに渡し、「この結果から導かれる医療安全上の示唆と、優先的に取り組むべき対策を3つ提案してください」と依頼することで、分析結果を具体的な行動計画に変換するプロセスも自動化できます。

## Step 5: 口頭発表原稿（スクリプト）の作成と練習支援

スライドが完成しても、口頭での発表が伴わなければ、そのメッセージは十分に伝わりません。AIは、作成したスライドの内容に基づき、発表時間内に収まる、聴衆を惹きつけるスクリプトを作成し、練習を支援します。

### 1. スライド内容に基づいたスクリプトの生成

各スライドのタイトル、キーメッセージ、本文（箇条書き）をAIに渡し、それらを繋ぎ合わせる自然な口頭発表用のスクリプトを生成させます。

実用的なプロンプト例：スクリプトの生成

```
[役割設定]（Step 1で設定した役割を再利用）

以下のスライド内容に基づき、各スライドの発表時間が平均40秒となるように、自然な日本語の口頭発表用スクリプトを作成してください。
専門用語は正確に、しかし聴衆が聞き取りやすいトーンで記述してください。

対象スライド: スライド9（結果）
タイトル: SGLT2阻害薬の早期導入は再入院リスクを35%低減
キーメッセージ: 早期介入により、心不全の予後が有意に改善した。
本文（最適化後）:
- 主要評価項目：心不全再入院 or 心血管死
- HR 0.65 (95% CI 0.52-0.81)
- P < 0.001 (統計学的有意差)
- KM曲線：早期からイベント抑制効果

スクリプトの例:
「次に、本研究の主要な結果をご報告します。ご覧の通り、主要評価項目である心不全による再入院または心血管死の複合エンドポイントについて、SGLT2阻害薬の早期導入群は、標準治療群と比較して、その発生リスクをハザード比0.65と、有意に低減しました（P値は0.001未満です）。このKaplan-Meier曲線からも明らかなように、イベント抑制効果は治療開始後、早期から明確に確認されました。これは、SGLT2阻害薬の早期介入が、心不全の予後を劇的に改善する可能性を示唆しています。」
```

### 2. 発表練習のためのフィードバック

生成されたスクリプトを実際に読み上げ、その録音データをAIに分析させることで、発表の改善点に関するフィードバックを得ることができます。

実用的なプロンプト例：発表の改善点に関するフィードバック

```
（ユーザーが録音したスクリプトのテキストまたは音声ファイルをAIにアップロード）

私が読み上げた上記のスクリプトについて、以下の観点からフィードバックと改善点を提案してください。

1.  時間配分: 各スライドの発表時間が適切か。長すぎる、または短すぎるスライドはどれか。
2.  強調表現: キーメッセージが聴衆に伝わるよう、強調すべき単語やフレーズが適切に発声されているか。
3.  トーンとスピード: 専門家としての信頼感を与えるトーンと、聴衆が理解しやすいスピードで話されているか。

フィードバックの例:
- スライド9の「ハザード比0.65」の部分が早口になっており、重要な数値が聞き取りにくい可能性があります。「ハザード比、ゼロ・テン・ロク・ゴ」のように、数字を強調してゆっくり話すことを推奨します。
- 全体的に結論部分のトーンが弱いため、最後の「結論と提言」のスライドでは、自信を持って力強く話すよう意識してください。
```

### 3. 質疑応答（Q&A）のシミュレーション

発表後に想定される鋭い質問をAIに生成させ、それに対する回答のドラフトを作成することで、質疑応答への準備を万全にします。

実用的なプロンプト例：質疑応答のシミュレーション

```
[役割設定]（Step 1で設定した役割を再利用）

私の発表内容（SGLT2阻害薬の早期導入に関する研究）に基づき、聴衆の専門家（循環器内科医）から想定される、最も鋭い質問を3つ生成してください。
また、それぞれの質問に対する、簡潔でエビデンスに基づいた回答のドラフトを作成してください。

想定質問と回答の例:
1.  質問: 早期導入の定義が、実臨床での「早期」と乖離しているのではないか？
    回答: 本研究では、診断後48時間以内を早期導入と定義しましたが、これは...（エビデンスに基づく回答を記述）
2.  質問: 腎機能低下例における安全性データについて、詳細なサブグループ解析の結果はどうか？
    回答: eGFR 45未満の患者群においても、主要評価項目に対する効果は維持されており、安全性プロファイルも...（具体的なデータを引用）
```

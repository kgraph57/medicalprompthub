# Step 2: 症例の新規性確認

### 📋 このステップの目的

「この症例は報告する価値があるかもしれない」と感じたら、次に行うべきは徹底的な先行研究調査です。
あなたの症例が本当に「新しい」のか、あるいは既知の事実の重要な亜種なのかを客観的に証明するため、国内外の文献を網羅的に検索し、既報告例との比較を行います。
このステップが、論文の質と採択の可能性を大きく左右します。

### ⏱️ 所要時間

4〜8時間（検索範囲と文献数による）

### 🎯 達成基準

- 関連するキーワードでの網羅的な文献検索が完了している。
- 類似の症例報告を5〜10本以上読み込み、比較表が作成できている。
- 自分の症例の新規性（または臨床的意義）を、既報告例との比較に基づき3つのポイントで説明できる。

### 📝 詳細手順

![患者同意取得プロセス](/assets/guides/case-report/01-preparation/consent_process.png)

#### 手順1: 検索戦略を立てる

やみくもに検索を始めるのではなく、まずは戦略を立てます。
PECO/PICOフレームワークを参考に、検索の軸を明確にしましょう。

- P (Patient/Problem): どんな患者か？ (例: 器質化肺炎)
- E (Exposure) / I (Intervention): どんな治療や曝露があったか？ (例: ステロイド抵抗性)
- C (Comparison): 何と比較するか？ (例: ステロイド反応群)
- O (Outcome): どんな結果になったか？ (例: シクロホスファミドで寛解)

これらの要素を組み合わせ、英語の検索キーワード（MeSHタームを含む）を複数パターン洗い出します。

#### 手順2: AIツールで網羅的に文献検索する

以下のAIツールを組み合わせることで、効率的かつ網羅的な文献検索が可能になります。
このワークフローを図にまとめました。

![文献検索ワークフロー](https://storage.googleapis.com/agent-tools-public-external/workflow_diagram_1719655585.png)

1.  Perplexity (Deep Research): まず、PerplexityのPro版に搭載されているDeep Research機能を使い、広範な初期調査を行います。
「ステロイド抵抗性器質化肺炎の症例報告について、2010年以降の文献をレビューしてください」のように、自然言語で指示するだけで、主要な論文や総説をまとめたレポートを生成してくれます。

2.  Research Rabbit / Connected Papers: Perplexityで見つけた重要な論文（キーペーパー）を起点に、Research RabbitやConnected Papersを使って関連論文を芋づる式に探索します。
これにより、キーワード検索だけでは見逃してしまう可能性のある文献を発見できます。

3.  PubMed / Google Scholar: 最後に、AIツールで作成した検索戦略を使い、PubMedやGoogle Scholarで最終的な網羅的検索を行います。
特にPubMedでは、詳細なフィルター機能（症例報告、出版年など）を活用します。

#### 手順3: 新規性を言葉にする

収集した類似症例報告を読み込み、NotebookLMを使って比較分析を行います。
類似症例のPDFをすべてNotebookLMにアップロードし、プロンプトを使って比較表を作成させます。
この比較表をもとに、自分の症例が持つユニークな点を明確に言葉にします。

### 🤖 推奨AIツール

| ツール | 用途 | 優先度 |

### 💡 プロンプト集

#### プロンプト1: 網羅的文献検索のための検索戦略を立案する (ChatGPT)

```
# 指示
あなたはメディカルライブラリアンです。以下のPICOに基づいて、PubMedで網羅的な文献検索を行うための検索戦略を立案してください。MeSHタームとフリーキーワードを組み合わせ、複数の検索パターンを提案してください。

# PICO
- P (Patient): Organizing Pneumonia
- I (Intervention): Steroid-resistant / Refractory
- C (Comparison): Steroid-sensitive
- O (Outcome): Treatment with cyclophosphamide

# 出力形式
1.  検索コンセプト: (例: 「器質化肺炎」 AND 「ステロイド抵抗性」)
2.  MeSHターム: 
3.  フリーキーワード: 
4.  PubMed検索式 (複数パターン):
```

活用例: このプロンプトで生成された検索式をPubMedで実行することで、専門家レベルの網羅的な検索が可能になります。

#### プロンプト2: 収集した論文から新規性を評価・言語化する (NotebookLM)

```
# 指示
アップロードした10本の類似症例報告と、これからインプットする自験例の概要を比較し、自験例の新規性あるいは特筆すべき点を3つ、具体的な根拠と共に指摘してください。

# 比較項目
- 患者背景（年齢、性別、基礎疾患）
- 臨床症状
- 画像所見
- 治療内容と反応
- 転帰

# 自験例の概要
[ここに自分の症例の概要を貼り付ける]
```

活用例: NotebookLMに類似症例のPDFをすべて読み込ませた後、このプロンプトを実行します。
これにより、膨大な文献情報を基に、客観的かつ説得力のある「新規性の主張」を構築できます。

### 📚 実践例

症例: 40代男性、ステロイド抵抗性器質化肺炎（OP）

1.  検索戦略: ChatGPTを使い、「("Organizing Pneumonia"[Mesh]) AND (Steroid-Resistant OR Refractory)」などの検索式を作成。
2.  文献検索: Perplexityで初期調査後、Research Rabbitで関連論文を収集。
最終的にPubMedで15件の類似症例報告を特定。
3.  比較分析: 15件のPDFをNotebookLMにアップロード。
プロンプト2を使い、自験例と比較分析。
4.  新規性の言語化:
    1.  早期の抵抗性: 既報告例の多くは数ヶ月かけて抵抗性と判断されているが、自験例は2週間という早期に増悪を認めた点が新しい。
    2.  治療選択: シクロホスファミドを使用した報告は過去に3例のみで、いずれも高用量ステロイドとの併用だが、自験例は比較的低用量ステロイドとの併用で効果を示した点が異なる。
    3.  バイオマーカー: 既報告にない血清KL-6の特異な変動パターンを認めており、治療反応性の予測因子となる可能性を示唆している。

### ⚠️ よくある失敗と対処法

- 失敗: 日本語の文献（医中誌など）しか検索しない。
  - 対処法: 国際誌への投稿を目指す以上、PubMedを中心とした英語文献の検索は必須です。
DeepL翻訳などを活用し、英語の壁を乗り越えましょう。

- 失敗: 検索して「類似報告なし」と安易に結論づけてしまう。
  - 対処法: 「報告がない」ことの証明（negative proof）は非常に困難です。
検索範囲を広げ、キーワードを変え、AIツールを駆使しても見つからない場合に、初めて「極めて稀」と言えます。
指導医にも確認してもらいましょう。

### 💪 上級者向けTIPS

文献検索の結果は、そのまま論文のIntroductionやDiscussionの重要な資源となります。
検索プロセス（使用したデータベース、検索式、検索日、ヒット件数）を記録しておくと、後の執筆が非常にスムーズになります。
ZoteroやPaperpileなどの文献管理ツールを使い、検索結果を整理・保存する習慣をつけましょう。

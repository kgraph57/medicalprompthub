# 01-basics/step-02.md: データ入力の原則と基本プロンプトの構築

## 1. AIへのデータ入力の原則：正確性と構造化

AIによるポリファーマシー対策の精度は、入力するデータの正確性と構造化に大きく依存します。特に、薬剤名や用量、検査値は誤りがないよう注意が必要です。

### データ入力のチェックリスト

| 項目 | 入力時の注意点 | 構造化のヒント |
| :--- | :--- | :--- |
| 薬剤名 | 一般名または商品名と、必ず剤形（錠、カプセル、散など）を併記する。 | 箇条書き（リスト形式）で入力する。 |
| 用量・用法 | 1回量、1日量、投与回数、投与期間を明確にする。 | 「薬剤名: 1回量/用法 (処方理由)」のように統一する。 |
| 患者情報 | 年齢、性別、身長、体重、特に腎機能（eGFRまたはCr）は必須。 | 冒頭にまとめて記載し、AIに患者像を把握させる。 |
| 処方理由 | 漫然投与を防ぐため、全ての薬剤について処方目的を明記する。 | 処方内容とセットで入力する。 |

## 2. 基本プロンプトの構築：テンプレートの活用

効率的かつ質の高い回答を得るために、以下の要素を組み合わせた基本プロンプトテンプレートを使用します。

### 基本プロンプトテンプレート

```
# 役割設定
あなたは、高齢者薬物療法に精通した認定薬剤師です。
以下の患者情報と処方内容に基づき、ポリファーマシー対策の観点から評価を行ってください。

# 患者情報
- 年齢: [〇〇歳]
- 性別: [男性/女性]
- 体重: [〇〇kg]
- 腎機能: eGFR [〇〇 mL/min/1.73m²]
- 既往歴: [高血圧、糖尿病、心房細動、転倒歴あり]

# 現在の処方内容（全〇剤）
- 薬剤A: 〇〇mg 1日1回 朝食後 (高血圧)
- 薬剤B: 〇〇mg 1日2回 朝夕食後 (糖尿病)
- 薬剤C: 〇〇mg 1日1回 就寝前 (不眠)
- 薬剤D: 〇〇mg 1日3回 毎食後 (疼痛)
- ...

# 評価依頼
1.  潜在的に不適切な処方（PIM）の候補を特定し、その根拠（例：STOPP-J基準）を提示してください。
2.  薬剤関連問題（DRP）のリスク（例：転倒、認知機能低下、相互作用）を指摘してください。
3.  減薬または代替薬の提案を、具体的な薬剤名と用量を含めて行ってください。
4.  提案のエビデンスレベル（例：ガイドライン推奨、専門家の意見）を簡潔に示してください。
```

### 💡 Tips: 評価依頼の明確化

AIは、質問が具体的であるほど、より質の高い回答を生成します。特に「減薬の提案」を依頼する際は、「患者のQOL向上」や「特定の有害事象の回避」といったアウトカムを明確に指示することが重要です。

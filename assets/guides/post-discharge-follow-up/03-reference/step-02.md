# 03-reference/step-02.md: 継続的な改善のためのフィードバックループ構築とAIの役割

## 1. AIによる計画の有効性評価とフィードバックループ

退院後フォローアップ計画は、一度作成して終わりではありません。計画の実行結果（再入院の有無、外来受診率、患者満足度など）を分析し、次の計画作成に活かすフィードバックループを構築することが、医療の質（Quality Improvement, QI）向上に不可欠です。

AIは、このフィードバックループにおいて、大量のデータ分析と改善点の提案という重要な役割を果たします。

### 実用的なプロンプト例：計画の有効性評価と改善提案

```
あなたは、医療の質改善（QI）コンサルタントです。
以下の[過去100件の退院後フォローアップ計画のデータ]と[再入院結果]を分析し、以下の質問に回答してください。

分析対象データ:
*   計画書に記載された介入戦略（例：訪問看護の頻度、初回外来までの期間）
*   患者の属性（年齢、主病名、併存疾患数）
*   結果（30日以内の再入院の有無）

質問:
1.  再入院した患者群と再入院しなかった患者群で、計画書の内容に統計的に有意な差が見られた項目は何ですか？（例：初回外来までの期間が短いほど再入院率が低いなど）
2.  現在の計画作成プロセスにおいて、最も改善の余地があると思われる点はどこですか？
3.  次回の計画作成から即座に導入すべき、具体的な改善策を3点提案してください。

[過去100件の退院後フォローアップ計画のデータ]
（例: CSV形式のデータ、または構造化されたテキストデータを貼り付けます）
```

## 2. 医療現場でのAI活用を推進するためのTips

AIを単なるツールとしてではなく、チームの一員として最大限に活用するためのTipsです。

| Tips | 内容 | 期待される効果 |
| :--- | :--- | :--- |
| プロンプトのライブラリ化 | 頻繁に使用する高品質なプロンプト（例：情報抽出、計画書ドラフト生成）をチーム内で共有し、ライブラリとして管理する。 | チーム全体のAI活用レベルの均質化と、作業時間の短縮。 |
| AIの「失敗例」の共有 | AIがハルシネーションを起こしたり、不適切な提案をした事例を共有し、「AIの限界」をチーム全体で理解する。 | AIへの過信を防ぎ、最終的な臨床判断の重要性を再認識させる。 |
| AI担当者の任命 | チーム内でAIの最新情報や活用法を学び、他のメンバーに共有する「AIチャンピオン」を任命する。 | 継続的な学習と、新しい技術の現場へのスムーズな導入。 |

## 3. まとめ：AIと医療従事者の協働

退院後フォローアップ計画におけるAIの役割は、「情報の整理と、個別化されたコミュニケーションのドラフト作成」です。AIは、医療従事者が本来集中すべき「患者との対話」「臨床的な判断」「感情的なサポート」のための時間を創出します。

AIの力を借りて、より質の高い、患者中心の退院後フォローアップを実現しましょう。

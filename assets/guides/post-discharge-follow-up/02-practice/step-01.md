# 02-practice/step-01.md: 患者・家族向け「実行可能ロードマップ」のドラフト作成

## 1. AIによる計画書ドラフトの自動生成

基本編で抽出した構造化データ（表形式）と、患者の生活背景情報をAIに入力し、患者・家族が理解しやすい言葉で書かれた実行可能な計画書のドラフトを生成させます。

計画書には、以下の要素を含めることが重要です。

1.  退院後の目標: 短期（1週間）、中期（1ヶ月）の具体的な目標。
2.  日々のタスク: 服薬、食事、運動、バイタルチェックなどの具体的な行動。
3.  危険なサインと対応: いつ、誰に、どのように連絡すべきかの明確な基準。
4.  利用サービス: 訪問看護、デイサービスなどの開始日と担当者名（仮）。

### 実用的なプロンプト例：計画書ドラフトの生成

```
あなたは、患者とその家族の不安を和らげ、行動を促すことに長けたコミュニケーションの専門家です。
以下の[抽出情報]と[患者背景]に基づき、中学2年生でも理解できる平易な言葉で、退院後の生活を円滑にするための「退院後フォローアップ計画書」のドラフトを作成してください。

トーン: 励ましと共感に満ちた、親しみやすいトーン。専門用語は避けるか、分かりやすく説明すること。

構成:
1.  はじめに: 退院おめでとうの言葉と、この計画書の目的。
2.  退院後の3つの目標: 短期目標（例: 毎日決まった時間に薬を飲む）
3.  毎日のチェックリスト: 具体的な行動（例: 朝食前に体重を測る）
4.  「これは大変！」なサイン: 緊急時の連絡基準を分かりやすく。
5.  サポートチーム: 利用するサービスと連絡先（仮）。

[抽出情報]
（01-basics/step-02でAIが生成したMarkdownの表をそのまま貼り付けます）

[患者背景]
（例: 80代女性、一人暮らし。娘が週に2回訪問。趣味はガーデニング。心不全で再入院歴あり。）
```

## 2. ドラフトの検証と修正のポイント

AIが生成したドラフトは、以下の観点から必ず検証・修正してください。

| 検証ポイント | 内容 | 修正の方向性 |
| :--- | :--- | :--- |
| 実行可能性 | 患者のADLレベルや生活環境、介護者の負担を考慮し、現実的に実行可能か。 | 実行困難なタスクは、より小さなステップに分割するか、外部サービスへの依頼を検討する。 |
| 具体性 | 「安静にする」「注意する」といった抽象的な表現ではなく、「〇〇したら、〇〇する」という具体的な行動指示になっているか。 | 「散歩は1回15分まで」のように、数値や時間で制限を設ける。 |
| 心理的配慮 | 計画書が患者や家族に過度なプレッシャーを与えていないか。 | 「完璧でなくて大丈夫です」「困ったらすぐに連絡してください」といった安心感を与える一文を追加する。 |

### 注意点とTips

*   Tips: 計画書を患者の「My Plan」として、患者自身が書き込める余白（例：目標達成度、質問事項）を設けるようAIに指示すると、主体性が引き出されます。
*   注意点: 医療的な判断に関わる部分は、必ず担当医師や専門職の最終確認を得てください。

# ステップ2: 文献検索と背景調査
![文献レビュープロセス](/Helix/assets/guides/case-report/images/step02_literature_review_process.png)

## このステップの目的

症例の新規性と臨床的意義を客観的に証明するため、最新のAIツールを駆使して網羅的な文献検索を行い、関連研究の背景を深く理解します。
このステップは、論文の科学的価値を決定づける重要なプロセスです。

## 達成基準

- 症例の新規性に関連するキーワードでの系統的な文献検索が完了している。
- Elicitを用いて関連論文が50〜100件抽出され、そのうち主要な10〜20件が精読されている。
- NotebookLMを用いて、精読した論文の統合的な分析が完了している。
- 自分の症例のユニークな点を、既報告例との比較に基づき3つ以上、明確に言語化できる。

## 詳細な手順

### 2-1. 系統的文献検索（Elicit活用）

現代の文献検索は、もはやキーワードを一つずつ入力する時代ではありません。
AI研究アシスタント「Elicit」を活用し、系統的かつ効率的な文献レビューを行います。

手順:

1.  Elicitプロジェクト作成: Elicitにログインし、新しいプロジェクトを開始します。
2.  検索クエリの策定: ステップ1で特定した「症例の新規性」を基に、検索クエリを作成します。
PICOフレームワークを参考にすると、より精度の高い検索が可能です。

    -   P (Patient/Problem): 疾患名、患者背景
    -   I (Intervention): 治療法、診断法
    -   C (Comparison): 比較対象
    -   O (Outcome): 転帰

3.  文献の抽出とフィルタリング: Elicitは、入力されたクエリに基づき、1億3800万件以上の論文データベースから関連論文を瞬時に抽出します。
さらに、抽出された論文を「Abstractの要約」や「介入」「結果」といった項目で自動的に整理し、表形式で提示します。
これにより、数十本の論文の概要を数分で把握できます。

4.  重要論文の特定: Elicitの分析結果を基に、特に重要と思われる論文を10〜20件選び、PDFをダウンロードします。

検索クエリ例:

-   `"atypical presentation of [disease name]"`
-   `"[symptom] AND [disease] case report"`
-   `"rare complication of [treatment]"`

### 2-2. 文献の統合分析（NotebookLM活用）

ダウンロードした論文PDFを、一つずつ読む必要はありません。
「NotebookLM」にすべてアップロードし、AIに統合的な分析を行わせます。

手順:

1.  論文のアップロード: NotebookLMの症例報告プロジェクトに、ダウンロードした論文PDFをすべてアップロードします。
2.  統合分析プロンプトの実行: 以下のプロンプトを用いて、複数文献から横断的な知見を抽出します。

プロンプト例（NotebookLM）:

```
# 指示
あなたは臨床研究の専門家です。アップロードされた15本の論文をすべて精読・分析し、以下の問いに答えてください。

# 質問
1.  類似症例の概要: これまでに報告されている類似症例の数、患者背景、臨床的特徴、治療法、転帰をまとめた表を作成してください。
2.  本症例との比較: 私の症例（[ステップ1で作成したサマリーを簡潔に記述]）と、これらの既報告例との決定的な違いは何ですか？
新規性あるいは特筆すべき点を3つ挙げてください。
3.  診断・治療の標準: これらの文献から、現在の診断および治療のベストプラクティスは何だと考えられますか？
4.  未解決の臨床的疑問 (Unmet Clinical Needs): これらの報告を読んでもなお、解決されていない臨床的な課題や疑問点は何ですか？
5.  引用すべき論文: 私の症例報告のIntroductionおよびDiscussionで引用すべき最も重要な論文を10本リストアップし、その理由を簡潔に説明してください。

# 出力形式
各質問に対して、明確かつ構造化された回答を生成してください。
特に質問1は表形式でお願いします。
```

### 2-3. 包括的背景調査（Gemini Deep Research活用）

より広い視野で疾患の背景を理解するために、Googleの「Gemini Deep Research」機能（Gemini Advancedに搭載）を活用します。
この機能は、数百のウェブサイトを自動的に参照し、多角的な情報をまとめたレポートを生成します。

手順:

1.  Deep Researchの起動: Gemini Advancedで新規チャットを開始し、Deep Research機能を有効にします。
2.  調査プロンプトの実行: 以下のプロンプトで、疾患に関する包括的な調査を指示します。

プロンプト例（Gemini Deep Research）:

```
# 指示
[疾患名]について、以下の観点から包括的な調査を実施し、最新の知見をまとめた5ページ程度の詳細なレポートを作成してください。

# 調査項目
1.  最新の診断基準と国際的ガイドライン（2023年以降の更新を含む）
2.  治療法の最新動向（新規承認薬、開発中の治療法など）
3.  予後に関する最新のデータ（長期予後、予後予測因子など）
4.  現在進行中の主要な臨床試験（ClinicalTrials.govなどから）
5.  この分野における未解決の臨床的課題と将来の展望
```

このステップにより、あなたの症例報告が、最新の医学的文脈の中でどのような位置づけにあるのかを明確に把握できます。
これは、論文のDiscussionパートを執筆する上で極めて重要な情報となります。

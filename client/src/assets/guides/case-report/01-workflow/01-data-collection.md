# ステップ1: 症例情報の収集と整理
![第1章: 準備編の5ステップ](/Helix/assets/guides/case-report/images/preparation_steps.png)

## このステップの目的

症例報告の執筆に必要なすべての臨床情報を一元的に収集し、構造化された形式で整理します。
このステップは、後の執筆プロセスをスムーズに進めるための基盤となります。

## 達成基準

- 症例に関するすべての関連情報（臨床経過、検査結果、画像所見など）が収集されている。
- 収集した情報がNotebookLM上で整理され、CARE guidelinesの各セクションに対応する形で構造化されている。
- 症例のユニークさや新規性が明確になっている。

## 詳細な手順

### 1-1. 症例情報の収集

まず、電子カルテや関連資料から、症例に関するすべての情報を収集します。
この際、以下の点に注意してください。

- 網羅性: 主訴、現病歴、既往歴、家族歴、身体所見、検査データ、画像所見、治療経過、転帰など、可能な限り詳細な情報を集めます。
- 時系列: すべての情報を時系列に沿って整理します。
- 匿名化: 患者のプライバシーを保護するため、氏名、生年月日、住所などの個人情報は必ず削除または匿名化します。

### 1-2. NotebookLMでの情報整理

収集した情報を、GoogleのAI搭載ノートツール「NotebookLM」を使って整理します。
NotebookLMは、アップロードした資料に基づいてAIが文脈を理解し、情報の整理や要約を強力にサポートします。

手順:

1.  新規ノートブック作成: NotebookLMで新しいノートブックを作成し、「症例報告: [疾患名]」のような名前を付けます。
2.  資料のアップロード: 収集した症例情報をテキストファイルにまとめ、NotebookLMにアップロードします。
PDF形式の検査レポートや画像所見レポートなども直接アップロードできます。
3.  構造化プロンプトの実行: 以下のプロンプトを使って、アップロードした情報をCARE guidelinesに沿って構造化します。

プロンプト例（NotebookLM）:

> 📌 プロンプト集へのリンク: [「AAC R] 症例情報の構造化」プロンプト](/prompts?id=aacr-data-structuring)

```
# 指示
アップロードされた以下の症例情報に基づき、CARE guidelinesの主要セクションに対応する形で情報を整理・抽出し、構造化されたサマリーを作成してください。

# 症例情報
[ここに収集した症例情報を貼り付け]

# 出力形式
## 1. Title（仮）
- 診断名 + "case report"

## 2. Patient Information
- 年齢、性別
- 主訴
- 既往歴、家族歴、社会的背景

## 3. Clinical Findings
- 主要な身体所見
- 主要な検査所見（血液、生化学、画像など）

## 4. Timeline
- 発症から現在までの主要な出来事を時系列でリストアップ

## 5. Diagnostic Assessment
- 診断に至るまでのプロセス
- 実施された主要な検査とその結果
- 鑑別診断
- 最終診断

## 6. Therapeutic Intervention
- 実施された治療（薬剤名、用量、期間）
- 治療の変更とその理由

## 7. Follow-up and Outcomes
- 治療後の経過
- 患者の転帰（改善、不変、悪化など）
- 有害事象の有無
```

### 1-3. 症例の新規性の特定

次に、整理した情報をもとに、この症例が医学文献においてどのような新規性・価値を持つのかを明確にします。
この作業には、高度な推論能力を持つClaude 3.5 Sonnetを使用します。

手順:

1.  Claude Projectを開く: 準備フェーズで作成した「Case Report Writing Assistant」プロジェクトを開きます。
2.  情報のインプット: NotebookLMで生成した構造化サマリーをClaudeにインプットします。
3.  新規性分析プロンプトの実行: 以下のプロンプトで、症例のユニークさを多角的に分析させます。

プロンプト例（Claude 3.5 Sonnet）:

> 📌 プロンプト集へのリンク: [「AACR] 症例の新規性分析」プロンプト](/prompts?id=aacr-novelty-analysis)

```
# 指示
あなたは経験豊富な臨床研究者です。以下の症例サマリーを読み、この症例が医学文献において持つ「新規性」と「臨床的意義」を分析してください。

# 症例サマリー
[NotebookLMで生成したサマリーを貼り付け]

# 分析の観点
1.  疾患の稀少性: この疾患自体が稀なものか？
2.  臨床像の非典型性: 症状、所見、経過が典型的なものとどう違うか？
3.  診断の新規性: 新しい診断モダリティや診断基準を適用したか？
4.  治療の新規性: 新規治療法の有効性、あるいは既存治療の予期せぬ効果・副作用を示したか？
5.  病態生理への示唆: 疾患の新たなメカニズムを示唆する所見はあるか？

# 出力
上記分析に基づき、この症例を学術雑誌に報告する価値がある理由を、最も重要なものから3つ、箇条書きで簡潔に述べてください。
```

このステップが完了すると、症例報告の骨子が出来上がります。
次のステップでは、ここで特定した「新規性」を軸に、文献検索戦略を立てていきます。

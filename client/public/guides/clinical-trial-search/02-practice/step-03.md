# 02-practice/step-03.md: バイアスリスク評価と患者向け説明文の作成

## 臨床試験の質を評価する：バイアスリスクの特定

臨床試験の検索と選定において、その試験結果の信頼性を評価することは極めて重要です。AIは、試験プロトコルや概要から、バイアスリスク（Risk of Bias）につながる要素を特定する作業を支援できます。

### 実用的なプロンプト例：バイアスリスクの特定

目的: 臨床試験の概要テキストを入力し、CochraneのRoB 2ツールなどの評価基準に基づき、バイアスリスクの主要な領域を指摘させる。

```
あなたはエビデンスベースの医療（EBM）の専門家です。以下の臨床試験の概要を読み、特にバイアスリスクが高いと考えられる領域を3つ特定し、その理由を簡潔に説明してください。

【臨床試験概要】
[臨床試験の概要テキスト]

【評価の視点】
1.  ランダム化プロセスからの逸脱
2.  介入の意図された遂行からの逸脱（盲検化の欠如など）
3.  アウトカムデータの欠測
4.  アウトカム測定のバイアス
5.  結果報告の選択的選択

【出力形式】
- リスク領域1: [領域名] / 理由: [具体的な記述と懸念点]
- リスク領域2: [領域名] / 理由: [具体的な記述と懸念点]
- リスク領域3: [領域名] / 理由: [具体的な記述と懸念点]
```

AIの期待される応答（例）:
- リスク領域1: 介入の意図された遂行からの逸脱 / 理由: 概要に「非盲検試験」と明記されている。これにより、患者や評価者が介入を知っていることによるパフォーマンスバイアスや検出バイアスが生じる可能性がある。
- リスク領域2: アウトカムデータの欠測 / 理由: 概要に「ドロップアウト率が15%を超えた」との記述がある。欠測データが結果に与える影響について、プロトコルでどのように対処されたかを確認する必要がある。
- リスク領域3: 結果報告の選択的選択 / 理由: 主要評価項目が複数設定されているが、中間解析の結果報告では特定の副次評価項目のみが強調されている。プロトコルとの整合性を確認する必要がある。

## 患者向けの説明文作成

臨床試験への参加を検討する患者に対し、専門的な内容を正確かつ平易に説明することは、インフォームド・コンセントの基本です。AIは、専門的な試験概要を、患者やその家族にも理解しやすい言葉に変換する作業を支援します。

### 実用的なプロンプト例：患者向け説明文の作成

目的: 専門的な試験概要を入力し、患者の教育レベルや不安に配慮した、平易な言葉による説明文を作成させる。

```
あなたは患者教育の専門家です。以下の臨床試験の概要を読み、患者（高校卒業程度の教育レベルを想定）とその家族向けに、以下の点を中心に平易な言葉で説明文を作成してください。

【臨床試験概要】
[臨床試験の概要テキスト]

【説明に含めるべき要素】
1.  この試験の目的（なぜこの研究が必要なのか）
2.  試験で受ける治療（新しい薬、標準的な治療、プラセボなど）
3.  参加することのメリットとリスク（副作用、通院回数など）
4.  参加しない場合の選択肢

【トーン】
共感的で、専門用語を避け、不安を和らげるようなトーンで記述してください。
```

AIの期待される応答（例）:
> 「この研究は、あなたの病気（〇〇）に対する新しい治療法（薬剤X）が、これまで使われてきた標準的な治療法（薬剤Y）よりも効果があるか、また安全かを調べるために行われます。…（中略）… 新しい薬を試せる可能性があるというメリットがある一方で、まだ知られていない副作用が出るリスクもあります。参加は完全に自由で、参加しない場合でも、現在の標準的な治療を継続できますのでご安心ください。」

---
注意点とTips:
*   最終確認: AIが作成した説明文は、必ず医療従事者自身が内容の正確性と倫理的な配慮を確認してから患者に提示してください。
*   リスクの強調: 患者向けの説明では、メリットだけでなく、リスクと不確実性についても明確かつ正直に伝えることが重要です。

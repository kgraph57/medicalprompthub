# ステップ2: 鑑別診断のマインドマップを作成する

## なぜマインドマップなのか？

鑑別診断のリストは、テキストだけでは全体像を把握しにくいことがあります。マインドマップを使って思考プロセスを可視化することで、以下のメリットがあります。

*   全体像の把握: 疾患同士の関連性や分類が一目でわかります。
*   思考の整理: 複雑な鑑別診断を構造的に整理できます。
*   記憶の定着: 文字情報よりも視覚的な情報の方が記憶に残りやすいです。
*   プレゼンテーション: カンファレンスなどで、自身の思考プロセスを他者に分かりやすく伝えられます。

---

## AIによるマインドマップ生成

AIに鑑別診断リストをマインドマップ形式で出力させることで、簡単に思考を可視化できます。

### マインドマップ生成プロンプト

```
あなたは経験豊富な臨床医です。以下の主訴と鑑別診断リストについて、思考プロセスが分かるようにマインドマップ形式で整理してください。

【主訴】
[ここに主訴を入力]

【鑑別診断リスト】
[ここに鑑別診断リストを貼り付け]

【出力形式】
*   主訴を中心に配置してください。
*   解剖学的な分類（心臓、肺、消化器など）や、病態生理学的な分類（血管性、感染性など）で枝分かれさせてください。
*   各診断名には、診断のキーワードとなる所見を3つ程度追記してください。
*   テキストベースの階層構造で表現してください。
```

---

## 実践例: 胸痛の鑑別診断マインドマップ

### 入力プロンプト

```
あなたは経験豊富な臨床医です。以下の主訴と鑑別診断リストについて、思考プロセスが分かるようにマインドマップ形式で整理してください。

【主訴】
胸痛

【鑑別診断リスト】
1. 急性冠症候群（ACS）
2. 大動脈解離
3. 肺塞栓症（PE）
4. 緊張性気胸
5. 胃食道逆流症（GERD）
6. 筋骨格系の痛み
7. 帯状疱疹

【出力形式】
*   主訴を中心に配置してください。
*   まず「Don't miss diagnosis」と「それ以外の疾患」で大きく分類してください。
*   各診断名には、診断のキーワードとなる所見を3つ程度追記してください。
*   テキストベースの階層構造で表現してください。
```

### AIの出力例 (テキストベース)

```
【胸痛の鑑別診断マインドマップ】

胸痛
└─── Don't miss diagnosis (生命を脅かす疾患)
    ├─── 心血管系 (Cardiovascular)
    │   ├─── 急性冠症候群 (ACS) ★ (締め付けられる, 放散痛, リスク因子)
    │   └─── 大動脈解離 (引き裂かれる, 背部痛, 血圧左右差)
    └─── 呼吸器系 (Respiratory)
        ├─── 肺塞栓症 (PE) (突然の呼吸困難, DVT所見, D-dimer上昇)
        └─── 緊張性気胸 (片側性の呼吸音低下, 気管偏位, ショック)

└─── それ以外の疾患
    ├─── 消化器系 (Gastrointestinal)
    │   └─── 胃食道逆流症 (GERD) (食後増悪, 胸焼け, PPI著効)
    └─── 胸壁・皮膚 (Chest wall / Skin)
        ├─── 筋骨格系の痛み (体動時痛, 圧痛, 除外診断)
        └─── 帯状疱疹 (片側性, 帯状分布, 皮疹)
```

## Mermaidを使ったマインドマップの描画

さらに進んで、Mermaidという記法を使うと、テキストから実際の図を生成できます。これは、ドキュメントやプレゼンテーションに埋め込む際に非常に便利です。

### Mermaid生成プロンプト

```
先ほどの胸痛の鑑別診断マインドマップを、Mermaidのmindmap構文で書き出してください。
```

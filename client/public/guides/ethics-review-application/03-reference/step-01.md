# 03-reference/step-01.md: AIによる申請書全体の論理チェックと審査員視点での最終レビュー

## AIを「模擬審査員」として活用する

申請書の各セクションが完成したら、提出前に申請書全体の論理的な一貫性と倫理的配慮の網羅性を最終チェックする必要があります。この段階でAIを「模擬審査員」として活用することで、審査員が指摘するであろう論理の飛躍や、倫理的リスクの記述不足を事前に発見し、修正することができます。

### 具体的な手順

1.  申請書全体の統合: 作成した各セクション（研究の意義、方法、IC、データ管理など）の記述を一つのテキストファイルにまとめます。
2.  AIへの役割設定: AIに「あなたは、非常に厳格で、論理的な一貫性を重視する倫理審査委員である」という役割を与えます。
3.  論理チェックの指示: AIに対し、申請書全体を読み込み、「研究の目的と方法が整合しているか」「リスクと利益のバランスが適切か」「IC文書とデータ管理計画が矛盾していないか」の3点に焦点を当ててレビューするよう指示します。

### 実用的なプロンプト例

申請書全体をレビューし、審査員が指摘しそうな論理的な欠陥を洗い出すためのプロンプトです。

```prompt
あなたは、[あなたの所属機関]の倫理審査委員会の委員です。
以下の[倫理審査申請書のドラフト]を読み、「差し戻し（修正指示）」につながる可能性が最も高い論理的な欠陥、または倫理的配慮の記述不足を3点指摘してください。

指摘の形式:
1.  指摘事項: [具体的な問題点]
2.  根拠: [申請書のどの部分に問題があるか]
3.  修正提案: [問題を解決するための具体的な記述の修正案]

特に以下の点に焦点を当ててください:
-   研究の目的（意義）と、研究方法（デザイン）の間の論理的な繋がり。
-   想定されるリスク（02-practice/step-01）と、それに対する対策（02-practice/step-03）のバランスと具体性。
-   申請書全体で使用されている専門用語の統一性。

[倫理審査申請書のドラフト]:
[ここに、作成した申請書の全セクションのテキストを貼り付け]
```

### 注意点とTips

*   指摘の優先順位付け: AIに指摘させる際、「差し戻しにつながる可能性が最も高い」という制約を加えることで、重要度の高い問題から優先的に修正することができます。
*   「審査員が納得する」表現への修正: AIの指摘に基づき修正を行った後、再度AIに「この修正案は、審査員が納得できる表現になっているか」をレビューさせることで、より洗練されたプロフェクトな記述に仕上げることができます。

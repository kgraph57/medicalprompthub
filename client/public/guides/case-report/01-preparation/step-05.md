# Step 5: 症例データの整理

### 📋 このステップの目的

本格的な執筆を始める前に、膨大な電子カルテの情報の中から、症例報告に必要なデータを抽出し、構造化することが目的です。
時系列に沿って臨床経過を整理し、検査データや画像所見を一覧化することで、後の「Case Presentation」セクションの執筆が飛躍的にスムーズになります。
この地道な作業が、抜け漏れのない正確な症例記述の土台となります。

![症例データ整理のプロセス](./data_organization_process.png)

### ⏱️ 所要時間

2〜4時間

### 🎯 達成基準

- 臨床経過が時系列で整理されたサマリー（経過表）が作成されている。
- 主要な検査データ（血液検査、画像所見など）が表形式でまとめられている。
- CAREガイドラインのチェックリスト項目が、カルテのどこに記載されているかマッピングできている。

### 📝 詳細手順

#### 手順1: 臨床経過の時系列サマリー（経過表）作成

経過表（Clinical Course Table）は、症例報告において臨床経過を一目で理解できるようにまとめた表です。
この表を作成することで、複雑な症例の経過を整理し、Case Presentationセクションの執筆が格段にスムーズになります。
また、論文掲載時にも、読者が理解しやすいTableとして活用できます。

**世界中の研究者・医師が実践している方法（実現可能な方法に限定）**

世界中のAI研究者や医師は、以下のような実践的な方法で経過表を作成しています。
ただし、日本では電子カルテからの自動抽出システム（HeaRT、ClinicalPathなど）は一般的に利用できないため、**自分で手入力した情報をAIツールで構造化する方法**を推奨します。

1. **CAREガイドライン準拠**: 国際的な症例報告標準であるCAREガイドラインに沿って、主訴、臨床所見、診断、介入、結果、経過を体系的に整理します [3]。世界中の医学雑誌で採用されている標準的な方法です。
2. **手入力データのAI構造化**: 電子カルテから必要な情報を手動で抽出し、匿名化したテキストをAIツール（ChatGPT、Claudeなど）に入力して、構造化された経過表を作成します。これは日本でも実現可能な実践的な方法です。
3. **SOAP形式の活用**: Subjective（主観的情報）、Objective（客観的情報）、Assessment（評価）、Plan（計画）の4要素で情報を構造化します。多くの医療機関で標準的に使用されている方法です。

電子カルテの情報を時系列に沿って整理します。
手作業でも可能ですが、AIを活用すると効率的です。
ただし、患者の個人情報をAIに直接入力することは絶対に避けてください。
匿名化した上で、以下のように整理します。

##### 1-1. カルテ情報の匿名化

AIツールを活用する前に、必ず患者の個人情報を匿名化します。

**削除または置換すべき情報:**
- 患者名（氏名）
- 患者ID（カルテ番号、診療番号など）
- 生年月日（年齢のみ残す）
- 住所、電話番号
- 医療機関名（必要に応じて「当院」「近医」などに置換）

**匿名化の実例:**
```
【変更前】
患者：田中太郎（45歳男性、カルテ番号123456）
生年月日：1979年5月15日
住所：東京都渋谷区...
2024年1月10日、東京大学病院にて初診。

【変更後】
患者：45歳男性
2024年1月10日、当院にて初診。
```

##### 1-2. 時系列データの抽出

電子カルテから、以下の重要な情報を時系列順に抽出します。

**抽出すべき情報:**
- 重要な日付: 発症日、受診日、入院日、検査実施日、治療開始日、退院日、外来フォローアップ日など
- 症状・所見: 主訴、現病歴、身体所見の変化
- 検査結果: 血液検査、画像検査、病理検査など主要な結果
- 治療内容: 薬剤名、用量、投与期間、手術内容など
- 転帰: 症状の改善・悪化、合併症の有無など

**抽出方法:**
1. 電子カルテの「臨床経過」セクションを確認
2. 「診療録」「看護記録」から重要な出来事を日付順に抽出
3. 「検査結果」セクションから主要な検査データを取得
4. 「処方」セクションから治療内容を確認
5. これらをテキストファイルにコピー＆ペースト（匿名化済み）

##### 1-3. AIによる構造化: 経過表の作成（手入力データの構造化）

**重要**: 日本では電子カルテからの自動抽出システム（HeaRT、ClinicalPathなど）は一般的に利用できないため、**自分で手入力した情報をAIツールで構造化する方法**を実践します。

手順：
1. 電子カルテから必要な情報を**手動で抽出**し、テキストファイルにコピー＆ペーストします。
2. 個人情報を匿名化します。
3. 匿名化したテキストデータをAIツール（ChatGPT、Claudeなど）に入力し、構造化された経過表を作成します。

この方法は、世界中の研究者が実践している「手入力データのAI構造化」アプローチで、日本でも実現可能な実践的な方法です。

**経過表に含めるべき列:**
- **日付**: 実際の日付（例：2024/01/10）または発症からの日数（Day 1, Day 6など）
- **イベント**: 症状、診察所見、検査、治療など、その日に起こった重要な出来事
- **主要な検査値**: 重要な検査データ（CRP、WBC、画像所見など）があれば簡潔に記載
- **治療内容**: 薬剤名、用量、治療変更点など
- **備考**: 特記事項や補足説明

**経過表のフォーマット例:**

| 日付 | Day | イベント | 主要な検査値 | 治療内容 | 備考 |
|------|-----|----------|--------------|----------|------|
| 2024/01/10 | Day 1 | 乾性咳嗽が出現 | - | - | - |
| 2024/01/15 | Day 6 | 近医受診、胸部X線で両側肺に浸潤影を指摘 | 胸部X線：両側下肺野に浸潤影 | - | 当科紹介依頼 |
| 2024/01/17 | Day 8 | 当科初診、SpO2 95% | CRP 5.8 mg/dL, WBC 8,200/μL | - | 気管支鏡検査を予定 |
| 2024/01/20 | Day 11 | 入院、気管支鏡検査実施 | BALF：リンパ球優位（CD4/CD8=2.5） | - | 病理検査に提出 |
| 2024/01/22 | Day 13 | 病理結果でOPと確定診断 | 病理：器質化肺炎の特徴 | プレドニゾロン 0.5 mg/kg/日開始 | - |
| 2024/01/29 | Day 20 | 咳嗽が軽快、胸部X線改善傾向 | CRP 1.2 mg/dL, 胸部X線：浸潤影減少 | プレドニゾロン継続 | - |
| 2024/02/05 | Day 27 | 退院、外来通院継続 | - | プレドニゾロン漸減開始 | - |

##### 1-4. 経過表作成のポイント

1. **発症日をDay 1とする**: 症例報告では、発症日をDay 1として日数をカウントすることが一般的です。これにより、読者が経過を追いやすくなります。

2. **重要なターニングポイントを明確に**: 診断確定日、治療開始日、転帰が変わる日など、症例のストーリーを語る上で重要な日付は必ず含めます。

3. **検査値は簡潔に**: すべての検査値を記載するのではなく、症例のストーリーに関連する主要な検査値のみを記載します。詳細な検査データは別途「検査データ表」として作成します。

4. **自己完結型にする**: 経過表だけを見ても、症例の経過が理解できるようにします。論文に掲載する際も、Tableとして独立して理解できる必要があります。

5. **治療の変更を明確に**: 薬剤の開始、変更、中止、用量変更などは必ず記載します。

#### 手順2: 検査・画像データの表形式化

臨床経過で変動した重要な検査データ（炎症反応、腫瘍マーカー、疾患特異的抗体など）や、画像所見の推移を表にまとめます。
これにより、読者は治療による変化を一目で理解できます。

- 検査データ: ExcelやGoogle Sheetsを使い、日付ごとに行を追加し、各検査項目を列にして入力します。
グラフ化も念頭に置いて作成すると良いでしょう。
- 画像データ: 主要な画像（CT、MRI、病理写真など）を選び、撮影日、所見を簡潔に記述したリストを作成します。
後のFigure作成の準備になります。

#### 手順3: CAREガイドラインとのマッピング

症例報告の国際的な標準であるCAREガイドラインのチェックリストを用意し、各項目に必要な情報がカルテのどこにあるかを確認・マッピングします。
これにより、記述の抜け漏れを防ぎます。

### 💡 プロンプト集（実現可能な方法に基づく）

以下のプロンプトは、**手入力したデータをAIツールで構造化する方法**に基づいています。
日本では電子カルテからの自動抽出はできないため、自分で手入力した情報を構造化する実践的な方法を紹介します。

#### プロンプト1: 手入力した匿名化カルテ情報から臨床経過のサマリーを作成する

**重要**: このプロンプトは、自分で手入力した情報をAIツールで構造化する方法です。
電子カルテから必要な情報を手動で抽出し、匿名化したテキストをAIツールに入力します。

```
# 指示
あなたは、臨床記録から時系列情報を抽出する専門家です。
以下の手入力された匿名化済み診療録のテキストから、時間経過に沿った臨床経過サマリーをマークダウン形式の表で作成してください。
臨床イベント（症状、診察所見、検査、治療など）を識別し、時系列で構造化してください。

# 出力形式
マークダウン形式の表を作成してください。列は以下の通りです：
- 日付（実際の日付）
- Day（発症日をDay 1とする）
- イベント（症状、診察所見、検査、治療など）
- 主要な検査値（重要な検査データがあれば簡潔に記載）
- 治療内容（薬剤名、用量、治療変更点など）
- 備考（特記事項や補足説明）

# 抽出項目の注意点
- 発症日をDay 1として、その後の日付はDay 2, Day 3...と数えてください
- イベントは簡潔に、しかし重要な情報は漏らさず記載してください
- 検査値は数値と単位を正確に記載してください
- 治療内容は薬剤名、用量、投与開始・変更・中止などを明確に記載してください

# 匿名化済み診療録テキスト
[ここに匿名化したカルテ情報を貼り付ける]
例：
2024/01/10: 乾性咳嗽が出現。
2024/01/15: 近医受診。胸部X線で両側肺に浸潤影を指摘され、当科紹介。
2024/01/17: 当科初診。SpO2 95%。血液検査でCRP 5.8 mg/dL, WBC 8,200/μL。気管支鏡検査を予定。
2024/01/20: 入院、気管支鏡検査実施。BALFでリンパ球優位（CD4/CD8=2.5）。
2024/01/22: 病理結果で器質化肺炎と確定診断。プレドニゾロン 0.5 mg/kg/日（30 mg/日）開始。
...
```

活用例: カルテのコピー＆ペースト（匿名化後）から、構造化された経過表を瞬時に作成できます。
これにより、複雑な経過を客観的に整理し、執筆の準備を整えることができます。

#### プロンプト2: 作成した経過表の質を評価・改善する（CAREガイドライン準拠）

このプロンプトは、国際的な症例報告標準であるCAREガイドライン [3] に基づいて、経過表の質を評価します。

```
# 指示
あなたは、CARE（Case Report）ガイドラインに精通した医学雑誌の査読者です。
以下の臨床経過表をレビューし、CAREガイドラインと国際的な症例報告の標準に基づいて評価してください。必要に応じて改善案を提示してください。

# 評価観点（CAREガイドライン準拠）
1. **完全性**: 重要なイベント（発症、診断確定、治療開始・変更、転帰）がすべて含まれているか？CAREガイドラインの「Timeline」要件を満たしているか？
2. **明確性**: 各イベントが簡潔かつ明確に記述されているか？医療専門家以外でも理解できるか？
3. **時系列**: 日付とDayが正確にカウントされているか？時系列の一貫性は保たれているか？
4. **自己完結性**: この表だけを見て、症例の経過が理解できるか？論文掲載時に独立して読めるか？
5. **論文掲載適性**: 学術論文のTableとして適切なフォーマットになっているか？主要医学雑誌（JAMA、BMJ、NEJMなど）の標準に準拠しているか？

# 経過表
[ここに作成した経過表を貼り付ける]

# 出力形式
- 評価結果（各観点について）
- 改善が必要な点（あれば）
- 改善後の経過表（改善が必要な場合）
```

活用例: 自分で作成した経過表に漏れや不明確な点がないか、AIにレビューしてもらいます。
論文掲載レベルの質に引き上げるための重要なステップです。

#### プロンプト3: 経過表からCase Presentationセクションの骨子を作成する

このプロンプトは、手入力して作成した経過表から、読みやすい文章形式の症例報告を作成します。

```
# 指示
あなたは、診療経過データを文章形式に変換する専門家です。
以下の臨床経過表を基に、症例報告の「Case Presentation」セクションの骨子を作成してください。
経過表の情報を、時系列に沿って自然な文章に展開してください。
CAREガイドラインに準拠し、主要医学雑誌（JAMA、BMJ、NEJMなど）の標準的なCase Presentation形式に従ってください。

# 出力形式
1. 患者背景（年齢、性別、主訴）
2. 現病歴（発症から初診までの経過）
3. 初診時の所見（身体所見、検査結果）
4. 入院後の経過（診断過程、治療経過）
5. 転帰（退院時、外来フォローアップ時）

各セクションを箇条書きで簡潔にまとめてください。

# 経過表
[ここに作成した経過表を貼り付ける]
```

活用例: 経過表から直接Case Presentationの骨子を作成することで、執筆効率が大幅に向上します。
経過表が詳細であればあるほど、質の高い骨子が生成されます。

### ⚠️ よくある失敗と対処法

- **失敗1: 患者の個人情報（氏名、IDなど）をそのままAIツールに入力してしまう。**
  - 対処法: 絶対に避けてください。
これは重大なプライバシー侵害であり、法的な問題に発展する可能性があります。
AIを利用する際は、必ず個人情報を削除・仮名に置換するなどの匿名化処理を徹底してください。
匿名化チェックリストを作成し、AIに入力する前に必ず確認する習慣をつけましょう。

- **失敗2: データを整理せず、カルテを見ながら直接書き始めてしまう。**
  - 対処法: 一見遠回りに見えますが、このステップを丁寧に行うことで、後の執筆効率が格段に上がります。
特に、共著者との情報共有や、査読者からの修正依頼に対応する際に、整理されたデータが非常に役立ちます。

- **失敗3: 経過表に主観的な解釈を記載してしまう。**
  - 対処法: 経過表は「客観的事実」のみを記載するべきです。
「ステロイド抵抗性」「診断困難」などの解釈や評価は、経過表ではなくDiscussionセクションで述べます。
経過表の各セルには、「何が起こったか」という事実のみを記載しましょう。

- **失敗4: 重要なイベントを漏らしてしまう。**
  - 対処法: カルテからデータを抽出する際は、以下のチェックリストを活用しましょう。
  - [ ] 発症日は記載されているか？
  - [ ] 初診日・入院日は記載されているか？
  - [ ] 診断確定日は記載されているか？
  - [ ] 治療開始日・変更日・中止日は記載されているか？
  - [ ] 合併症や副作用が発生した日は記載されているか？
  - [ ] 転帰（退院日、外来フォローアップ日など）は記載されているか？

- **失敗5: 経過表が長すぎて読みにくい。**
  - 対処法: 経過表は簡潔であるべきです。
すべての検査値を記載するのではなく、症例のストーリーに関連する主要な検査値のみを記載します。
詳細な検査データは別途「検査データ表」として作成しましょう。
一般的に、経過表は10-15行程度が目安です。
1ヶ月以上の長期間の経過がある場合は、重要なターニングポイントのみを抽出するか、複数の表に分けることを検討します。

- **失敗6: Dayのカウントが不正確。**
  - 対処法: 発症日をDay 1として、その後の日付を正確にカウントします。
AIが生成した経過表でも、Dayのカウントが間違っている場合があります。
必ず手動で確認し、必要に応じて修正しましょう。
Excelの日付計算機能を使えば、発症日から各日付までの日数を自動計算できます。

- **失敗7: 検査値の単位が記載されていない、または不統一。**
  - 対処法: すべての検査値には必ず単位を記載します。
CRPなら「mg/dL」、WBCなら「/μL」など、標準的な単位を使用します。
単位が統一されていないと、読者が混乱するだけでなく、論文の質も低下します。

### 💪 上級者向けTIPS（実現可能な方法に基づく）

以下のTIPSは、日本で実現可能な高度な手法を紹介しています。
**注意**: 電子カルテからの自動抽出システムは日本では一般的に利用できないため、手入力データを活用する方法に限定しています。

1. **経過表を論文のTableとして直接利用する（CAREガイドライン準拠）**
   - 作成した経過表は、そのまま論文のTable 1（または適切な番号）として掲載できます。
   - ただし、投稿規定に合わせて、フォーマットを調整する必要があります。
   - 一般的に、学術論文のTableは以下の形式が推奨されます：
     - 表のタイトルは「Table 1. Clinical course of the patient」のように簡潔に
     - 縦線は最小限に（横線のみを使用）
     - 各セルは簡潔に、しかし重要な情報は漏らさず
     - 脚注（footnote）で略語や単位の説明を追加

2. **時系列データの可視化（手入力データの可視化）**
   - 手入力して作成した経過表に記載した検査値の推移をグラフ化することで、読者の理解を助けるFigure（図）を作成できます。
   - ExcelやGoogle Sheetsのグラフ機能を使えば、簡単に折れ線グラフや棒グラフを作成できます。多くの医療機関で標準的に使用されています。
   - Pythonの`matplotlib`や`seaborn`ライブラリを使えば、論文掲載レベルの高品質なグラフをプログラムで作成することも可能です。上級者向けの方法です。
   - グラフには、治療開始日などの重要なイベントを縦線で示すと、時系列の関係がより明確になります。

3. **複数の経過表を使い分ける**
   - 長期間の経過がある場合や、複数の疾患が併存している場合は、経過表を複数に分けることが有効です。
   - 例：「Table 1. Initial clinical course（初期経過）」「Table 2. Treatment course and outcome（治療経過と転帰）」
   - または疾患ごとに分ける：「Table 1. Clinical course of primary disease」「Table 2. Clinical course of complications」

4. **経過表を基にしたタイムライン図の作成**
   - 手入力して作成した経過表の情報を、より視覚的なタイムライン図（Timeline diagram）として作成することもできます。
   - ガントチャート形式で情報を表示することで、医師が患者の診療経過を迅速に把握できます。
   - PowerPointやIllustrator、またはオンラインツール（例：Lucidchart）などで作成できます。

5. **経過表とCase Presentationの連携**
   - 経過表を作成したら、プロンプト3を使用してCase Presentationセクションの骨子を自動生成できます。
   - 経過表の各イベントを、自然な文章に展開することで、効率的に執筆できます。
   - ただし、生成された文章は必ず医療専門家が確認・修正する必要があります。

6. **共著者との情報共有**
   - 経過表は、共著者との情報共有にも非常に有効です。
   - Google SheetsやExcel Onlineなどで共有することで、リアルタイムで共同編集できます。
   - これにより、複数の医師が症例の経過を迅速に理解し、正確な共著者レビューが可能になります。

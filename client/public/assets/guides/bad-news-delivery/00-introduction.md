# Bad News Delivery（悪い知らせの伝え方）ガイド

## はじめに

医師として避けて通れないのが、患者さんやご家族に「悪い知らせ」を伝える場面です。がんの告知、予後不良の説明、治療中止の提案など、重い内容を伝えなければならない時、どのように話せばよいのでしょうか。

> 「どう伝えればいいのか分からず、言葉に詰まってしまった...」

このガイドでは、**SPIKESプロトコル**に基づいた、共感的で適切なコミュニケーション方法を学びます。

---

## なぜBad News Deliveryが重要か？

### 伝え方が患者に与える影響

| 良い伝え方 | 悪い伝え方 |
| :--- | :--- |
| 心理的サポートを感じる | 見捨てられた感覚 |
| 情報を正確に理解 | 混乱・誤解 |
| 治療への前向きな姿勢 | 治療拒否・絶望 |
| 医師への信頼 | 不信感・怒り |

---

### 医師側の困難

多くの医師がBad News Deliveryに困難を感じています。

| 困難 | 割合 |
| :--- | :--- |
| 感情的に辛い | 約70% |
| 適切な言葉が見つからない | 約60% |
| 患者の反応が予測できない | 約50% |
| 正式なトレーニングを受けていない | 約80% |

## SPIKESプロトコルとは

<!-- 図解挿入ポイント -->
<!--

下から上に6段の階段形式:
1. S (Setting) - 環境設定アイコン「環境を整える」

**SPIKES**は、Bad News Deliveryのための標準的なプロトコルです。

| 要素 | 内容 | 目的 |
| :--- | :--- | :--- |
| **S**etting | 環境設定 | 適切な場所・時間の確保 |
| **P**erception | 認識の確認 | 患者の理解度を把握 |
| **I**nvitation | 招待 | 情報提供の許可を得る |
| **K**nowledge | 知識・情報 | 悪い知らせを伝える |
| **E**motions | 感情への対応 | 共感的な対応 |
| **S**trategy/Summary | 方針と要約 | 今後の計画を共有 |

## 注意事項

### このガイドの位置づけ

- 実際の場面では**マニュアル通りにはいかない**
- 患者の個別性を常に考慮
- 経験豊富な上級医への相談・同席を推奨

### AIの活用について

- シナリオ練習やロールプレイの準備に活用
- **実際の会話をAIに代替させることはできない**
- 患者の個人情報はAIに入力しない

# Step 5: 症例データの整理 - 執筆前の「宝探し」をしよう！

### 📋 このステップの目的

さあ、いよいよ本格的な執筆の準備ですよ！

電子カルテには、あなたの症例に関する**膨大な情報**が詰まっていますよね。その中から、症例報告に**本当に必要なデータ**だけを、まるで宝探しのように見つけ出して、わかりやすく整理するのがこのステップの目的です。

臨床経過を**時系列**でバッチリ整理して、検査データや画像所見を**一覧化**しておけば、後の「Case Presentation（症例提示）」セクションの執筆が**驚くほどスムーズ**になりますよ！

「地味な作業だな…」と思うかもしれませんが、この**丁寧な下準備**こそが、抜け漏れのない、**読者に伝わる正確な症例記述**の土台になるんです。一緒に頑張りましょう！

### ⏱️ 所要時間

**2〜4時間**

（ちょっと時間がかかるかもしれませんが、後で何倍も楽になりますからね！）

### 🎯 達成基準

このステップが終わったら、以下の3つが揃っているかチェックしてみましょう！

-   臨床経過が、いつ何があったか**時系列**で整理されたサマリー（経過表）ができている。
-   重要な検査データ（血液検査、画像所見など）が、**パッと見てわかる表形式**にまとまっている。
-   症例報告の国際的な基準である**CAREガイドライン**のチェックリスト項目が、カルテのどこに書いてあるか**マッピング**できている。

### 📝 詳細手順

#### 手順1: 臨床経過の時系列サマリー作成

電子カルテの情報を、**物語**のように時系列に沿って整理していきましょう。手作業でもできますが、ここはAIの力を借りると**超効率的**です！

**⚠️ ただし、絶対に忘れないでください！**
患者さんの**個人情報（氏名、ID、生年月日など）をAIに直接入力することは、**絶対に**避けてくださいね。これはプライバシー保護の**最重要ルール**です。

1.  **カルテ情報の匿名化**: 患者名、ID、生年月日などは、必ず削除するか、仮名（例：「鈴木一郎さん」）に置き換えてください。
2.  **時系列データの抽出**: 受診日、入院日、症状が出た日、検査日、治療開始日、退院日など、**重要なイベントの日付**と、その日に起こったこと（症状、診察所見、治療内容など）をテキストとしてコピーします。
3.  **AIによる構造化**: 匿名化したテキストをChatGPTやClaudeにインプットして、「時系列の経過表を作って！」とお願いしましょう。

#### 手順2: 検査・画像データの表形式化

臨床経過の中で**変動した**り、**診断の決め手**になったりした重要な検査データ（炎症反応、腫瘍マーカー、疾患特異的抗体など）や、画像所見の推移を**表**にまとめます。

この表があれば、読者（そして査読者！）は、**治療によって何がどう変わったのか**を**一目で理解**できますよ。

-   **検査データ**: ExcelやGoogle Sheetsを使って、日付を行に、各検査項目を列にして入力しましょう。**グラフ化**も視野に入れて作っておくと、後のFigure作成が楽になります。
-   **画像データ**: 重要な画像（CT、MRI、病理写真など）を選び、**撮影日**と**所見**を簡潔に記述したリストを作っておきましょう。これがFigure作成の準備になります。

#### 手順3: CAREガイドラインとのマッピング

症例報告の国際的な標準である**CAREガイドライン**のチェックリストをご存知ですか？これを用意して、各項目（例：患者背景、診断評価、治療介入など）に必要な情報が、**あなたのカルテのどこにあるか**を一つ一つ確認・マッピングしていくんです。

この作業で、**「あれ、この情報が抜けてる！」**という事態を未然に防げます。完璧な症例報告を目指しましょう！

### 🤖 推奨AIツール

| ツール | 用途 | 優先度 |
|---|---|---|
| **ChatGPT / Claude** | 匿名化したテキストデータからの臨床経過サマリー（時系列の表）作成 | ⭐⭐⭐⭐ |
| **Excel / Google Sheets** | 検査データの整理とグラフ化 | ⭐⭐⭐ |

### 💡 プロンプト集

#### プロンプト1: 匿名化されたカルテ情報から臨床経過のサマリーを作成する

```
# 指示
あなたは優秀な病棟クラークです。以下の匿名化された診療録のテキストから、時間経過に沿った臨床経過サマリーをマークダウン形式の表で作成してください。

# 抽出項目
- **日付 (Day)**: 発症日をDay 1とする
- **イベント**: 症状、診察所見、検査、治療など
- **メモ**: 特記事項

# 匿名化済み診療録テキスト
[ここに匿名化したカルテ情報を貼り付ける]
例：
2024/01/10 (Day 1): 乾性咳嗽が出現。
2024/01/15 (Day 6): 近医受診。胸部X線で両側肺に浸潤影を指摘され、当科紹介。
2024/01/17 (Day 8): 当科初診。SpO2 95%。血液検査でCRP 5.8。気管支鏡検査を予定。
...
```

**活用例**: カルテのコピー＆ペースト（匿名化後）をするだけで、複雑な経過が**構造化された経過表**に早変わり！これで、客観的に経過を整理し、執筆の準備は万端ですね。

### 📚 実践例

**症例**: ステロイド抵抗性OP（器質化肺炎）

1.  **データ抽出**: 電子カルテから、患者背景、現病歴、入院後経過、検査データ、画像レポートをテキストファイルにコピー。
2.  **匿名化**: 患者情報を仮名「鈴木一郎、45歳男性」などに**しっかり置換**。
3.  **経過サマリー作成**: 匿名化したテキストをChatGPTにインプットし、プロンプト1を実行。入院日から退院後までの**経過表が自動生成**されました！
4.  **データ整理**: 生成された経過表をもとに、ExcelでCRP、KL-6、WBCなどの推移をまとめた**検査データ表**を作成。主要な胸部CT画像を時系列に並べたリストも作成。
5.  **CAREマッピング**: CAREチェックリストの「Clinical findings」「Diagnostic assessment」などの項目が、作成したサマリーのどこに対応するかを**最終確認**。

### ⚠️ よくある失敗と対処法

-   **失敗**: **患者さんの個人情報（氏名、IDなど）をそのままAIツールに入力しちゃう。**
    -   **対処法**: **これは絶対にダメですよ！** 重大なプライバシー侵害で、法的な問題にもなりかねません。AIを使うときは、必ず個人情報を削除・仮名に置き換えるなどの**匿名化処理**を**徹底**してくださいね。
-   **失敗**: データを整理せず、カルテを見ながらいきなり書き始めてしまう。
    -   **対処法**: 「遠回りかな？」と思うかもしれませんが、この**整理のステップ**を丁寧に行うことで、後の執筆スピードが**格段にアップ**します。共著者との情報共有や、査読者からの修正依頼にも、整理されたデータがあれば**自信を持って**対応できますよ！

### 💪 上級者向けTIPS

せっかく作った検査データ表は、そのまま論文の**Table（表）**として使えます！また、時系列データをグラフ化すれば、読者の理解を助ける**Figure（図）**にもなりますよね。

Pythonの`matplotlib`や`seaborn`ライブラリを使えば、**論文掲載レベルの高品質なグラフ**をプログラムでサクッと作れちゃいます。ぜひチャレンジしてみてください！

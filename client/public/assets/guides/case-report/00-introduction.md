# はじめに：AIと一緒に、あなたの「気づき」を論文にしよう

## このガイドで得られること

「症例報告、書かなきゃな…でも、何から始めたらいいんだ？」

もし先生がそう思っているなら、このガイドはきっと役に立ちます。

このガイドは、「この通りにやれば、誰でも症例報告が書ける」をコンセプトに作りました。
論文のアイデア出しから投稿まで、全部で18のステップに分けて、やるべきことを具体的に解説しています。

しかも、ただの手順書ではありません。
各ステップで、ChatGPTやNotebookLMといった最新のAIツールをどう使えばもっと楽に、もっと質の高い論文が書けるのか。
そのための具体的な「指示書（プロンプト）」と実践例をたっぷり紹介します。

これまで何ヶ月もかかっていた論文執筆が、半分以下の時間で終わるかもしれません。
AIという優秀なアシスタントと一緒に、サクッと論文を仕上げてしまいましょう。

## そもそも「症例報告」って、なんだっけ？

簡単に言えば、「先生が経験した、たった一人の患者さんの貴重な記録」を、世界中の医療者と共有するための論文です。

- 「あれ、この病気、教科書とちょっと違うぞ？」
- 「この薬、予期せぬ効果があったな」
- 「こんな珍しいケースは初めてだ」

そんな臨床現場での「おや？」という小さな気づき。
それが、新しい治療法のヒントになったり、未来の医療を大きく変えるきっかけになったりするんです。
症例報告は、そのための大事な第一歩なんですね [1]。

| 項目 | 一言でいうと… |
|---|---|
| 目的 | 現場の「気づき」を世界にシェアして、医療の発展に貢献する |
| 対象 | 先生が受け持った、たった一人の患者さん |
| 特徴 | 一人の患者さんの経過を、ドラマのように詳しく書く |
| なぜ重要？ | あなたの経験が、未来の誰かを救うかもしれないから |

## なんで、忙しいのに症例報告を書くの？

「ただでさえ忙しいのに、なんで論文まで…」と思いますよね。
でも、症例報告を書くことには、明日からの臨床がもっと面白くなる、たくさんのメリットがあるんです。

- 臨床能力がアップする: 一人の患者さんを深く掘り下げることで、診断や治療への理解が驚くほど深まります。
まるで、専門医の思考を追体験するようなものです。
- 「科学の目」が養われる: 論文を読み、書くプロセスを通じて、情報を批判的に吟味する力（批判的吟味）や、物事を筋道立てて考える力（論理的思考）が自然と身につきます。
- キャリアの武器になる: 症例報告は、立派な業績です。
専門医の取得やキャリアアップにも繋がります。

そして何より、先生の経験が、未来の医療の1ページを作る。
これって、すごくワクワクしませんか？

## 全体像はこんな感じ（だいたい4〜7週間）

このガイドでは、論文執筆のプロセスを3つの章、18のステップに分けました。
AIを使えば、もっと早く終わるかもしれません。

| 章 | ステップ | やること | 所要時間（目安） |
|---|---|---|
| 第1章: 準備編 | 1-5 | どの症例にするか決め、関連論文を読み、データを整理する | 1-2週間 |
| 第2章: 執筆編 | 6-12 | 論文の骨格を考え、各パートを書き進める | 2-3週間 |
| 第3章: 仕上げ編 | 13-18 | 参考文献を整え、英語をチェックし、いざ投稿！ | 1-2週間 |

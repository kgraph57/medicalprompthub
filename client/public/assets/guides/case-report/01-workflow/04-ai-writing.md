# ステップ4: AIによる執筆とファクトチェック

## 目的

設計したマスタープロンプトを用いて、AIに論文の各セクションを高品質かつ効率的に執筆させます。同時に、AIの出力に対する厳格なファクトチェックと修正（チューニング）を行い、科学的正確性と論理的一貫性を確保します。

## 達成基準

- Claude 3.5 Sonnetを用いて、論文の主要セクション（Introduction, Case Presentation, Discussion）の初稿が完成している。
- AIが生成したすべての記述について、元の情報源（症例情報、引用文献）との照合（ファクトチェック）が完了している。
- AIの出力を基に、人間の専門家（あなた）による修正・加筆が加えられ、ドラフトが完成している。

## 詳細な手順

### 4-1. 段階的プロンプティングによる執筆

一度に論文全体を執筆させようとすると、AIの出力品質は低下します。セクションごとに、あるいはパラグラフごとに段階的に指示を与える「**段階的プロンプティング**」が極めて重要です。

**手順**:

1.  **Claude Projectを開く**: ステップ3で設計したマスタープロンプトがインプットされた「Case Report Writing Assistant」プロジェクトを開きます。
2.  **Introductionの執筆**: マスタープロンプトの「タスク」部分を「Introduction執筆」に設定し、実行します。
3.  **Case Presentationの執筆**: 次に、「タスク」を「Case Presentation執筆」に変更し、実行します。この際、個人情報が漏洩しないよう、特に注意深く監視します。
4.  **Discussionの執筆**: 同様に、「Discussion執筆」を実行します。ここが最もAIの能力が発揮される部分です。文献比較やメカニズム考察など、複雑な論理構築をAIに行わせます。

**プロンプト例（マスタープロンプトのタスク部分を修正）**:

```
# タスク
以上のすべてのコンテキストを完全に理解した上で、以下の指示に従ってください。

**指示2**: 次に、詳細アウトラインの【Case Presentation】セクションの記述に基づき、症例提示部分のドラフトを作成してください。ターゲットジャーナルの規定とCAREガイドラインを遵守し、患者のプライバシーに最大限配慮してください。時系列に沿って、客観的な事実のみを記述してください。
```

### 4-2. 厳格なファクトチェック

AIは時に、もっともらしい嘘（ハルシネーション）をつくことがあります。AIの出力を鵜呑みにすることは、科学者として絶対にあってはなりません。すべての記述について、厳格なファactチェックを行います。

**チェック項目**:

-   **症例情報**: AIが記述した患者の年齢、検査データ、治療内容などが、ステップ1で収集した元の情報と完全に一致しているか？
-   **引用の正確性**: 引用された文献の内容が、AIの記述と一致しているか？文脈に合った適切な引用か？
-   **数値の正確性**: 単位、小数点、統計値などに誤りはないか？

この検証作業には、高度な医学的知識と推論能力が求められます。ここでは、**OpenAI o1**を「ファクトチェッカー」として活用します。o1は、複雑な推論と正確性に特化しており、Claudeの出力を客観的に検証するのに最適です。

**プロンプト例（OpenAI o1）**:

> 📌 **プロンプト集へのリンク**: [「AACR] AIファクトチェック」プロンプト](/prompts?id=aacr-fact-check)


```
# 指示
あなたは査読経験豊富な医学研究者です。以下の論文ドラフト（Claude 3.5 Sonnetが生成）と、その根拠となったコンテキスト情報を比較し、事実と異なる点、論理的に矛盾する点、不適切な引用がないかを徹底的に検証してください。

# 論文ドラフト（一部）
[Claudeが生成したテキストを貼り付け]

# 根拠情報
## 症例サマリー
[ステップ1で作成したサマリー]

## 引用文献
[引用された文献の要約または本文]

# 出力
検証結果を以下の形式で報告してください。
- **問題点**: [具体的な問題箇所を指摘]
- **根拠**: [なぜそれが問題なのかを根拠情報に基づいて説明]
- **修正案**: [どのように修正すべきかを提案]

問題がなければ、「問題なし」と報告してください。
```

### 4-3. 人間による最終チューニング

AIによる執筆とファクトチェックが完了したら、最後にあなた（人間の専門家）が全体を推敲します。AIには表現できない、行間のニュアンス、臨床現場の肌感覚、そして何よりも**あなたの論文に対する情熱**を吹き込む重要なプロセスです。

-   全体の論理構成は自然か？
-   読者に最も伝えたいメッセージは明確に伝わるか？
-   よりインパクトのある表現はできないか？

この人間とAIの協働プロセスこそが、AACR Methodの品質を担保する鍵となります。

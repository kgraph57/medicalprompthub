# Step 7: 図表の作成

### 📋 このステップの目的

「百聞は一見に如かず」。
複雑な臨床経過やデータの羅列も、優れた図（Figure）や表（Table）にまとめることで、読者は瞬時に状況を理解できます。
このステップでは、論文の説得力を飛躍的に高める、分かりやすく美しい図表を作成する技術を学びます。
図表は単なる飾りではなく、論文の「第二の顔」です。

![図表の作成](https://files.manuscdn.com/user_upload_by_module/session_file/90346460/xBbPTeVVfNktxwsD.png)

### ⏱️ 所要時間

3〜5時間

### 🎯 達成基準

- 論文のストーリーを補強するために必要な図表がすべて作成されている。
- 各図表は、それ単体で内容が理解できる自己完結型（self-explanatory）になっている。
- すべての図表が、投稿規定で定められたフォーマット（解像度、ファイル形式、番号付け）に準拠している。

### 📝 詳細手順

![図表の作成](./figure_creation_process.png)

#### 手順1: 図表にする情報の選定

まず、論文全体のストーリーを考え、どの情報を視覚化すれば最も効果的かを戦略的に判断します。

- 表 (Table) に適した情報: 
  - 正確な数値を示す必要があるもの（例: 入院時検査データ）。
  - 複数の項目を比較するもの（例: 類似症例報告の比較）。
  - 時系列のサマリー（例: 臨床経過）。
- 図 (Figure) に適した情報:
  - 視覚的な変化を示すもの（例: CT画像、病理組織像の経時的変化）。
  - データの傾向やパターンを示すもの（例: 検査値の推移グラフ）。
  - 複雑なメカニズムや手順を説明するもの（例: 病態生理の模式図、治療アルゴリズム）。

#### 手順2: 表 (Table) の作成

Step 5で作成した検査データや、Step 4の文献レビューで作成した類似症例比較表を、論文掲載用に清書します。
ExcelやGoogle Sheetsで作成し、最終的にWordなどに貼り付けます。
デザインはシンプル・イズ・ベスト。
罫線は水平線を基本とし、垂直線は極力使わない方が洗練されて見えます。

#### 手順3: 図 (Figure) の作成

目的に応じてツールを使い分けます。

- 画像: CTやMRI、病理写真など。
患者情報が写り込まないようにトリミングし、重要な所見を矢印や記号で示します。
PowerPointやKeynote上で複数の画像を組み合わせ、1つのFigureを作成するのが一般的です。
- グラフ: 検査値の推移など。
Excelでも作成可能ですが、より高品質なものを目指すなら、Python（matplotlib/seaborn）の活用がおすすめです。
AIにコードを生成させることもできます。
- 模式図: 病態生理や治療アルゴリズムなど。
BioRenderのような専門ツールを使うと、プロ品質のイラストを簡単に作成できます。

#### 手順4: 図の説明文 (Legend) の作成

図表の作成で最も重要な作業の一つです。
Legendは、本文を読まなくても図表の内容が完全に理解できるように記述する必要があります。
以下の要素を含めましょう。

- タイトル: 図表が何を示しているかを簡潔に記述。
- 説明: 図中の略語、記号、矢印、統計手法などをすべて説明。
- 結論: この図表から導かれる最も重要なメッセージを1文で記述。

### 🤖 推奨AIツール

| ツール | 用途 | 優先度 |
|---|---|---|
| PowerPoint / Keynote | 画像の組み合わせ、矢印・テキストの追加 | ⭐⭐⭐⭐⭐ |
| Excel / Google Sheets | 表の作成、基本的なグラフの作成 | ⭐⭐⭐⭐ |
| ChatGPT | 図の説明文（Legend）の草案作成、Pythonコード生成 | ⭐⭐⭐⭐ |
| BioRender | 専門的な医学イラスト・模式図の作成 | ⭐⭐⭐ |
| Python (matplotlib/seaborn) | 高品質な学術グラフの作成 | ⭐⭐ |

### 💡 プロンプト集

#### プロンプト1: 図の説明文（Legend）の草案を作成する (ChatGPT)

```
# 指示
あなたは医学雑誌の編集者です。以下の図の内容を説明する、プロフェッショナルな図の説明文（Figure Legend）を作成してください。説明文は、図のタイトル、図中の記号や略語の説明、そして図から読み取れる主要な結論を含む、自己完結型（self-explanatory）にしてください。

# 図の内容
- 種類: グラフ
- X軸: 入院後の日数 (Day)
- Y軸: CRP (mg/dL) と 血清KL-6 (U/mL)
- プロット: プレドニゾロン(PSL)とシクロホスファミド(IVCY)の投与量も棒グラフで示されている。PSL投与開始後もCRPとKL-6は上昇を続けたが、IVCY投与開始後に劇的に低下した。

# 出力形式
Figure X. [タイトル]
[詳細な説明]
```

活用例: 質の高いLegendを自力で書くのは意外と難しいものです。
このプロンプトで生成された草案を叩き台にすることで、時間を大幅に節約できます。

### 📚 実践例

症例: ステロイド抵抗性OP

- Table 1: 患者背景、入院時現症、主要な検査所見をまとめた。
- Table 2: Step 4で作成した類似症例比較表を清書した。
- Figure 1: 入院時、ステロイド治療中、IVCY治療後の胸部CT画像を3枚並べ、浸潤影の変化を矢印で示した。
- Figure 2: 治療経過（PSL, IVCY投与量）と、CRPおよびKL-6の推移を1つのグラフにまとめた。
ChatGPTのプロンプトを使い、グラフ作成のためのPythonコードを生成し、matplotlibで描画した。

### ⚠️ よくある失敗と対処法

- 失敗: 解像度の低い画像を使用してしまい、投稿後に再提出を求められる。
  - 対処法: 投稿規定で要求される解像度（通常は300 dpi以上）を確認し、元データから高解像度の画像ファイル（TIFF, EPS, 高画質JPEGなど）を書き出しましょう。

- 失敗: 他の論文や教科書の図を無断で転載してしまう。
  - 対処法: 重大な著作権侵害です。
必ず出典を明記し、出版社から転載許諾（Permission）を得る必要があります。
基本的には、自分で描き直す（トレースする）のが安全です。

- 失敗: 情報を詰め込みすぎた、ごちゃごちゃした図表を作成してしまう。
  - 対処法: 1つの図表で伝えるメッセージは1つか2つに絞りましょう。
不要な情報を削ぎ落とし、余白を活かすことが、分かりやすさの鍵です。
平賀先生の論文[1]にあるデザインの4原則（整列・反復・近接・対比）は非常に参考になります。

### 💪 上級者向けTIPS

色覚多様性（いわゆる色弱）を持つ読者にも配慮し、赤と緑の組み合わせを避ける、色だけでなく線種やマーカー形状でも区別できるようにするなど、ユニバーサルデザインを意識したグラフ作成を心がけましょう。

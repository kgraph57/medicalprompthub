# ステップ2: 患者情報の構造化

## 効率的な情報収集

引き継ぎサマリーを作成する際、**必要な情報を漏れなく、効率的に**収集することが重要です。

---

## 情報収集のチェックリスト

### 基本情報

- [ ] 患者ID、氏名、年齢、性別
- [ ] 主病名・入院日
- [ ] 担当医・主治医

### 現在の状態

- [ ] 現在のバイタルサインの傾向
- [ ] 本日の重要な変化
- [ ] 現在の治療内容（抗菌薬、点滴など）

### 今後の予定

- [ ] 本日・翌日の検査予定
- [ ] 予定されている処置・手術
- [ ] 退院・転科の予定

### 注意事項

- [ ] アレルギー歴
- [ ] DNARなどのAdvance Directive
- [ ] 特別な注意事項（転倒リスク、感染対策など）

### 引き継ぎの優先順位

| 優先度 | 重症度 | 変化の可能性 | 対応 |
| :--- | :--- | :--- | :--- |
| 🔴 最優先 | 高 | 高 | 詳細な引き継ぎ、頻回確認 |
| 🟠 高優先 | 高 | 低 | 詳細な引き継ぎ |
| 🟡 注意 | 低 | 高 | 変化時の対応を明確に |
| 🟢 定時 | 低 | 低 | 簡潔な引き継ぎ |

## 情報の構造化テンプレート

以下のテンプレートを使って、情報を整理しましょう。

```
━━━━━━━━━━━━━━━━━━━━━━━━
患者: [ID] [氏名] [年齢]歳 [性別]
入院日: [日付] 病名: [主病名]
担当: [主治医名]
━━━━━━━━━━━━━━━━━━━━━━━━

【現在の状態】
バイタル傾向: [安定/改善傾向/悪化傾向]
本日の変化: [特記事項]
現治療: [主要な治療内容]

【To-Do】
□ [タスク1] - [タイミング]
□ [タスク2] - [条件]

【If-Then】
もし [状況] → [対応]

【注意】
⚠️ [特別な注意事項]
━━━━━━━━━━━━━━━━━━━━━━━━
```

## カルテレビューの効率化

### 時間短縮のコツ

1. **サマリーから読む**
   - 入院時サマリー → 直近のプログレスノート

2. **変化点に注目**
   - 最新のバイタル傾向
   - 新しいオーダー
   - 検査結果の推移

3. **システムを活用**
   - 電子カルテのサマリー機能
   - バイタル一覧画面
   - オーダー履歴

---

## よくある落とし穴

### 避けるべき失敗

| 失敗パターン | 問題点 | 解決策 |
| :--- | :--- | :--- |
| 情報過多 | 重要点が埋もれる | I-PASSで構造化 |
| 情報不足 | 対応が遅れる | チェックリスト使用 |
| 曖昧な表現 | 解釈のずれ | 具体的な数値・条件 |
| 確認なし | 誤解が生じる | Synthesisで確認 |

## 実践例

### Before（構造化前）

> 「3号室の鈴木さんですが、肺炎で入院していて、今日は熱が少し上がって、抗菌薬は継続していて、明日採血予定です。呼吸状態は大丈夫です」

### After（構造化後）

```
【重症度】🟡 要注意

【サマリー】
鈴木太郎（75M）肺炎Day4、CTRX継続中

【本日の変化】
- 発熱38.2℃（昨日37.5℃から上昇）
- SpO2 95%（O2 1L）- 昨日と同等

【To-Do】
□ 発熱時 → アセトアミノフェン頓用
□ 明朝 → 採血でCRP確認

【If-Then】
SpO2 93%以下持続 → O2 2Lに増量、改善なければ報告

【確認事項】
質問ありますか？
```

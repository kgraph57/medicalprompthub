# 当直引き継ぎサマリー作成ガイド

## はじめに

当直業務における引き継ぎは、患者安全の要です。しかし、多忙な業務の中で、漏れなく正確に引き継ぎを行うことは容易ではありません。

> 「あの患者さんの経過観察のこと、伝え忘れていた...」

このような事態を防ぐため、このガイドではAIを活用した効率的で安全な引き継ぎサマリーの作成方法を紹介します。

---

## なぜ引き継ぎが重要なのか？

### 引き継ぎエラーの実態

| 問題 | 発生率 |
| :--- | :--- |
| 情報伝達エラー | 医療事故の約80%に関与 |
| 引き継ぎ時の情報欠落 | 約30%の引き継ぎで発生 |
| 引き継ぎエラーによる有害事象 | 入院患者の約25%に影響 |

---

## I-PASSフレームワークとは

このガイドでは、**I-PASS**フレームワークを採用します。これは米国で開発された、標準化された引き継ぎ方法論です。

### I-PASSの5つの要素

| 要素 | 内容 | 例 |
| :--- | :--- | :--- |
| **I**llness severity | 重症度 | 安定/要注意/不安定 |
| **P**atient summary | 患者サマリー | 主訴、現病歴、主要な問題点 |
| **A**ction list | To-Doリスト | 検査確認、投薬調整など |
| **S**ituation awareness | 状況認識と予測 | 予想される変化、対応計画 |
| **S**ynthesis by receiver | 受け手による要約 | 確認と質問 |

## 注意事項

### 個人情報の取り扱い

⚠️ **重要**: AIツールに患者情報を入力する際は、必ず匿名化してください

- 患者名 → 「Aさん」「70代男性」など
- ID番号 → 削除または架空の番号
- 特定可能な情報 → 一般化

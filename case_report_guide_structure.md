# 【完全版】症例報告執筆ガイド：構想から投稿まで - 詳細構成設計

## 全体コンセプト

**「このガイドを読めば、初めてでも症例報告を完成させて投稿できる」**

- 自己完結型：他の資料を探さなくても完結
- 実行可能：各ステップが具体的で、すぐに行動に移せる
- 時間効率：全体の所要時間とスケジュールが明確
- トラブル対応：詰まったときの解決策を提示
- 品質保証：チェックリストで最低限の品質を担保
- 次世代ツール：最新のAIツールと従来ツールを併記

## 想定読者

- 初めて症例報告を書く研修医・若手医師
- 英語論文の執筆に不慣れな医師
- 忙しい臨床の合間に効率的に論文を書きたい医師

## 全体の所要時間

- **最短**: 20-30時間（集中して取り組む場合）
- **標準**: 2-3ヶ月（週末に数時間ずつ）
- **余裕**: 6ヶ月（月に数回のペース）

## 第1部：導入編（Before You Start）

### 1.1 症例報告とは何か
- **定義**: 1例または少数例の臨床経験を報告する医学論文
- **意義**: 
  - 稀な疾患・症状の報告
  - 既知の疾患の非典型的な提示
  - 新しい治療法の効果
  - 予期しない有害事象
  - 診断・治療の教訓
- **原著論文との違い**: データ数、研究デザイン、統計解析の有無

### 1.2 なぜ症例報告を書くのか
- **医学への貢献**: 稀な症例の知見を共有
- **自己成長**: 文献検索、論理的思考、英語執筆のスキル向上
- **キャリア**: 業績として評価される
- **指導**: 後輩への教育材料

### 1.3 このガイドの使い方
- **全体の流れ**: 12ステップで構成
- **推奨ペース**: 週1-2ステップ
- **チェックリスト**: 各ステップの完了基準を確認
- **固定ナビゲーション**: 左側のサイドバーで現在地を確認

### 1.4 全体のロードマップ（視覚化）

```
準備編 → 情報収集 → 執筆 → 投稿準備 → 投稿・査読
  ↓        ↓         ↓       ↓         ↓
 Step 1   Step 2-4  Step 5-8  Step 9-11  Step 12
 (1週間)  (2-3週間) (3-4週間) (1-2週間)  (変動)
```

## 第2部：準備編（Preparation）

### Step 0: 必要なもの・準備すべきこと

**所要時間**: 1-2時間

#### 0.1 必須アイテム
- [ ] カルテ情報（経過記録、検査結果、画像）
- [ ] 患者同意書（病院の書式を確認）
- [ ] 指導医の承認
- [ ] 倫理委員会の承認（必要な場合）

#### 0.2 推奨ツール

**文献管理**
- 従来: EndNote, Mendeley
- 次世代: **Zotero**（無料、オープンソース）+ Better BibTeX

**文献検索**
- 従来: PubMed, Google Scholar
- 次世代: 
  - **Consensus**（AI論文検索、220M+論文）
  - **Elicit**（研究質問への回答抽出）
  - **Connected Papers**（関連論文の可視化）

**執筆・翻訳**
- 従来: DeepL, Grammarly
- 次世代:
  - **Claude/ChatGPT**（医学論文特化プロンプト）
  - **Writefull**（学術英語特化）
  - **Paperpal**（論文校正AI）

**図表作成**
- 従来: PowerPoint, Illustrator
- 次世代:
  - **BioRender**（生物医学イラスト）
  - **Mermaid**（コードからフローチャート）
  - **Canva**（デザインテンプレート）

#### 0.3 倫理的配慮と同意取得

**患者同意**
- 書面での同意が必須
- 匿名化の方法を説明
- 拒否の権利を明示

**倫理委員会**
- 病院の規定を確認
- 未承認薬・適応外使用の場合は必須
- 申請に1-2ヶ月かかる場合あり

**匿名化のルール**
- 氏名、ID番号は削除
- 日付は「入院1日目」などに変換
- 画像から識別情報を削除
- 稀な疾患の場合は地域情報も配慮

#### 0.4 想定スケジュール

**集中型（1-2ヶ月）**
- 週10-15時間確保
- 毎週2-3ステップ進行

**標準型（2-3ヶ月）**
- 週5-7時間確保
- 毎週1-2ステップ進行

**余裕型（6ヶ月）**
- 週2-3時間確保
- 隔週で1ステップ進行

## 第3部：実践編（Step-by-Step）

### Step 1: 症例の選定と情報収集

**所要時間**: 2-3時間

#### 1.1 目的
報告する価値のある症例を選び、必要な情報を漏れなく収集する。

#### 1.2 報告価値のある症例とは

**以下のいずれかに該当**
- 稀な疾患（有病率 < 1/10,000）
- 既知の疾患の非典型的な提示
- 診断が困難だった症例
- 新しい治療法の効果
- 既知の治療の予期しない効果
- 薬剤の有害事象
- 診断・治療上の教訓

#### 1.3 情報収集チェックリスト

**患者背景**
- [ ] 年齢、性別
- [ ] 主訴
- [ ] 現病歴（発症から受診まで）
- [ ] 既往歴
- [ ] 家族歴（関連する場合）
- [ ] 内服薬
- [ ] アレルギー歴
- [ ] 社会歴（喫煙、飲酒、職業）

**臨床所見**
- [ ] バイタルサイン
- [ ] 身体所見
- [ ] 検査所見（血液、尿、画像）
- [ ] 病理所見（該当する場合）

**診断・治療**
- [ ] 鑑別診断のプロセス
- [ ] 最終診断
- [ ] 治療内容（薬剤名、用量、期間）
- [ ] 治療の変更とその理由

**経過・転帰**
- [ ] 治療への反応
- [ ] 有害事象
- [ ] フォローアップ期間
- [ ] 最終転帰

#### 1.4 使用するプロンプト

**プロンプト名**: `res-case-selection`（症例選定アドバイス）

**入力例**:
```
以下の症例は報告する価値がありますか？

患者：60代男性
主訴：発熱、咳嗽
経過：2週間前から発熱と咳嗽が出現。近医で抗菌薬を処方されたが改善せず。
胸部CTで両側肺野にすりガラス陰影を認めた。
COVID-19、インフルエンザは陰性。
気管支鏡検査で〇〇を認め、△△と診断。
××で治療し、症状は改善した。
```

**出力例**:
```
この症例は報告価値があります。理由：
1. △△は稀な疾患です（有病率：約1/50,000）
2. 初期症状が非特異的で、診断に気管支鏡が必要だった点は教訓的
3. ××による治療効果の報告は限られています

報告のポイント：
- 診断に至るまでの鑑別診断のプロセスを詳述
- 気管支鏡所見の画像を含める
- ××の用量と治療期間を明記
- 類似症例との比較を考察に含める
```

#### 1.5 よくある問題と対処法

**問題**: カルテ情報が不完全
**対処法**: 
- 患者に再度問診（可能な場合）
- 前医に情報提供を依頼
- 不明な点は「不明」と明記

**問題**: 画像データが見つからない
**対処法**:
- 放射線科・検査科に依頼
- PACS（画像管理システム）で検索
- 紙カルテの場合はスキャン

### Step 2: CAREガイドラインチェック

**所要時間**: 1-2時間

#### 2.1 目的
CAREガイドラインに沿って、報告に必要な要素が揃っているか確認する。

#### 2.2 CAREガイドラインとは
- **CARE**: CAse REport guidelines
- 2013年に国際的な専門家グループが開発
- 症例報告の完全性と透明性を確保
- 多くのジャーナルが準拠を要求

#### 2.3 CAREチェックリスト（13項目）

1. **Title**: 診断または介入 + "case report"
2. **Key Words**: 2-5個のキーワード
3. **Abstract**: 構造化抄録
4. **Introduction**: 症例のユニークさ
5. **Patient Information**: 匿名化された患者情報
6. **Clinical Findings**: 身体所見
7. **Timeline**: 時系列の図表
8. **Diagnostic Assessment**: 診断プロセス
9. **Therapeutic Intervention**: 治療内容
10. **Follow-up and Outcomes**: 転帰
11. **Discussion**: 考察
12. **Patient Perspective**: 患者の視点
13. **Informed Consent**: 同意書

#### 2.4 使用するプロンプト

**プロンプト名**: `res-check-care`（CAREチェックリスト）

**入力例**:
```
以下の情報でCAREガイドラインの要件を満たしていますか？

【収集済み情報】
- 患者背景：60代男性、主訴、既往歴
- 臨床所見：バイタル、身体所見、血液検査、CT画像
- 診断：気管支鏡所見、病理診断
- 治療：薬剤名、用量、期間
- 転帰：症状改善、6ヶ月フォロー

【未収集情報】
- 家族歴
- 患者の治療に対する感想
```

**出力例**:
```
CAREガイドライン適合状況：

✅ 満たしている項目：
- Patient Information（患者背景）
- Clinical Findings（臨床所見）
- Diagnostic Assessment（診断評価）
- Therapeutic Intervention（治療介入）
- Follow-up and Outcomes（転帰）

⚠️ 不足している項目：
- Timeline（時系列図表）→ 作成が必要
- Patient Perspective（患者の視点）→ 患者に治療の感想を聞く
- Family history（家族歴）→ 遺伝性疾患でなければ「特記事項なし」でOK

推奨アクション：
1. タイムライン図表を作成（Step 3で実施）
2. 患者に「治療を受けてどう感じたか」を聞く
3. 家族歴の有無を確認
```

### Step 3: タイムライン作成

**所要時間**: 1-2時間

#### 3.1 目的
症例の経過を時系列で整理し、読者が理解しやすい図表を作成する。

#### 3.2 タイムラインの重要性
- CAREガイドラインの必須項目
- 複雑な経過を一目で理解できる
- 診断・治療の時間的関係を明示

#### 3.3 タイムラインに含める情報

**時間軸**
- 発症日を「Day 0」とする
- または「入院1日目」「外来初診日」を起点

**イベント**
- 症状の出現・変化
- 検査の実施と結果
- 診断
- 治療の開始・変更・終了
- 有害事象
- フォローアップ

#### 3.4 タイムラインの形式

**表形式**
| 時期 | イベント | 詳細 |
|------|----------|------|
| Day 0 | 発症 | 発熱38.5℃、咳嗽 |
| Day 14 | 近医受診 | 抗菌薬処方 |
| Day 21 | 当院紹介 | 胸部CT施行 |

**図形式**
```
Day 0 -----> Day 14 -----> Day 21 -----> Day 28
発症      近医受診      当院紹介      診断確定
```

#### 3.5 使用するプロンプト

**プロンプト名**: `res-timeline-builder`（タイムライン作成）

**入力例**:
```
以下のカルテ情報からタイムライン表を作成してください。

2週間前：発熱38.5℃、咳嗽が出現
1週間前：近医受診、抗菌薬（レボフロキサシン500mg/日）処方
3日前：症状改善せず、当院紹介受診
当日：胸部CT施行、両側すりガラス陰影
翌日：気管支鏡検査施行
3日後：病理結果で〇〇と診断
4日後：××開始（△mg/日）
2週間後：症状改善、CT所見改善
6ヶ月後：外来フォロー、再発なし
```

**出力例**:
```markdown
| 時期 | イベント | 詳細 |
|------|----------|------|
| Day -14 | 発症 | 発熱38.5℃、咳嗽出現 |
| Day -7 | 近医受診 | レボフロキサシン 500mg/日 開始 |
| Day -3 | 当院紹介受診 | 症状持続、抗菌薬無効 |
| Day 0 | 胸部CT | 両側すりガラス陰影 |
| Day 1 | 気管支鏡検査 | 実施 |
| Day 3 | 診断確定 | 病理結果：〇〇 |
| Day 4 | 治療開始 | ×× △mg/日 開始 |
| Day 18 | 評価 | 症状改善、CT所見改善 |
| Day 180 | フォローアップ | 外来、再発なし |
```

### Step 4: 文献検索と新規性の確認

**所要時間**: 3-5時間

#### 4.1 目的
類似症例を検索し、自分の症例の新規性（何が新しいか）を明確にする。

#### 4.2 文献検索の重要性
- 既報告との違いを明確にする
- 考察の根拠となる文献を集める
- 投稿先ジャーナルの傾向を把握

#### 4.3 検索戦略

**Step 4.1: キーワードの抽出**
- 疾患名
- 主要な症状・所見
- 治療法
- "case report" or "case series"

**Step 4.2: PubMed検索**

基本検索式:
```
(疾患名[Title/Abstract]) AND ("case report"[Publication Type] OR "case reports"[Title])
```

例:
```
(organizing pneumonia[Title/Abstract]) AND ("case report"[Publication Type])
```

**Step 4.3: 次世代ツールの活用**

**Consensus（推奨）**
- 自然言語で質問できる
- 220M+論文から回答を抽出
- エビデンスレベルを表示

使い方:
1. https://consensus.app/ にアクセス
2. 質問を入力: "What are the treatment options for organizing pneumonia?"
3. 結果から関連論文を選択

**Elicit**
- 研究質問への回答を抽出
- 表形式で比較可能

**Connected Papers**
- 関連論文を可視化
- 重要な論文を発見

#### 4.4 使用するプロンプト

**プロンプト名**: `res-pubmed-query`（PubMed検索式作成）

**入力例**:
```
以下の症例の類似報告を検索するPubMed検索式を作成してください。

診断：器質化肺炎
主要所見：両側すりガラス陰影
治療：ステロイド
```

**出力例**:
```
推奨検索式：

基本検索:
("organizing pneumonia"[Title/Abstract] OR "cryptogenic organizing pneumonia"[Title/Abstract] OR "COP"[Title/Abstract]) AND ("case report"[Publication Type] OR "case series"[Title/Abstract])

詳細検索（画像所見を含む）:
("organizing pneumonia"[Title/Abstract]) AND ("ground glass"[Title/Abstract] OR "GGO"[Title/Abstract]) AND ("case report"[Publication Type])

治療に焦点:
("organizing pneumonia"[Title/Abstract]) AND ("corticosteroid"[Title/Abstract] OR "steroid"[Title/Abstract]) AND ("case report"[Publication Type])

検索のコツ:
- 最初は広く検索（基本検索）
- ヒット数が多すぎる場合は詳細検索
- 過去5年に絞る: AND ("2020"[Date - Publication] : "2025"[Date - Publication])
```

#### 4.5 文献管理

**Zoteroの使い方**

1. **インストール**
   - https://www.zotero.org/ からダウンロード
   - ブラウザ拡張機能をインストール

2. **文献の追加**
   - PubMedで論文を開く
   - ブラウザのZoteroボタンをクリック
   - 自動的にライブラリに追加

3. **引用文献リストの作成**
   - Word/Google Docsのプラグインを使用
   - 引用形式を選択（Vancouver, AMA等）
   - 自動的に引用番号と文献リストを生成

#### 4.6 新規性の特定

**プロンプト名**: `res-gap-analysis`（ギャップ特定）

**入力例**:
```
以下の類似症例と比較して、私の症例の新規性は何ですか？

【私の症例】
60代男性、器質化肺炎、ステロイド治療で改善

【類似症例1】
50代女性、器質化肺炎、ステロイド治療で改善（文献A）

【類似症例2】
70代男性、器質化肺炎、ステロイド治療で改善（文献B）
```

**出力例**:
```
新規性の候補：

1. 年齢・性別の違い
   → 60代男性は報告が少ない可能性（要確認）

2. 臨床所見の違い
   → あなたの症例特有の所見はありますか？
   （例：特殊な画像所見、検査値の異常）

3. 治療反応の違い
   → ステロイドの用量・期間は？
   → 治療抵抗性だった？または急速に改善？

4. 合併症・有害事象
   → 既報告にない合併症は？

5. フォローアップ期間
   → 長期フォローの報告は少ない可能性

推奨アクション：
- 類似症例の詳細（年齢、性別、治療内容）を表にまとめる
- あなたの症例との違いを明確にする
- 「〇〇の点で既報告と異なる」と言語化する
```

### Step 5: 症例提示（Case Presentation）の執筆

**所要時間**: 3-4時間

#### 5.1 目的
症例の臨床経過を、読者が理解しやすい英語で記述する。

#### 5.2 Case Presentationの構成

1. **Patient Information**（患者情報）
2. **Clinical Findings**（臨床所見）
3. **Diagnostic Assessment**（診断評価）
4. **Therapeutic Intervention**（治療介入）
5. **Follow-up and Outcomes**（転帰）

#### 5.3 執筆の原則

**客観的に**
- 事実のみを記述
- 解釈や考察は含めない

**時系列で**
- タイムラインに沿って記述
- 過去形を使用

**具体的に**
- 数値を明記（体温、検査値、用量）
- 曖昧な表現を避ける

#### 5.4 セクション別の書き方

**Patient Information**

悪い例:
```
高齢男性が発熱と咳で来院した。
```

良い例:
```
A 65-year-old man presented to our hospital with a 2-week history of fever (up to 38.5°C) and dry cough. He had no significant medical history and was not taking any medications. He was a non-smoker and had no history of occupational exposure.
```

**Clinical Findings**

```
On physical examination, his vital signs were as follows: body temperature, 38.2°C; blood pressure, 120/70 mmHg; heart rate, 88 beats/min; respiratory rate, 20 breaths/min; and oxygen saturation, 94% on room air. Chest auscultation revealed fine crackles in both lower lung fields.
```

**Diagnostic Assessment**

```
Laboratory tests showed elevated C-reactive protein (8.5 mg/dL) and lactate dehydrogenase (450 U/L). Chest computed tomography (CT) revealed bilateral ground-glass opacities predominantly in the subpleural regions (Figure 1). Bronchoscopy with transbronchial lung biopsy was performed, and histopathological examination demonstrated organizing pneumonia.
```

**Therapeutic Intervention**

```
The patient was treated with oral prednisolone at an initial dose of 0.5 mg/kg/day (30 mg/day). The dose was gradually tapered over 6 months.
```

**Follow-up and Outcomes**

```
His symptoms improved within 2 weeks of treatment initiation. Follow-up chest CT at 3 months showed complete resolution of the ground-glass opacities. At 6 months, he remained asymptomatic with no recurrence.
```

**間違い1: 曖昧な表現**

❌ "The patient was examined by us."

### Step 7: 考察（Discussion）の執筆

**所要時間**: 4-5時間

#### 7.1 目的
Step 6で設計した構成に沿って、Discussionを執筆する。

#### 7.2 執筆のコツ

**主張→根拠の順で**
```
✅ Organizing pneumonia typically responds well to corticosteroids [1]. In our case, the patient showed rapid improvement within 2 weeks.

❌ The patient showed rapid improvement. Organizing pneumonia responds well to corticosteroids.
```

**文献を適切に引用**
```
✅ Previous studies have reported that 80-90% of patients with organizing pneumonia respond to corticosteroids [1,2].

❌ Most patients respond to corticosteroids.（根拠なし）
```

**比較を明確に**
```
✅ Unlike the case reported by Smith et al. [1], our patient had no history of drug exposure.

❌ Our case is different.（何が違うか不明）
```

### Step 8: Introduction/Abstractの作成

**所要時間**: 2-3時間

#### 8.1 目的

#### 8.2 Introductionの構成

```
Organizing pneumonia is a clinicopathological syndrome characterized by granulation tissue in the alveoli. It can be idiopathic or secondary to various causes.
```

```
However, the diagnosis is often delayed because the clinical and radiological findings are nonspecific and can mimic infectious pneumonia.
```

```
Here, we report a case of organizing pneumonia in which the lack of response to antibiotics was a key diagnostic clue.
```

**構造化抄録（推奨）**

```

A 65-year-old man presented with fever and cough unresponsive to antibiotics. Chest CT showed bilateral ground-glass opacities. Bronchoscopic biopsy confirmed organizing pneumonia. He responded well to corticosteroids.
```

```
Organizing pneumonia should be considered in patients with antibiotic-refractory pneumonia. Early diagnosis and corticosteroid therapy are essential.
```

### Step 12: 投稿と査読対応

**所要時間**: 変動（査読に1-3ヶ月）

#### 12.1 目的
オンラインシステムから投稿し、査読コメントに適切に対応する。

#### 12.2 投稿手順

**1. アカウント作成**
- ジャーナルのウェブサイトにアクセス
- "Submit Manuscript"をクリック
- アカウントを作成（メールアドレス、パスワード）

**2. 投稿フォーム入力**
- Article Type: Case Report
- Title, Abstract, Keywordsを入力
- 著者情報（名前、所属、メールアドレス）
- Corresponding Author（連絡先著者）を指定

**3. ファイルアップロード**
- Manuscript（本文）: Word形式
- Figures: 個別にアップロード
- Tables: Wordに埋め込むか個別ファイル
- Cover Letter: PDF or Word

**4. 確認と送信**
- PDFプレビューで確認
- "Submit"をクリック
- 受付番号（Manuscript ID）を受け取る

#### 12.3 査読プロセス

**1. Editorial Check（1-2週間）**
- エディターが投稿規定を確認
- 不備があれば修正依頼

**2. Peer Review（1-2ヶ月）**
- 2-3名の査読者が評価
- コメントが返ってくる

**3. 判定**
- **Accept**: 採択（そのまま掲載）
- **Minor Revision**: 軽微な修正
- **Major Revision**: 大幅な修正
- **Reject**: 不採択

#### 12.4 査読コメントへの対応

**基本方針**
- すべてのコメントに丁寧に回答
- 感情的にならない
- 論理的に反論（必要な場合）

**Response Letter（返答レター）の構成**

```
Dear Editor,

Thank you for the opportunity to revise our manuscript. We appreciate the reviewers' constructive comments. We have carefully addressed all comments as follows:

Reviewer 1:
Comment 1: The timeline figure is unclear.
Response: We have revised Figure 2 to improve clarity. The revised figure now includes specific dates and treatment details.

Comment 2: The discussion lacks comparison with similar cases.
Response: We have added a comparison table (Table 1) summarizing five similar cases from the literature (references 5-9).

Reviewer 2:
Comment 1: The informed consent statement is missing.
Response: We have added the informed consent statement in the Methods section (page 3, line 45).

We believe that these revisions have substantially improved the manuscript. Thank you again for your consideration.

Sincerely,
[Your Name]
```

#### 12.5 使用するプロンプト

**プロンプト名**: `res-reviewer-response`（査読対応）

**入力例**:
```
以下の査読コメントへの返答を作成してください。

Reviewer 1:
"The authors should provide more details about the bronchoscopy procedure."

私の対応:
気管支鏡の詳細（使用機器、麻酔方法、生検部位）をMethodsに追加しました。
```

**出力例**:
```
Reviewer 1, Comment 1: The authors should provide more details about the bronchoscopy procedure.

Response: Thank you for this valuable comment. We have added detailed information about the bronchoscopy procedure in the Case Presentation section (page 4, lines 78-82). Specifically, we have described the bronchoscope model (Olympus BF-260), the anesthesia method (topical lidocaine), and the biopsy sites (right middle lobe and left lower lobe). We believe this additional information enhances the reproducibility of our diagnostic approach.
```

#### 12.6 よくある査読コメント

**コメント1**: "The case lacks novelty."
**対応**: 新規性を再度強調。類似症例との違いを明確にする。

**コメント2**: "The discussion is too long."
**対応**: 冗長な部分を削除。1パラグラフ1主張に絞る。

**コメント3**: "The figures are of poor quality."
**対応**: 解像度を上げる。注釈を追加して分かりやすくする。

**コメント4**: "The English needs improvement."
**対応**: 英文校正サービス（Editage, AJE等）を利用。

#### 12.7 不採択（Reject）の場合

**次のアクション**
1. 査読コメントを冷静に読む
2. 改善点を論文に反映
3. 別のジャーナルに投稿

**ジャーナルの選び方**
- 同レベルまたは少し低いImpact Factor
- より広いスコープのジャーナル
- Open Accessジャーナル

### FAQ

**Q1: 症例報告に倫理委員会の承認は必要ですか？**

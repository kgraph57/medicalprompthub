# ステップ5: 図解を挿入する

## 📋 このステップの目的

文章だけでは伝わりにくい情報を、図解で視覚化します。このステップでは、図解の挿入タイミング、スタイル、方法を学びます。

## ⏱️ 所要時間

20分

## 🎯 達成基準

- 図解を挿入すべきタイミングを判断できる
- Helix推奨の図解スタイルを理解している
- 実際にMarkdownで図解を挿入できる

## 📝 詳細手順

### 手順1: 図解を挿入すべきタイミングを判断する

図解は、以下のタイミングで挿入すると効果的です。

#### タイミング1: セクションの冒頭

全体像を示すために、セクションの冒頭に図解を挿入します。

**例**:
```markdown
# ステップ1: 症例の選定

![症例選定のワークフロー](./workflow_diagram.png)

## 目的
...
```

#### タイミング2: 複雑な手順の前

ワークフローを視覚化するために、複雑な手順の前に図解を挿入します。

**例**:
```markdown
## 詳細手順

![文献検索のステップ](./search_steps.png)

### 手順1: PubMedで検索
...
```

#### タイミング3: 重要な概念の説明

図で理解を助けるために、重要な概念の説明に図解を挿入します。

**例**:
```markdown
## PICO形式とは？

![PICO形式の説明](./pico_diagram.png)

PICO形式は、臨床疑問を構造化するためのフレームワークです。
...
```

### 手順2: Helix推奨の図解スタイルを理解する

Helixでは、以下のスタイルの図解を推奨しています。

#### スタイル1: 手描き風

Notion/YouMind/Manusスタイルの手描き風図解を使います。

**特徴**:
- 親しみやすい
- シンプル
- カラフル（パステルカラー）

**ツール**:
- Excalidraw
- Figma
- Miro

#### スタイル2: シンプル

情報を詰め込みすぎず、シンプルに保ちます。

**ポイント**:
- 1つの図解に1つのメッセージ
- 文字は最小限
- 矢印やアイコンを活用

#### スタイル3: カラフル

パステルカラーで親しみやすく仕上げます。

**推奨カラーパレット**:
- 青系: #E3F2FD, #BBDEFB
- 緑系: #E8F5E9, #C8E6C9
- 黄系: #FFF9C4, #FFF59D
- ピンク系: #FCE4EC, #F8BBD0

### 手順3: Markdownで図解を挿入する

Markdownで図解を挿入する方法は簡単です。

**基本形式**:
```markdown
![図解のタイトル](./image_name.png)
```

**例**:
```markdown
![症例選定のワークフロー](./workflow_diagram.png)
```

**ポイント**:
- 画像ファイルは、Markdownファイルと同じディレクトリに配置
- 相対パスで参照（`./image_name.png`）
- タイトルは簡潔に

### 手順4: 図解のファイル名を決める

図解のファイル名は、以下のルールで決めます。

**ルール**:
- 小文字とアンダースコア（`_`）を使う
- 内容がわかる名前にする
- 拡張子は`.png`

**例**:
- `workflow_diagram.png`
- `search_steps.png`
- `pico_diagram.png`
- `consent_process.png`

### 手順5: 図解を作成する

図解を作成するツールは、以下がおすすめです。

#### Excalidraw（推奨）

手描き風の図解を簡単に作成できます。

**使い方**:
1. https://excalidraw.com/ を開く
2. 図形やテキストを配置
3. PNGでエクスポート

#### Figma

より洗練された図解を作成できます。

**使い方**:
1. Figmaでデザインを作成
2. PNGでエクスポート

#### Miro

ワークフローや複雑な図解に適しています。

**使い方**:
1. Miroでボードを作成
2. PNGでエクスポート

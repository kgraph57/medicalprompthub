# ステップ1: 目標とする文体を理解する

## 📋 このステップの目的

トーンマナー修正の第一歩は、「どんな文体を目指すのか」を明確にすることです。このステップでは、Helixが目指す文体の特徴を学びます。

## ⏱️ 所要時間

10分

## 🎯 達成基準

- Helixが目指す文体の特徴を3つ以上、言葉で説明できる
- 参考スタイル（Kensuu、Notion、Manus）の共通点を理解している

## 📝 詳細手順

### 手順1: 基本方針を理解する

Helixのドキュメントは、以下の4つの基本方針に基づいて作成されます。

#### 1. 親しみやすさ

堅苦しくなく、読者に寄り添う文章を目指します。

**悪い例**:
```
以下の手順を実行する。データを収集する必要がある。
```

**良い例**:
```
以下の手順を実行します。データを収集する必要があります。
```

「です・ます」調で統一することで、読者との距離が縮まります。

#### 2. 実践的

すぐに使える具体的な情報を提供します。

**悪い例**:
```
適切に対応する。
```

**良い例**:
```
指導医に相談して、次のステップを決めます。
```

抽象的な表現を避け、具体的なアクションを示します。

#### 3. わかりやすさ

専門用語は必要最小限にし、説明を添えます。

**悪い例**:
```
エビデンスレベルの高いRCTを参照する。
```

**良い例**:
```
信頼性の高い臨床試験（RCT）を参照します。
```

専門用語には、必ず説明を添えます。

#### 4. 視覚的

図解やリストを活用して、理解を助けます。

**悪い例**:
```
症例報告は、臨床現場で遭遇した稀な疾患や予期せぬ経過、新しい治療法の効果など、医学的に価値のある情報を学術コミュニティと共有するための重要な手段である。
```

**良い例**:
```
症例報告は、臨床現場の貴重な経験を学術コミュニティと共有する手段です。

以下のような情報を伝えます：
- 稀な疾患
- 予期せぬ経過
- 新しい治療法の効果
```

長い文章は分割し、リストを活用します。

### 手順2: 参考スタイルを確認する

Helixは、以下の3つのスタイルを参考にしています。

#### Kensuu (梅田健太郎)

nanapi創業者、起業家。わかりやすく実践的な文章で知られます。

**特徴**:
- 具体例が豊富
- 比喩やたとえ話を活用
- 読者に直接語りかける

**例**:
```
「これは、みんなに知ってほしい！」と思った症例です。
```

#### Notion/YouMind

親しみやすく、視覚的にわかりやすいドキュメント。

**特徴**:
- シンプルなデザイン
- 図解やアイコンを活用
- 情報を詰め込みすぎない

**例**:
```
📋 このステップの目的
⏱️ 所要時間
🎯 達成基準
```

#### Manus

ユーザーフレンドリーで実用的なガイド。

**特徴**:
- ステップバイステップの説明
- 実践的な例
- フィードバックを重視

**例**:
```
1. まずは全体を読む
2. ルールを確認する
3. 実践する
```

### 手順3: 共通点を見つける

これら3つのスタイルには、以下の共通点があります。

| 共通点 | 説明 |
|---|---|
| 読者目線 | 読者が何を知りたいか、何に困っているかを常に意識している |
| 具体的 | 抽象的な表現を避け、具体例を豊富に使う |
| 視覚的 | 図解やリスト、アイコンを活用して理解を助ける |
| 親しみやすい | 堅苦しくなく、読者に寄り添う語り口 |

## 💡 ヒント

- **最初は完璧を目指さない**: まずは基本方針を理解することが大切です。実践しながら、徐々に身につけていきましょう。
- **参考スタイルを読む**: Kensuu、Notion、Manusの実際のドキュメントを読んで、トーンを体感してください。
- **フィードバックをもらう**: 修正したドキュメントを他の人に読んでもらい、「読みやすいか」「わかりやすいか」を確認してください。

# 感染対策マニュアル作成ガイド：応用編1

## AIによるサーベイランスデータ分析とリスク予測

マニュアル作成後の最も重要な課題の一つは、マニュアルが現場で適切に機能しているかを評価し、改善を続けることです。このステップでは、AIを感染対策のサーベイランス（監視）とリスク予測に活用し、マニュアルの「生きた運用」を支援する方法を探ります。

### 1. AIを活用したサーベイランスデータの解析

病院情報システム（HIS）や電子カルテに蓄積された膨大なデータ（微生物検査結果、抗菌薬使用量、患者の入退院情報など）は、人間の手で全てを分析するには限界があります。AIは、これらのデータを高速で処理し、感染対策上の重要なシグナルを検出できます。

具体的な手順:

1.  データ連携の準備: 匿名化されたサーベイランスデータ（例：菌種、検出場所、日時、耐性パターン）をCSV形式などでAIにインプットできる状態にします。
2.  異常値の検出指示: AIに対し、「過去6ヶ月間の平均値と比較して、特定の菌種（例：MRSA、C.difficile）の検出率が標準偏差の2倍を超えた病棟」を検出するよう指示します。
3.  クラスタリング分析: 感染経路の特定を支援するため、AIに「時間的・地理的に近接して発生した感染事例」をグループ化（クラスタリング）するよう依頼します。

### 2. 実用的なプロンプト例：異常シグナルの検出と報告書作成

ここでは、AIにサーベイランスデータの一部をインプットし、異常シグナルを検出させ、その結果をICD（感染制御医）向けの報告書形式で出力させるプロンプト例を示します。

プロンプト:

```
あなたは、感染対策チーム（ICT）のデータアナリストです。
以下の微生物検査データ（架空）を分析し、ICD向けの「週次サーベイランス報告書」を作成してください。

[データ（例）]
- 菌種: MRSA, 病棟: 3A, 検出数: 5件 (先週: 2件, 過去6ヶ月平均: 2.5件)
- 菌種: ESBL産生菌, 病棟: 5B, 検出数: 1件 (先週: 1件, 過去6ヶ月平均: 0.8件)
- 菌種: C.difficile, 病棟: 2C, 検出数: 4件 (先週: 3件, 過去6ヶ月平均: 3.0件)

[報告書の要件]
1.  異常シグナルの特定: 過去6ヶ月平均と比較して、検出数が2倍以上に増加している菌種と病棟を「要警戒シグナル」として特定すること。
2.  考察の提案: 特定されたシグナルに対し、考えられる原因（例：特定の医療行為の増加、手指衛生の遵守率低下）を2つ提案すること。
3.  推奨される行動: ICTとして、直ちに推奨される行動（例：3A病棟での手指衛生監査の強化）を具体的に提案すること。
4.  報告書は、結論から先に述べる簡潔なビジネス文書形式で記述すること。
```

### 3. 注意点とTips

*   Tips: リアルタイム監視の構築: 理想的には、AIを電子カルテシステムと連携させ、異常シグナルをリアルタイムでICTメンバーにアラートとして通知するシステムを構築します。AIは、マニュアルの改訂が必要な「現場の失敗」をデータから教えてくれる最高の教師となります。
*   注意点: データの匿名化と倫理: 医療データをAIに分析させる際は、個人情報保護と医療倫理を最優先する必要があります。AIにインプットするデータは、患者を特定できないよう完全に匿名化されていることを確認してください。

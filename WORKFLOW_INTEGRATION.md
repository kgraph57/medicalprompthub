# ワークフロー統合とオンボーディング設計

## 🎯 コンセプト

Medical Prompt Hubの価値は「プロンプトの提供」ではなく、「実際の業務フローへの統合」と「段階的なスキルアップ」にあります。

## 📋 現在の課題

1. **単発のプロンプト提供**: ユーザーが「どのプロンプトをいつ使うか」が分からない
2. **オンボーディング不足**: 初心者が「どこから始めればいいか」が分からない
3. **ワークフロー統合の欠如**: 実際の業務プロセスにどう組み込むかが不明確

## 🚀 解決策：ワークフロー統合型ガイド

### 1. オンボーディングフロー

#### レベル1: 初心者向け「今日から使える3ステップ」
```
Step 1: 自分の役割を選択
  - 医師（臨床）
  - 医師（研究）
  - 看護師
  - 薬剤師
  - 学生・研修医

Step 2: よくあるシナリオから選択
  - 「診療録を書く時間を短縮したい」
  - 「患者説明がうまくいかない」
  - 「症例報告を書きたいが何から始めれば...」
  - 「学会発表の準備を効率化したい」

Step 3: 最初の1つのプロンプトを試す
  - 実際のケースで試す
  - 結果を確認
  - フィードバックを提供
```

#### レベル2: 中級者向け「ワークフローに組み込む」
```
- 日常業務のどのタイミングで使うか
- 複数のプロンプトをどう組み合わせるか
- 効率化のポイント
```

#### レベル3: 上級者向け「カスタマイズと最適化」
```
- 自分の施設・専門に合わせたカスタマイズ
- チームでの共有方法
- 継続的な改善
```

### 2. シナリオベースのガイド設計

#### 例：症例報告作成ワークフロー（改善版）

**Before（現在）**: 10ステップのリスト形式

**After（改善案）**: 実際のタイムラインに沿ったワークフロー

```
【Day 1: 症例の選定】
  09:00 - カンファレンスで症例を報告
  10:00 - 「この症例、報告に値するか？」をAIに相談
  11:00 - CAREガイドラインでチェック
  12:00 - 不足情報をカルテから収集

【Day 2-3: 文献検索と準備】
  09:00 - PubMed検索式を作成
  10:00 - 類似症例を検索
  14:00 - 新規性を明確化
  15:00 - ターゲットジャーナルを選定

【Day 4-7: 執筆】
  09:00 - Case Presentationを書く
  11:00 - Discussionを書く
  14:00 - IntroductionとAbstractを書く
  16:00 - 推敲と校正

【Day 8-10: 投稿準備】
  09:00 - 指導医にレビュー依頼
  10:00 - カバーレター作成
  11:00 - 投稿
```

### 3. ロール別のワークフロー

#### 臨床医向け「1日の診療フロー」
```
【朝の準備】
  08:00 - 前日の診療録を確認
  08:15 - 本日の予定患者の情報を整理
  08:30 - 必要なプロンプトを準備

【診療中】
  10:00 - 患者面接 → 症状分析プロンプト
  10:15 - 身体診察 → 診断推論プロンプト
  10:30 - 検査オーダー → 検査選択プロンプト
  11:00 - 診断確定 → 治療計画プロンプト

【診療後】
  12:00 - 診療録作成 → SOAPプロンプト
  13:00 - 紹介状作成 → 紹介状プロンプト
  14:00 - 患者説明 → 患者説明プロンプト
```

#### 研究者向け「論文執筆フロー」
```
【Week 1: 準備】
  - 研究質問の明確化（PICO）
  - 文献検索戦略の立案
  - データ整理計画

【Week 2-4: 執筆】
  - Methods執筆
  - Results執筆と図表作成
  - Discussion執筆
  - IntroductionとAbstract

【Week 5-6: 投稿準備】
  - ジャーナル選定
  - カバーレター作成
  - 投稿
```

### 4. プログレッシブオンボーディング

#### 段階1: 観察（Observe）
- 他のユーザーがどう使っているかを見る
- サンプルケースを読む
- ビデオチュートリアルを見る

#### 段階2: 模倣（Imitate）
- テンプレートをそのまま使う
- サンプルケースを参考にする
- 基本的なプロンプトから始める

#### 段階3: 実践（Practice）
- 自分のケースで試す
- 結果を評価する
- フィードバックを得る

#### 段階4: 統合（Integrate）
- 日常業務に組み込む
- 複数のプロンプトを組み合わせる
- 効率化のポイントを見つける

#### 段階5: 最適化（Optimize）
- 自分のスタイルにカスタマイズ
- チームで共有する
- 継続的に改善する

## 🛠️ 実装すべき機能

### 1. オンボーディングウィザード
```
1. ロール選択（医師/看護師/薬剤師/学生）
2. よくあるシナリオから選択
3. 最初のプロンプトを試す
4. 結果を確認してフィードバック
5. 次のステップを提案
```

### 2. ワークフロービルダー
```
- 自分の業務フローを可視化
- 各ステップで使うプロンプトを設定
- テンプレートとして保存
- チームで共有
```

### 3. プログレストラッキング
```
- どのプロンプトを何回使ったか
- どのガイドを完了したか
- スキルレベル（初心者→中級者→上級者）
- 達成バッジ・認定証
```

### 4. シナリオテンプレート
```
- 「初めての症例報告」
- 「忙しい外来での診療録作成」
- 「緊急時の患者説明」
- 「学会発表の準備」
```

### 5. コミュニティ機能
```
- 他のユーザーのワークフローを見る
- ベストプラクティスを共有
- Q&A
- 成功事例の紹介
```

## 📊 ガイドの改善例

### Before: ステップの羅列
```
Step 1: 症例の選定
Step 2: 文献検索
Step 3: 執筆
...
```

### After: 実際のワークフローに統合
```
【シナリオ】あなたは研修医で、初めて症例報告を書くことになりました。

【Day 1: 準備】
  → カンファレンスで症例を報告した後、指導医に「報告に値するか？」を相談
  → AIプロンプト「症例価値評価」で自己チェック
  → 不足情報をカルテから収集

【Day 2-3: 文献検索】
  → PubMed検索式を作成（プロンプト使用）
  → 類似症例を検索
  → 新規性を明確化

【実際のタイムライン】
  - 朝のカンファレンス後（10分）
  - 昼休みに文献検索（30分）
  - 帰宅後に執筆（1時間×3日）
```

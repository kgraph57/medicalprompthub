# 日本語文章作成・完全指示書（医療 × 教育 × 技術解説 × note/書籍対応）

## はじめに

この指示書は、医師としての信頼性を保ちながら、多様なジャンルで効果的な日本語文章を作成するための完全ガイドです。テクニカルライティングの原則と、日本語の特性を深く理解した上で、実践的なルールを体系化しています。

---

## 第1章｜日本語文章の本質構造（ディープリサーチ統合）

### 1.1 可読性の3層構造

日本語文章の可読性は、以下の3層で構成されます：

#### 論理層（Logical Layer）
- **定義**：文章全体の論理的な流れと構造
- **要素**：段落構成、論理展開、情報の階層化
- **重要性**：読者が内容を理解し、記憶に定着させるための基盤

#### 文層（Sentence Layer）
- **定義**：個々の文の構造と表現
- **要素**：文の長さ、接続詞の使用、主語・述語の関係
- **重要性**：一文一文の明確性が全体の理解度を左右する

#### 視覚層（Visual Layer）
- **定義**：文章の視覚的な配置とレイアウト
- **要素**：改行、段落の長さ、見出し、箇条書き、余白
- **重要性**：読者の視覚的負担を軽減し、情報の吸収を促進する

### 1.2 日本語の「後出し言語」特性

日本語は「後出し言語」であり、重要な情報が文末に配置される特性があります。

#### 基本原則
- **結論を後ろに置く**：重要な情報（述語、結論）は文末に配置
- **修飾関係の明確化**：長い修飾語句は被修飾語の直前に配置
- **情報の階層化**：補足情報→主要情報の順で配置

#### 実践例
```
❌ 悪い例：AIは、医療現場で、診断支援システムとして、多くの病院で、導入されている。
✅ 良い例：多くの病院で、診断支援システムとして、医療現場にAIが導入されている。
```

### 1.3 医師としての信頼性を保つ表現

#### 避けるべき表現
- **YouTuber的表現**：「めちゃくちゃ」「やばい」「すごい」などの過度にカジュアルな表現
- **断定的すぎる表現**：「必ず」「絶対に」「間違いなく」などの過度な断定
- **感情的な表現**：過度な感動詞、感嘆符の多用

#### 推奨される表現
- **客観的な表現**：事実に基づいた記述、数値やエビデンスの提示
- **適度な謙遜**：「と考えられる」「可能性がある」「一般的には」などの適切な留保
- **専門性の示し方**：専門用語の適切な使用と、初出時の簡潔な説明

## 第2章｜ジャンル別「正解の文章設計」

### A. 技術解説・AIガイド

#### 基本パラメータ
- **文体**：です・ます調（読みやすさ重視）
- **読者**：技術に興味があるが、専門家ではない層
- **目的**：再現可能性の高い情報提供
- **感情の抑制**：最小限（比喩は控えめに）

#### 構造設計
```
1. 結論（Conclusion）
   - この記事で得られる具体的な成果を1文で提示
   - 例：「この記事を読むと、ChatGPT APIをPythonで呼び出す最小実装ができるようになります」

2. 前提（Premise）
   - 必要な知識・環境・ツールを箇条書きで明示
   - 例：「前提条件：Python 3.8以上、OpenAI APIキー、requestsライブラリ」

3. 手順・構造（Procedure/Structure）
   - ステップバイステップで説明
   - 各ステップに「なぜ」を1文で補足
   - コード例は必ず動作確認済みであることを明記

4. 失敗例（Failure Examples）
   - よくある間違いとその理由を簡潔に
   - 例：「エラー：APIキーが無効 → 原因：環境変数の設定漏れ」

5. 最小実装例（Minimal Implementation Example）
   - コピペで動く最小限のコード
   - 動作確認済みであることを明記
```

#### 文レベルルール
- **1文の長さ**：60文字以内を目安（専門用語が多い場合は80文字まで可）
- **接続詞の使用**：「また」「さらに」「一方で」を適切に使用
- **指示語の使用**：「これ」「それ」は1段落内で1回まで

#### 視覚設計
- **段落の長さ**：3-5文（150-250文字）
- **改行**：段落間は1行空ける
- **コードブロック**：必ず言語指定と説明を付ける

### B. 医療教育・指導書

#### 基本パラメータ
- **文体**：だ・である調（学術的権威性）
- **読者**：医療従事者、医学生
- **目的**：正確な知識の伝達と実践への橋渡し
- **感情の抑制**：完全に抑制（事実のみ）

#### 構造設計
```
1. 定義・概念（Definition/Concept）
   - 専門用語の明確な定義
   - 例：「敗血症性ショックとは、感染症に起因する循環不全により、組織灌流が維持できなくなった状態を指す」

2. 判断基準・数値（Criteria/Numerical Values）
   - 診断基準、カットオフ値、基準範囲を必ず明記
   - 数値は半角で統一
   - 例：「SOFAスコア2点以上で敗血症と診断する」

3. 機序・病態（Mechanism/Pathophysiology）
   - なぜそうなるのかを論理的に説明
   - 図表がある場合は参照を明記

4. 治療・対応（Treatment/Management）
   - エビデンスレベルを可能な限り明記
   - 推奨度と根拠を分けて記述
   - 例：「推奨：早期抗菌薬投与（エビデンスレベル：A）」

5. 注意点・落とし穴（Pitfalls）
   - よくある誤解や注意すべき点
   - 例：「注意：CRP上昇のみでは敗血症を診断できない」
```

#### 文レベルルール
- **1文の長さ**：80文字以内（複雑な概念は100文字まで可）
- **主語の明示**：省略しない（「〜と考えられる」ではなく「〜は〜と考えられる」）
- **数値の表記**：必ず単位を明記（「10mg/dL」ではなく「10 mg/dL」）

#### 視覚設計
- **段落の長さ**：4-6文（200-300文字）
- **改行**：段落間は1行空ける
- **図表**：必ずキャプションと出典を明記

### C. note / ブログ（医師の発信）

#### 基本パラメータ
- **文体**：です・ます調（親しみやすさ）
- **読者**：一般読者、医療に興味がある層
- **目的**：理解、共感、信頼の獲得
- **感情の抑制**：適度に抑制（比喩は使用可、ただし控えめに）

#### 構造設計
```
1. 価値提案（Value Proposition）
   - 読者が得られる具体的な価値を提示
   - 例：「この記事を読むと、AI診断支援システムの現状と課題が理解できるようになります」

2. 誤解の先回り（Preempting Misunderstandings）
   - よくある誤解を先に指摘
   - 例：「AIが医師を代替するわけではありません。あくまで支援ツールです」

3. 本論（Main Body）
   - 具体例を交えながら説明
   - 専門用語は初出時に簡潔に説明
   - 数値やエビデンスを適切に引用

4. 行動・視点の提示（Action/Perspective）
   - 読者に取ってほしい行動や、持ってほしい視点を提示
   - 例：「次回の診察時には、AI診断支援の可能性について主治医に尋ねてみてください」
```

#### 文レベルルール
- **1文の長さ**：50文字以内を目安（読みやすさ重視）
- **接続詞の使用**：「そして」「しかし」「つまり」を適切に使用
- **指示語の使用**：「これ」「それ」は1段落内で1回まで

#### 視覚設計
- **段落の長さ**：2-4文（100-200文字）
- **改行**：段落間は1行空ける
- **見出し**：適切に使用し、視覚的な階層を作る

### D. 学術論文（参考）

#### 基本パラメータ
- **文体**：だ・である調（学術的権威性）
- **読者**：専門家
- **目的**：新たな知見の提示と議論の促進
- **感情の抑制**：完全に抑制（事実のみ）

#### 構造設計
```
1. 序論（Introduction）
   - 研究背景と目的を明確に
   - 先行研究の簡潔なレビュー

2. 方法（Methods）
   - 再現可能な詳細な記述
   - 統計手法の明記

3. 結果（Results）
   - 客観的事実のみを記述
   - 解釈は含めない

4. 考察（Discussion）
   - 結果の解釈と意義
   - 限界の明記

5. 結論（Conclusion）
   - 簡潔な要約
```

#### 文レベルルール
- **1文の長さ**：100文字以内（複雑な概念は120文字まで可）
- **主語の明示**：必ず明示
- **数値の表記**：必ず統計的有意性を明記

#### 視覚設計
- **段落の長さ**：5-7文（250-350文字）
- **改行**：段落間は1行空ける
- **図表**：必ずキャプションと出典を明記

## 第3章｜文レベル完全ルール（職人領域）

### 3.1 文の長さ

#### 基本原則
- **技術解説・ブログ**：50-60文字
- **医療教育**：60-80文字
- **学術論文**：80-100文字

#### 長文の分割方法
```
❌ 悪い例：AI診断支援システムは、医療現場での診断精度向上に貢献する可能性があるが、一方で、医師の判断を補完するツールとして位置づけられるべきであり、完全に医師を代替するものではない。

✅ 良い例：AI診断支援システムは、医療現場での診断精度向上に貢献する可能性がある。一方で、医師の判断を補完するツールとして位置づけられるべきである。完全に医師を代替するものではない。
```

### 3.2 接続詞の使用

#### 推奨接続詞（ジャンル別）

**技術解説・ブログ**
- 順接：「そして」「また」「さらに」
- 逆接：「しかし」「一方で」「ただし」
- 説明：「つまり」「すなわち」「例えば」

**医療教育・学術論文**
- 順接：「また」「さらに」「加えて」
- 逆接：「しかし」「一方で」「ただし」
- 説明：「すなわち」「つまり」「例えば」

#### 接続詞の使用頻度
- **1段落内**：接続詞は2-3回まで
- **連続使用**：同じ接続詞を連続で使用しない

### 3.3 指示語の使用

#### 基本原則
- **「これ」「それ」**：1段落内で1回まで
- **「この」「その」**：1段落内で2回まで
- **参照範囲**：直前の段落内の内容を指す

#### 使用例
```
❌ 悪い例：AI診断支援システムは診断精度を向上させる。これは医療現場で有用である。これは今後さらに普及するだろう。

✅ 良い例：AI診断支援システムは診断精度を向上させる。このシステムは医療現場で有用である。今後さらに普及する可能性が高い。
```

### 3.4 主語・述語の関係

#### 基本原則
- **主語の明示**：省略しない（特に医療教育・学術論文）
- **主語と述語の距離**：できるだけ近くに配置
- **受動態の使用**：適切な場合のみ（学術論文では多用可）

#### 使用例
```
❌ 悪い例：AI診断支援システムによる診断精度の向上が、医療現場での活用を促進している。

✅ 良い例：AI診断支援システムは診断精度を向上させる。このため、医療現場での活用が促進されている。
```

## 第4章｜視覚設計ルール（noteで差が出る）

### 4.1 改行のルール

#### 基本原則
- **段落間**：1行空ける
- **見出しの前後**：前後1行ずつ空ける
- **箇条書きの前後**：前後1行ずつ空ける

#### 改行のタイミング
- **話題の転換時**：新しい話題に移る際は必ず改行
- **長文の分割時**：長い段落は適切に分割

### 4.2 段落の長さ

#### 基本原則
- **技術解説・ブログ**：2-4文（100-200文字）
- **医療教育**：4-6文（200-300文字）
- **学術論文**：5-7文（250-350文字）

#### 段落の分割方法
- **1段落1主題**：1つの段落は1つの主題に集中
- **主題の転換時**：新しい主題に移る際は必ず改段落

### 4.3 見出しの使用

#### 基本原則
- **階層構造**：適切な階層構造を維持（H1→H2→H3）
- **見出しの頻度**：長い記事では適切に使用（500-1000文字ごとに1つ）
- **見出しの内容**：そのセクションの内容を簡潔に表現

#### 見出しの書き方
```
❌ 悪い例：AI診断支援システムについて（掴み？）

✅ 良い例：AI診断支援システムの現状と課題
```

### 4.4 箇条書きの使用

#### 基本原則
- **使用タイミング**：複数の項目を列挙する際
- **項目数**：3-7項目が最適
- **項目の長さ**：1項目は1-2文（50-100文字）

#### 箇条書きの書き方
```
✅ 良い例：
- 早期抗菌薬投与（エビデンスレベル：A）
- 輸液負荷（エビデンスレベル：B）
- 血管作動薬の使用（エビデンスレベル：C）
```

## 第5章｜用語・ワークフロー完全テンプレ

### 5.1 用語定義のテンプレート

#### 基本テンプレート
```
【用語名】
定義：[明確な定義を1文で記述]
補足：[必要に応じて補足説明]
例：[具体例があれば記述]
```

#### 実践例
```
【敗血症性ショック】
定義：感染症に起因する循環不全により、組織灌流が維持できなくなった状態を指す。
補足：SOFAスコア2点以上で敗血症と診断する。
例：肺炎に伴う敗血症性ショックでは、早期抗菌薬投与が重要である。
```

### 5.2 ワークフロー説明のテンプレート

#### 基本テンプレート
```
【ワークフロー名】
目的：[このワークフローの目的を1文で記述]
手順：
1. [手順1]：[説明]
2. [手順2]：[説明]
3. [手順3]：[説明]
注意点：[注意すべき点があれば記述]
```

#### 実践例
```
【AI診断支援システムの導入ワークフロー】
目的：医療現場にAI診断支援システムを安全に導入する。
手順：
1. システム選定：診断精度、安全性、コストを総合的に評価する。
2. パイロット運用：限定的な範囲で試行的に運用し、問題点を洗い出す。
3. 本格導入：パイロット運用の結果を踏まえ、段階的に本格導入する。
注意点：医師の判断を補完するツールとして位置づけ、過度な依存を避ける。
```

# ChatGPT APIをPythonで呼び出す最小実装

この記事を読むと、ChatGPT APIをPythonで呼び出す最小実装ができるようになります。

## 前提条件
- Python 3.8以上
- OpenAI APIキー
- requestsライブラリ

## 実装手順

### 1. ライブラリのインストール
まず、requestsライブラリをインストールします。

```bash
pip install requests
```

### 2. APIキーの設定
環境変数にAPIキーを設定します。

```bash
export OPENAI_API_KEY="your-api-key-here"
```

### 3. 最小実装コード
以下のコードをコピーして実行してください。

```python
import os
import requests

api_key = os.getenv("OPENAI_API_KEY")
url = "https://api.openai.com/v1/chat/completions"

headers = {
    "Authorization": f"Bearer {api_key}",
    "Content-Type": "application/json"
}

data = {
    "model": "gpt-3.5-turbo",
    "messages": [
        {"role": "user", "content": "Hello, world!"}
    ]
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```

## よくあるエラー
- **エラー：APIキーが無効** → 原因：環境変数の設定漏れ
- **エラー：リクエストが失敗** → 原因：ネットワーク接続の問題
```

## 定義
敗血症性ショックとは、感染症に起因する循環不全により、組織灌流が維持できなくなった状態を指す。

## 診断基準
SOFAスコア2点以上で敗血症と診断する。ショックの診断には、輸液負荷後も平均血圧65 mmHg未満が持続することが必要である。

## 病態機序
感染症により、サイトカインストームが引き起こされる。これにより、血管拡張と血管透過性亢進が生じ、循環不全に至る。

## 治療
### 早期抗菌薬投与
推奨：早期抗菌薬投与（エビデンスレベル：A）
根拠：抗菌薬投与の遅延は死亡率を上昇させる。

### 輸液負荷
推奨：初期輸液負荷（エビデンスレベル：B）
根拠：循環血液量の確保が重要である。

## 注意点
CRP上昇のみでは敗血症を診断できない。必ずSOFAスコアを計算し、総合的に判断する。

AI診断支援システムは、医師の判断を補完するツールとして、今後さらに普及する可能性が高いです。次回の診察時には、AI診断支援の可能性について主治医に尋ねてみてください。

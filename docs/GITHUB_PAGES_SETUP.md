# GitHub Pages設定ガイド：あなたのプロジェクトを世界に公開しよう！

「この素晴らしいプロジェクト、みんなにも見てほしい！」

そう思ったら、GitHub Pagesの出番です。まるで、あなたのリポジトリに魔法をかけて、世界中の誰もがアクセスできるウェブサイトに変えてしまうようなもの。

このガイドでは、その魔法をかけるための呪文（設定）を、一つひとつ丁寧に解説していきます。

![GitHub Pages設定ガイド](https://files.manuscdn.com/user_upload_by_module/session_file/90346460/JDtUEirAXjtOTUzX.png)

---

## 1. 最初の呪文：Viteの設定

まず、プロジェクトの「住所」を決めます。GitHub Pagesでは、`https://<ユーザー名>.github.io/<リポジトリ名>/`という住所になるので、その設定をします。

- **`vite.config.ts`を開く**：プロジェクトの設計図です。
- **`base: '/Helix/'` を追記**：これで、あなたのプロジェクトの住所が `/Helix/` であることをViteに教えます。

---

## 2. 自動化の魔法：GitHub Actions

次に、あなたが`main`ブランチに新しい変更を加えるたびに、自動でウェブサイトを更新してくれる魔法のロボット（GitHub Actions）を設定します。

- **`.github/workflows/deploy.yml`を作成**：ロボットへの指示書です。
- **「mainブランチにpushされたら、ビルドしてデプロイしてね」と記述**：これで、あなたはもう手動でデプロイする必要はありません。

---

## 3. いざ、公開！：デプロイフロー

設定は完了しました。あとは、あなたが`main`ブランチにpushするだけ。

1. **`main`ブランチにpush**
2. **ロボットが自動でビルド＆デプロイ**
3. **あなたのウェブサイトが世界に公開！**

---

## 4. ちゃんと動いてる？：動作確認

魔法が正しくかかったか、確認しましょう。

- **リポジトリの「Settings」→「Pages」を開く**
- **「Your site is live at ...」というメッセージとURLが表示されていれば成功！**

もし「404 Not Found」と表示されたら、ロボットへの指示書（`deploy.yml`）が正しいか、もう一度確認してみてください。

---

## 5. 賢い開発スタイル：ブランチ戦略

- **`main`ブランチ**：本番環境。ここにpushすると、世界中の人が見るウェブサイトが更新されます。
- **`feature/*`ブランチ**：開発用のブランチ。ここで自由に実験し、完成したら`main`にマージします。

これにより、開発中の未完成な状態が世界に公開されることなく、安心して開発を進めることができます。

さあ、これであなたもGitHub Pagesマスターです。どんどんプロジェクトを開発し、世界に発信していきましょう！

# Sentry設定ガイド：あなたのアプリの頼れる「見張り番」を迎えよう！

「アプリがユーザーの手元でエラーを起こしていないか、心配…」

開発者なら誰でも抱えるそんな不安を、24時間365日、あなたの代わりに監視してくれる頼もしい「見張り番」、それがSentryです。ユーザーがバグに遭遇した瞬間、Sentryは見逃さずにキャッチし、何が起きたのかを詳細にレポートしてくれます。

このガイドでは、そんな頼れる見張り番「セントリー・ウォッチマン」を、あなたのアプリケーションに配属するための手順を、一緒に見ていきましょう。

---

## 🚀 セントリー・ウォッチマン配属までの4ステップ

![Sentry設定ガイド](https://files.manuscdn.com/user_upload_by_module/session_file/90346460/PToPRGEotaqWtVuI.png)

### ステップ1：Sentryの基地で隊員登録（アカウント作成）

まずは、ウォッチマンが所属するSentryの公式サイトで、あなた自身のアカウントを作成します。

1.  **[Sentry公式サイト](https://sentry.io/)にアクセス**
2.  メールアドレスとパスワードを登録してサインアップ。これであなたもSentry部隊の一員です！

### ステップ2：ウォッチマンの配属先を決める（プロジェクト設定）

次に、新しく配属するウォッチマンが担当するプロジェクト（アプリケーション）を登録します。

1.  Sentryダッシュボードで「新しいプロジェクト」を作成します。
2.  プラットフォームとして「React」を選びます。
3.  設定が完了すると、そのプロジェクト専用の「**DSNキー**」という合言葉が発行されます。これは、ウォッチマンが基地（Sentryダッシュボード）と通信するための大切な鍵です。

### ステップ3：ウォッチマンをアプリに配属！（コードに組み込み）

いよいよ、ウォッチマンをあなたのアプリケーションのコードに常駐させます。

1.  まず、Sentryを動かすためのパッケージをインストールします。
    ```bash
    pnpm add @sentry/react
    ```
2.  アプリケーションのメインファイル（`main.tsx`など）で、先ほど取得したDSNキーを使ってSentryを初期化します。
    ```javascript
    import * as Sentry from "@sentry/react";

    Sentry.init({
      dsn: "あなたのDSNキーをここに貼り付け",
      // ...その他の設定
    });
    ```

これで、ウォッチマンはあなたのアプリの隅々まで見守り始めます。

### ステップ4：監視レポートを確認しよう！（エラー監視）

配属は完了です！

これからは、アプリケーションで予期せぬエラーが発生すると、セントリー・ウォッチマンがリアルタイムでバグを捕獲し、詳細なレポートをSentryダッシュボードに送信してくれます。いつ、どこで、どんなエラーが起きたのか、一目瞭然。アプリの健康状態を常に最高の状態に保つことができます。

さあ、あなたもSentryと一緒に、ユーザーに最高の体験を届けましょう！

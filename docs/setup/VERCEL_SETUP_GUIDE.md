# Vercel設定ガイド：ブランチごとのプレビューURLを使いこなそう！

「この修正、実際に動かしてみたらどう見えるんだろう？」

開発中に、そんな風に思ったことはありませんか？Vercelを使えば、ブランチをプッシュするだけで、魔法のようにプレビュー用のURLが自動で発行されます。まるで、未来の完成形を覗き見るような体験です。

このガイドでは、その魔法を手に入れるためのステップを、一つひとつ丁寧に解説していきます。



---

## 1. Vercelの世界へようこそ！ (アカウント作成)

![Vercel設定ガイド](https://files.manuscdn.com/user_upload_by_module/session_file/90346460/kGNtlTcmTyfsbzNU.png)

まずは、Vercelの世界への扉を開きましょう。冒険の始まりは、アカウント作成からです。

1. **[Vercel公式サイト](https://vercel.com)にアクセス**
2. **「Continue with GitHub」をクリック**：一番簡単でオススメの方法です。あなたのGitHubアカウントと連携させるだけで、準備は完了です。

---

## 2. あなたのプロジェクトをVercelに招待しよう (プロジェクトのインポート)

次に、あなたの傑作（`kgraph57/medicalprompthub`リポジトリ）をVercelに招待します。

1. **「Add New...」→「Project」を選択**：ダッシュボードの目立つ場所にあるボタンから、新しいプロジェクトを追加します。
2. **リポジトリを選択**：あなたのリポジトリ一覧が表示されるので、`kgraph57/medicalprompthub`を選びましょう。もし見つからなければ、「Configure GitHub App」でVercelがあなたのリポジトリにアクセスできるよう、許可を与えてあげてください。

---

## 3. プロジェクトの「設計図」を確認しよう (プロジェクト設定)

Vercelはとても賢いので、ほとんどの設定を自動で読み取ってくれます。ここでは、その「設計図」が正しいか、一緒に確認していきましょう。

- **Framework Preset**: `Vite`
- **Root Directory**: `./` (そのままでOK)
- **Build Command**: `pnpm build:client`
- **Output Directory**: `dist`
- **Install Command**: `pnpm install`

まるで、優秀なアシスタントが、あなたのプロジェクトに合わせて最適な設定を提案してくれるようです。将来的にAPIキーなどが必要になったら、「Environment Variables」にそっと追加してあげましょう。

---

## 4. いざ、打ち上げ！ (デプロイ)

準備は整いました。いよいよ、あなたのプロジェクトを世界に向けて打ち上げます！

1. **「Deploy」ボタンをクリック**：このボタン一つで、ビルドから公開までが全自動で始まります。
2. **打ち上げの様子を見守る**：ビルドログがリアルタイムで表示され、ロケットが宇宙へ向かうように、あなたのプロジェクトがデプロイされていく様子を見守ることができます。通常、2〜3分で完了します。
3. **最初のURLが誕生！**：完了すると、`medical-prompt-hub.vercel.app`のような、記念すべき本番URLが発行されます。

---

## 5. ブランチごとに「別の未来」を覗き見よう (プレビュー環境)

ここからがVercelの真骨頂です。新しいブランチをプッシュするたびに、そのブランチだけの「別の未来」を覗き見ることができます。

```bash
# 新しい冒険（ブランチ）を始める
git checkout -b feature/new-idea

# 開発し、歴史に刻む（コミット＆プッシュ）
git add .
git commit -m "feat: 新しいアイデアを実装！"
git push origin feature/new-idea
```

これだけで、Vercelが自動的にプレビューURL（例：`new-idea-xxx.vercel.app`）を作成し、GitHubのPull Requestに「見てみて！」とコメントしてくれます。

### プレビュー環境のすごいところ

- **本番そっくりな環境で試せる**：ローカル環境とは違う、リアルな環境で動作確認ができます。
- **とにかく楽ちん**：プッシュするだけで、あとは全自動。面倒な手作業は一切ありません。
- **いろんな未来を同時に比較**：ブランチAとブランチB、どちらが良いか、実際のURLで並べて比較検討できます。

---

## 🔧 もしも、の時の「お守り」 (トラブルシューティング)

冒険にトラブルはつきもの。でも、ご安心を。困った時のための「お守り」です。

- **ビルドがうまくいかない時**：Vercelダッシュボードの「Build Logs」に、失敗の原因が必ず書かれています。まずはログを読んで、何が起きているのか確認しましょう。
- **プレビューURLが作られない時**：Vercelの「Settings」→「Git」で、ブランチの自動デプロイが有効になっているか確認してみてください。

さあ、これであなたもVercelマスターです。ブランチを切って、プッシュして、どんどん新しいアイデアを試していきましょう！

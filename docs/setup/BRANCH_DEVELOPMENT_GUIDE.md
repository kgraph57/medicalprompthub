# 📖 ブランチ開発ガイド：安全な冒険の書

## 🚀 はじめに：なぜ「ブランチ」という冒険に出るのか？

新しい機能を開発したり、ちょっとした修正を試したりするとき、いきなり本番環境（`main`ブランチ）を触るのは、まるで街のど真ん中で危険な実験をするようなもの。もし失敗したら、街は大混乱に陥ってしまいます。

そこで登場するのが**「ブランチ」**という、あなただけの**「安全な作業島」**です。この島では、何をしてもOK。失敗を恐れずに、自由に試行錯誤できます。このガイドは、そんな安全な冒険の進め方を記した、あなたのための「冒険の書」です。



---

## 🗺️ 冒険のステップ：6つのステップで安全に開発を進めよう

![ブランチ開発ガイド](https://files.manuscdn.com/user_upload_by_module/session_file/90346460/JlwnQmpsxsKsKwju.png)

### ステップ1：ブランチ作成（安全な作業島へ）

まずは、本番環境である`main`ブランチという「城」から、あなた専用の`feature`ブランチという「作業島」へと橋をかけます。これで、本番環境に一切影響を与えない、安全な開発環境が手に入ります。

```bash
# 今いる場所を確認（まだ城の中）
$ git branch --show-current
main

# 新しい作業島（ブランチ）を作って、そこへ移動する
$ git checkout -b feature/new-adventure
Switched to a new branch 'feature/new-adventure'
```

### ステップ2：開発（自由に試行錯誤！）

作業島では、あなたは王様です。新しい機能のアイデアを形にしたり、様々な実験をしたり、心ゆくまで試行錯誤を繰り返しましょう。失敗しても、この島が爆発するだけ。城はびくともしません。

### ステップ3：テスト（自分の島で徹底的に試す）

開発した機能が思い通りに動くか、作業島の中で徹底的にテストします。ロボットがちゃんと歩くか、変な動きをしないか、隅々までチェックしましょう。

```bash
# ローカルの開発サーバーを起動して、ブラウザで動きを確認
$ pnpm dev
```

### ステップ4：同期（本番の最新情報を取り込む）

あなたが作業島で開発している間にも、城では他の冒険者たちが新しい変更を加えているかもしれません。その最新情報をドローンで受け取り、あなたの開発内容と合体させます。これで、いざ城に戻るときに「話が違う！」なんてことになりません。

```bash
# 一度、城（main）に戻って最新情報を受け取る
$ git checkout main
$ git pull origin main

# 再び自分の作業島（feature）に戻る
$ git checkout feature/new-adventure

# 最新情報を自分の開発内容と合体させる
$ git merge main
```

### ステップ5：プルリクエスト（完成品を本番へ送る準備）

開発した機能が完璧に仕上がったら、いよいよ城に「こんなすごいものができました！」とお披露目する準備です。これを**「プルリクエスト」**と呼びます。完成品をドローンに乗せて、城の門番にレビューをお願いするイメージです。

```bash
# 変更を記録（コミット）して、GitHubに送る（プッシュ）
$ git add .
$ git commit -m "feat: 新機能『空飛ぶロボット』が完成！"
$ git push origin feature/new-adventure
```

この後、GitHubのサイトでプルリクエストを作成します。

### ステップ6：マージ（レビューOK！本番に仲間入り）

城の門番（他の開発者やリーダー）があなたの完成品をレビューし、「素晴らしい！ぜひ城に迎え入れよう」となれば、あなたのブランチは`main`ブランチに合流（マージ）されます。これにて、あなたの開発した機能が、正式に本番環境の一部となるのです。おめでとうございます！

---

## 🎉 まとめ：ブランチ冒険の心得

- **本番は鉄壁の守り**：ブランチを使えば、本番環境は常に安全です。
- **失敗は成功のもと**：作業島では、何度失敗しても大丈夫。恐れず挑戦しましょう。
- **準備と確認を怠らない**：テストと同期をしっかり行い、万全の状態で城に戻りましょう。

さあ、あなたもブランチという名の冒険へ旅立ち、素晴らしい機能を世界に届けましょう！

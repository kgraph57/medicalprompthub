# 症例報告作成ワークフロー：AIを活用した効率化ガイド

## 記事の目的
症例報告（Case Report）の作成プロセスを、AI（Medical Prompt Hub）を活用して効率化する具体的な手順を解説する。単なるプロンプトの羅列ではなく、実際の執筆フローに沿って「どのタイミングで」「どのプロンプトを」「どのように使うか」をガイドする。

## ターゲット読者
- 初めて症例報告を書く研修医・若手医師
- 英語論文の執筆に不慣れな医師
- 忙しい臨床の合間に効率的に論文を書きたい医師

## 記事構成案

### 1. はじめに：症例報告は「400m走」である
- 症例報告はマラソン（原著論文）ではなく、短距離走（400m走）であるべき。
- 適切なツールと手順があれば、驚くほど短時間で完走できる。
- 本ガイドでは、Medical Prompt Hubを使った具体的なワークフローを紹介する。

### 2. Phase 1: 素材の準備とCAREチェック (Start Dash)
- **タスク**: カルテ情報の整理と不足情報の確認。
- **使用プロンプト**: `res-check-care` (CAREチェックリスト)
- **手順**:
    1. カルテの経過（現病歴、検査、治療、転帰）をざっくりまとめる。
    2. `res-check-care` プロンプトに入力し、CAREガイドライン上の不足点（患者の視点、タイムラインなど）を洗い出す。
    3. 不足情報をカルテから追記する。

### 3. Phase 2: 症例提示の執筆 (Case Presentation)
- **タスク**: 英語でのCase Presentationセクションの作成。
- **使用プロンプト**: `case-presentation` (症例提示作成)
- **手順**:
    1. 日本語の経過要約を `case-presentation` プロンプトに入力。
    2. 英語のCase Presentationドラフトを出力させる。
    3. 医学用語や数値の誤りがないか人間がチェック。

### 4. Phase 3: 考察の構成と文献検索 (Discussion & Search)
- **タスク**: 考察のロジック構築と文献検索。
- **使用プロンプト**: `res-pubmed-query` (PubMed検索), `res-gap-analysis` (ギャップ特定)
- **手順**:
    1. 「なぜこの症例が重要か？」を言語化する。
    2. `res-pubmed-query` で検索式を作成し、類似症例を検索。
    3. 類似症例との違い（新規性）を明確にする。

### 5. Phase 4: 考察の執筆と推敲 (Writing Discussion)
- **タスク**: Discussionセクションの執筆。
- **使用プロンプト**: `res-intro-flow` (構成案), `res-ref-format-convert` (引用整形)
- **手順**:
    1. `res-intro-flow` を応用してDiscussionのパラグラフ構成を作る（要約→類似症例との比較→機序→結論）。
    2. 各パラグラフを執筆（または翻訳プロンプト活用）。
    3. `res-ref-format-convert` で引用文献リストを投稿規定に合わせる。

### 6. Phase 5: 投稿準備 (Submission)
- **タスク**: タイトル、抄録、カバーレター作成。
- **使用プロンプト**: `res-cover-letter` (カバーレター作成)
- **手順**:
    1. 論文全体の内容から魅力的なタイトル案を作成。
    2. Abstractを作成（文字数制限に注意）。
    3. `res-cover-letter` でエディターへのアピール文を作成。

### 7. おわりに：AIは「共著者」
- AIは執筆を代行するだけでなく、思考の整理や品質管理（ガイドラインチェック）のパートナーとなる。
- このフローを一度体験すれば、次からはもっと速く走れるはず。

## 実装機能
- 各フェーズで紹介するプロンプトは、記事内のボタンから直接「Medical Prompt Hub」の該当プロンプトページ（入力フォーム済み）に飛べるようにする、または記事内で直接コピーできるようにする。

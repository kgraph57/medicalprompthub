# モバイルUI実装ドキュメント

## 概要

医療従事者向けAIプロンプトライブラリのウェブサイトに、Manusのスマホアプリのようなモバイル体験を実装しました。

## 実装内容

### 1. ボトムナビゲーション

画面下部に主要5項目を配置したネイティブアプリライクなナビゲーションを実装。

**配置項目**：
- **Home**：ホームページ
- **Courses**：コース一覧
- **Tips**：Tips一覧
- **Workflow**：ワークフローガイド
- **Fav**：お気に入り

**デザイン仕様**：
- 高さ：64px（16 Tailwind単位）
- アイコンサイズ：20px × 20px
- フォントサイズ：10px
- アクティブ状態：プライマリカラー + アイコン塗りつぶし
- 非アクティブ状態：ミュートカラー

**実装ファイル**：
- `client/src/components/BottomNav.tsx`

### 2. スワイプジェスチャー

左からスワイプでサイドバーを開く、右にスワイプで閉じる機能を実装。

**仕様**：
- スワイプ閾値：50px
- モバイルのみ有効（画面幅 < 1024px）
- スムーズなアニメーション

**実装ファイル**：
- `client/src/hooks/useSwipeGesture.ts`

### 3. レイアウト調整

モバイルとデスクトップで最適なレイアウトを提供。

**モバイル（< 1024px）**：
- 上部：固定ヘッダー（高さ64px）
- 下部：ボトムナビゲーション（高さ64px）
- メインコンテンツ：上下にパディング（pt-16 pb-16）
- サイドバー：ハンバーガーメニューからオーバーレイ表示

**デスクトップ（≥ 1024px）**：
- 左側：展開・折りたたみ可能なサイドバー
- メインコンテンツ：サイドバーの幅に応じてマージン調整
- ボトムナビゲーション：非表示

### 4. アニメーション

ネイティブアプリのような滑らかな動きを実現。

**適用箇所**：
- サイドバーの開閉：300ms transition
- ボトムナビゲーションのアクティブ状態：色変化アニメーション
- スワイプジェスチャー：スムーズな追従

## 技術スタック

- **React**：コンポーネントベースのUI構築
- **Tailwind CSS**：レスポンシブデザイン
- **Wouter**：軽量ルーティング
- **Lucide React**：アイコンライブラリ

## ファイル構成

```
client/src/
├── components/
│   ├── BottomNav.tsx          # ボトムナビゲーションコンポーネント
│   └── Layout.tsx             # レイアウトコンポーネント（更新）
└── hooks/

```

## 使用方法

### ローカル開発

```bash
cd /home/ubuntu/medicalprompthub
pnpm dev
```

ブラウザで `http://localhost:3000/medicalprompthub/` にアクセス。

### モバイル表示確認

1. **ブラウザのデベロッパーツール**を開く（F12）
2. **デバイスツールバー**を有効化（Ctrl+Shift+M / Cmd+Shift+M）
3. **モバイルデバイス**を選択（例：iPhone 12 Pro）
4. 以下を確認：
   - 画面下部にボトムナビゲーションが表示
   - 左からスワイプでサイドバーが開く
   - 右にスワイプでサイドバーが閉じる
   - ボトムナビゲーションのタップでページ遷移

## デザインの特徴

### Manusライクな体験

1. **直感的なナビゲーション**：画面下部の親指で届きやすい位置に配置
2. **スワイプジェスチャー**：ネイティブアプリのような操作感
3. **スムーズなアニメーション**：遅延なく反応する快適なUI
4. **コンパクトなデザイン**：必要な情報だけを表示

### アクセシビリティ

- **aria-label**：すべてのボタンに適切なラベル
- **セマンティックHTML**：nav要素の使用
- **キーボードナビゲーション**：デスクトップでも操作可能

## パフォーマンス

- **軽量なコンポーネント**：不要な再レンダリングを防止
- **CSS Transition**：GPU加速されたアニメーション
- **条件付きレンダリング**：画面サイズに応じて最適なUIを表示

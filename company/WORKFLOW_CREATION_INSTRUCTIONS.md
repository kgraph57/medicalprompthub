# ワークフロー作成指示書（AI向け）

**最終更新日**: 2025年12月5日

このドキュメントは、他のAIアシスタントに対して、新しいワークフローを作成するよう依頼する際に使用する指示書です。

---

## 依頼の概要

Medical Prompt Hubやpediatric-exam-appなどのサービス開発チームでは、各種活動（アプリ開発、書籍執筆、講演など）に対して、標準化されたワークフローを定義しています。

既存のワークフローは `DEVELOPMENT_WORKFLOW.md` に記載されており、詳細なガイドラインは `WORKFLOW_GUIDE.md` に記載されています。

新しいワークフローを追加する際は、既存のワークフローと同じ構造とフォーマットを維持してください。

---

## 依頼時の必須情報

新しいワークフローを作成する際は、以下の情報を含めて依頼してください：

1. **活動タイプ**: 例「ブログ記事執筆」「ウェビナー開催」「研究論文執筆」など
2. **活動の目的**: この活動で何を達成したいか
3. **医学的・医療関連の内容を含むか**: はい/いいえ
4. **外部に公開するか**: はい/いいえ
5. **技術的実装を伴うか**: はい/いいえ
6. **視覚的なデザインを伴うか**: はい/いいえ
7. **想定される期間**: 全体の期間（概算）

---

## 依頼例

```
以下のワークフローを作成してください：

**活動タイプ**: ブログ記事執筆
**活動の目的**: 医学・医療関連のブログ記事を執筆し、Webサイトに公開する
**医学的・医療関連の内容を含むか**: はい
**外部に公開するか**: はい
**技術的実装を伴うか**: いいえ
**視覚的なデザインを伴うか**: 一部あり（画像・図表）
**想定される期間**: 2-3週間

既存の `DEVELOPMENT_WORKFLOW.md` の構造を参考に、同じフォーマットでワークフローを作成してください。
特に、「書籍執筆ワークフロー」や「講演ワークフロー」を参考にしてください。
```

---

## 作成時の必須要件

1. **ファイルの場所**: `DEVELOPMENT_WORKFLOW.md` の末尾に追加してください

2. **構造の維持**: 既存のワークフローと同じ構造を使用してください：
   - ワークフロー概要と目的
   - フェーズ概要図（```コードブロックで）
   - 各フェーズの詳細（期間、主要チーム、目的、担当チームと役割、成果物、チェックポイント）

3. **チェックポイントの命名**: 
   - ブログ記事: `CP-Blog1`, `CP-Blog2`, ...
   - ウェビナー: `CP-Web1`, `CP-Web2`, ...
   - 研究論文: `CP-Paper1`, `CP-Paper2`, ...
   - その他: 適切な接頭辞を使用

4. **必須チェックの明記**:
   - 医学的監修が必要な場合: `承認条件: メディカルAI・倫理監修チームの承認（**必須**）`
   - 法務チェックが必要な場合: `承認条件: 法務・知財チームの承認（**必須**）`
   - 最終承認: `承認条件: 全参加チームの承認（**[活動名]可否の最終判断**）`

5. **ワークフロー比較表への追加**: 
   - 「ワークフロー比較表」セクションに新しい活動を追加してください

## 参考すべき既存ワークフロー

### 書籍執筆ワークフロー（コンテンツ作成系）
- 企画 → 構成 → 執筆 → レビュー → 医学的監修 → 法務チェック → 最終承認 → 公開
- 詳細なレビューとチェックプロセス
- 医学的監修と法務チェックが必須

### 講演ワークフロー（発表系）
- 企画 → アウトライン → スライド作成 → 医学的監修 → 法務チェック → リハーサル → 実施 → 事後レビュー
- スライドデザインを含む
- 比較的短い期間

### アプリ開発ワークフロー（技術実装系）
- 企画 → 市場調査 → 戦略 → 技術選定 → デザイン → 開発 → テスト → セキュリティ → リリース → 運用
- 複数の技術チームが関与
- 長期的なプロジェクト

## チームの役割（簡易版）

### 常に関与
- **プロダクト戦略チーム**: 目的と価値提案
- **プロジェクト管理チーム**: プロセス管理

### 医学的・医療関連の内容がある場合（必須）
- **メディカルAI・倫理監修チーム**: 医学的信頼性、安全性、倫理性

### 外部公開する場合（必須）
- **法務・知財チーム**: 著作権、プライバシー、免責事項
- **コンテンツ・編集チーム**: 言語表現、編集品質

### 技術実装がある場合（必須）
- **天才エンジニアチーム**: 技術実装
- **品質保証・テストチーム**: 品質確認

### デザインが必要な場合
- **デザインチーム**: UI/UX、視覚的デザイン

### マーケティング・プロモーションがある場合
- **マーケティング・成長チーム**: マーケティング戦略

### データ分析・測定がある場合
- **データ分析・BIチーム**: メトリクス設計、データ分析

### 規制対象の場合（必須）
- **医療規制・コンプライアンスチーム**: 規制対応

## チェックポイント設計のガイドライン

### 必ず設定すべきチェックポイント

1. **企画承認**: 活動開始時（プロダクト戦略チームが主導）
2. **医学的監修承認**: 医学的・医療関連の内容がある場合（メディカルAI・倫理監修チームが主導、**必須**）
3. **法務・知財チェック承認**: 外部公開する場合（法務・知財チームが主導、**必須**）
4. **最終承認**: リリース/公開前（全関係チームが参加、**最終判断**）

### その他のチェックポイント

- 構成・設計の承認
- 中間レビュー（長期プロジェクトの場合）
- 品質確認の承認
- リリース準備完了の承認

---

## 成果物の定義

各フェーズの成果物は、具体的で確認可能なものを記載してください：

- [ ] [成果物名]（簡潔な説明）

例：
- [ ] 書籍企画書（タイトル、副題、テーマ、ターゲット読者）
- [ ] 医学的監修レポート
- [ ] エビデンスリスト

---

## 期間の目安

- **企画・構想**: 1-2週間
- **設計・作成**: 2-8週間（規模による）
- **レビュー・チェック**: 1-4週間（深さによる）
- **修正**: 1-3週間
- **最終承認**: 1週間
- **リリース準備**: 1-2週間

---

## 作成後の確認事項

ワークフローを作成したら、以下を確認してください：

- [ ] 既存のワークフローと同じ構造になっている
- [ ] フェーズ概要図がある
- [ ] 各フェーズに「期間」「主要チーム」「目的」が記載されている
- [ ] 各フェーズに「担当チームと役割」が記載されている
- [ ] 各フェーズに「成果物」が記載されている
- [ ] 重要なタイミングに「チェックポイント」が設定されている
- [ ] 医学的監修が必要な場合、必須チェックポイントが設定されている
- [ ] 法務チェックが必要な場合、必須チェックポイントが設定されている
- [ ] 最終承認のチェックポイントが設定されている
- [ ] チェックポイントの命名規則に従っている
- [ ] ワークフロー比較表に追加されている

# ワークフロー作成ガイド

**最終更新日**: 2025年12月5日

このドキュメントは、新しい活動タイプに対するワークフローを作成する際のガイドラインです。既存のワークフロー（アプリ開発、書籍執筆、講演）を参考に、同じ構造とフォーマットで新しいワークフローを追加できます。

---

## 既存ワークフローの一覧

現在、以下のワークフローが定義されています：

1. **アプリ開発ワークフロー**（フェーズ1-10）
2. **書籍執筆ワークフロー**（フェーズ1-10）
3. **講演ワークフロー**（フェーズ1-8）

各ワークフローは `DEVELOPMENT_WORKFLOW.md` に記載されています。

---

## ワークフロー構造の基本要素

新しいワークフローを作成する際は、以下の構造を必ず含めてください：

### 1. ワークフロー概要セクション

```
## [活動名]ワークフロー

**目的**: [この活動を行う目的と、このワークフローが解決する問題]
```

### 2. フェーズ概要図

フェーズの流れを視覚的に示す図を追加：

```
[フェーズ1名]
    ↓
[フェーズ2名]
    ↓
...
```

### 3. 各フェーズの詳細

各フェーズには以下を含める：

#### フェーズヘッダー
```
### フェーズ[N]: [フェーズ名]

**期間**: [期間]  
**主要チーム**: [主要な担当チーム]  
**目的**: [このフェーズの目的]
```

#### 担当チームと役割
```
#### [N].[M] 担当チームと役割

**[チーム名]（主導/参加）**
- **[メンバー名]**: [具体的な役割]
  - [具体的なタスクや確認項目]
```

#### 成果物
```
#### [N].[M] 成果物
- [ ] [成果物1]
- [ ] [成果物2]
```

#### チェックポイント
```
#### [N].[M] チェックポイント
**[CP-[N]]: [チェックポイント名]**
- **参加チーム**: [参加するチームリスト]
- **判断基準**: 
  - [基準1]
  - [基準2]
- **承認条件**: [承認条件]（必要に応じて「**必須**」や「**承認可否の最終判断**」を明記）
```

## チーム別の関与タイミング

以下のチームが関与する可能性があります。活動タイプに応じて適切なチームを選択してください：

### 必ず関与すべきチーム

- **プロダクト戦略チーム**: 活動の目的と価値提案の明確化
- **プロジェクト管理チーム**: プロセス管理と進捗管理

### 内容によって関与すべきチーム

#### 医学的・医療関連の内容を含む場合（**必須**）
- **メディカルAI・倫理監修チーム**: 医学的信頼性、安全性、倫理性の確認

#### 外部に公開するコンテンツを作成する場合（**必須**）
- **法務・知財チーム**: 著作権、プライバシー、免責事項の確認
- **コンテンツ・編集チーム**: 言語表現、編集品質の確認

#### ユーザー向け製品・サービスを開発する場合（**必須**）
- **天才エンジニアチーム**: 技術実装
- **デザインチーム**: UI/UX設計（視覚的な要素がある場合）
- **品質保証・テストチーム**: 品質確認

#### マーケティング・プロモーションを伴う場合
- **マーケティング・成長チーム**: マーケティング戦略、プロモーション計画

#### データ分析・測定を伴う場合
- **データ分析・BIチーム**: メトリクス設計、データ分析

#### 医療機器や規制対象の製品を扱う場合（**必須**）
- **医療規制・コンプライアンスチーム**: 規制対応

## チェックポイントの設計原則

### 必須チェックポイント

以下のタイミングでは必ずチェックポイントを設定してください：

1. **企画・開始時**: 活動の目的と方向性が明確か
2. **医学的・倫理的チェック（該当する場合）**: 医学的信頼性と安全性
3. **法務・知財チェック（外部公開する場合）**: 法的リスク
4. **最終承認**: リリース/公開可否の最終判断

### チェックポイントの命名規則

- アプリ開発: `CP1`, `CP2`, ..., `CP12`
- 書籍執筆: `CP-B1`, `CP-B2`, ..., `CP-B7`
- 講演: `CP-T1`, `CP-T2`, ..., `CP-T5`

新しいワークフローでは、適切な接頭辞を使用してください：
- ブログ記事: `CP-Blog1`, `CP-Blog2`, ...
- ウェビナー: `CP-Web1`, `CP-Web2`, ...
- 研究論文: `CP-Paper1`, `CP-Paper2`, ...

### 承認条件の重要度表記

- 通常の承認: `承認条件: 全参加チームの合意`
- 必須承認（この承認がないと次に進めない）: `承認条件: [チーム名]の承認（**必須**）`
- 最終判断（リリース/公開可否）: `承認条件: 全参加チームの承認（**[活動名]可否の最終判断**）`

## フェーズ設計のポイント

### 1. 企画・構想フェーズ
- プロダクト戦略チームが主導
- メディカルAI・倫理監修チーム（医療関連の場合）が参加
- 目的と価値提案を明確化

### 2. 設計・作成フェーズ
- コンテンツ・編集チームが主導（コンテンツ作成の場合）
- デザインチームが主導（デザインが必要な場合）
- 天才エンジニアチームが主導（技術実装の場合）
- 構成や設計のレビューを実施

### 3. 医学的監修・倫理チェック（医療関連の場合）
- メディカルAI・倫理監修チームが主導
- **必須チェックポイント**を設定
- エビデンス、安全性、倫理性を確認

### 4. 法務・知財チェック（外部公開する場合）
- 法務・知財チームが主導
- **必須チェックポイント**を設定
- 著作権、プライバシー、免責事項を確認

### 5. 品質確認フェーズ
- 品質保証・テストチームが主導（製品開発の場合）
- コンテンツ・編集チームが主導（コンテンツの場合）
- 最終的な品質確認

### 6. 最終承認フェーズ
- 全関係チームが参加
- **最終判断のチェックポイント**を設定
- リリース/公開可否を決定

### 7. リリース・実施フェーズ
- 実際のリリース/実施
- マーケティング・成長チーム（プロモーションが必要な場合）
- カスタマーサクセステーム（サポートが必要な場合）

### 8. 事後レビュー（継続的改善）
- フィードバック収集
- 改善点の整理
- 次の活動に活かす

## 成果物の定義

各フェーズの成果物は、以下のような形式で定義してください：

- [ ] [成果物名]（説明が不明確な場合は簡潔な説明を追加）

成果物は、次フェーズの入力となるもの、またはチェックポイントで確認するものを含めてください。

---

## 期間の目安

フェーズの期間は、活動の複雑さに応じて設定してください：

- **企画・構想**: 1-2週間
- **設計・作成**: 2-8週間（活動の規模による）
- **レビュー・チェック**: 1-4週間（チェックの深さによる）
- **修正**: 1-3週間
- **最終承認**: 1週間
- **リリース準備**: 1-2週間

## 追加すべきワークフローの候補

以下のようなワークフローが考えられます：

1. **ブログ記事執筆ワークフロー**
   - 企画 → 執筆 → 編集 → 医学的監修（医療記事の場合）→ 法務チェック → 公開

2. **ウェビナー・オンラインセミナーワークフロー**
   - 企画 → コンテンツ作成 → スライド作成 → 医学的監修 → 法務チェック → 実施 → 事後レビュー

3. **研究論文執筆ワークフロー**
   - 研究計画 → データ収集 → 分析 → 執筆 → 医学的監修 → 投稿準備 → 投稿

4. **プロンプトライブラリ追加ワークフロー**
   - プロンプト作成 → 技術的精度チェック → 医学的監修 → 法務チェック → 公開

5. **SNS投稿ワークフロー**
   - コンテンツ作成 → 簡易医学的チェック → 公開

6. **パートナーシップ・共同研究ワークフロー**
   - パートナー選定 → 契約検討 → 法務チェック → 実施 → レビュー

7. **製品アップデートワークフロー**
   - 変更企画 → 技術実装 → テスト → 医学的監修 → 法務チェック → リリース

## ワークフロー作成時のチェックリスト

新しいワークフローを追加する際は、以下を確認してください：

- [ ] ワークフロー概要と目的が明確に記載されている
- [ ] フェーズ概要図がある
- [ ] 各フェーズに「期間」「主要チーム」「目的」が記載されている
- [ ] 各フェーズに「担当チームと役割」が記載されている
- [ ] 各フェーズに「成果物」が記載されている
- [ ] 重要なタイミングに「チェックポイント」が設定されている
- [ ] 医学的・医療関連の内容がある場合、メディカルAI・倫理監修チームのチェックが含まれている
- [ ] 外部公開する場合、法務・知財チームのチェックが含まれている
- [ ] 最終承認のチェックポイントが設定されている
- [ ] チェックポイントの命名規則に従っている（CP-[接頭辞][番号]）
- [ ] 必須チェックポイントに「**必須**」が明記されている
- [ ] 最終判断のチェックポイントに「**[活動名]可否の最終判断**」が明記されている

## 質問事項

新しいワークフローを作成する際は、以下の質問に答えることで、適切なフェーズとチェックポイントを設計できます：

1. **この活動の目的は何か？**
2. **医学的・医療関連の内容を含むか？** → メディカルAI・倫理監修チームのチェックが必要
3. **外部に公開するか？** → 法務・知財チームのチェックが必要
4. **技術的実装を伴うか？** → 天才エンジニアチームが必要
5. **視覚的なデザインを伴うか？** → デザインチームが必要
6. **品質保証・テストが必要か？** → 品質保証・テストチームが必要
7. **マーケティング・プロモーションを伴うか？** → マーケティング・成長チームが必要
8. **データ分析・測定を伴うか？** → データ分析・BIチームが必要
9. **規制対象か？** → 医療規制・コンプライアンスチームが必要
10. **どのタイミングでどのチームの承認が必要か？**

## 注意事項

1. **柔軟性**: ワークフローはガイドラインです。プロジェクトの特性に応じて調整してください。

2. **並行実行**: 一部のフェーズは並行して進めることができますが、依存関係を考慮してください。

3. **必須チェックの厳格性**: 医学的監修や法務チェックなど、必須とされるチェックポイントは絶対にスキップしないでください。

4. **医療サービスの場合**: メディカルAI・倫理監修チームと医療規制・コンプライアンスチームの関与が必須となります。

5. **一貫性**: 既存のワークフローと同じフォーマットと構造を維持してください。
